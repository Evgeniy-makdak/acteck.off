Changing the state of a parent in a child component in React

In the previous lesson, the state with the data was stored in the parent component, and the child components received this data in the form of props.

Let us now want to change our products. Let's make, for example, a button that will place our product in the cart. First, let's add a field to our array with productsinCart, showing whether the product is in the cart or not:

const initProds = [ {id: id(), name: 'product1', cost: 100, inCart: false}, {id: id(), name: 'product2', cost: 200, inCart: false}, {id: id(), name: 'product3', cost: 300, inCart: false}, ];

In componentProductslet's add one more attribute to the product taginCart:

function Products() { const [prods, setProds] = useState(initProds); const items = prods.map(prod => { return <Product key ={prod.id} name ={prod.name} cost ={prod.cost} inCart={prod.inCart} />; }); return <div> {items} </div>; }

Let's in the child componentProductLet's display information about the cart and a button to add to cart:

function Product({ id, name, cost, inCart }) { return <div> name: <span>{name}</span>, cost: <span>{cost}</span>, <span>{inCart ? 'in cart' : 'not in cart'}</span> <button>to cart</button> </div>; }

Let's implement the addition

According to React rulesthe component should not change its props. This means that a child component cannot put itself in the trash by changing the propsinCart. This is not correct.

It would be correct to ask the parent component to change its stateprodsby adding a specific product to your cart.

Let's see how it's done.

Let's create a function in the parent componentaddToCart, which takes as a parameteridproduct and for this product changes the propertyinCartontrue:

function addToCart(id) { setProds(prods.map(prod => { if (prod.id === id) { prod.inCart = true; } return prod; })); }

To the product tag we will add an attribute to which we will pass the function we created, as well as an attribute to which we will passidproduct:

const items = prods.map(prod => { return <Product key ={prod.id} id ={prod.id} name ={prod.name} cost ={prod.cost} inCart ={prod.inCart} addToCart={addToCart} />; });

As you can see, you can pass not only some data to component props, but also functions.

Final class codeProductsit will turn out like this:

function Products() { const [prods, setProds] = useState(initProds); function addToCart(id) { setProds(prods.map(prod => { if (prod.id === id) { prod.inCart = true; } return prod; })); } const items = prods.map(prod => { return <Product key ={prod.id} id ={prod.id} name ={prod.name} cost ={prod.cost} inCart ={prod.inCart} addToCart={addToCart} />; }); return <div> {items} </div>; }

Now in the child class we will have access to the functionaddToCart. Let's call this function by clicking on the button, passing it the parameteridproduct:

function Product({ id, name, cost, inCart, addToCart }) { return <div> name: <span>{name}</span>, cost: <span>{cost}</span>, <span>{inCart ? 'in cart' : 'not in cart'}</span> <button onClick={() => addToCart(id)}>to cart</button> </div>; }

It turns out that when you click on the button in the child, the parent function will be called, which will change the parent state. Changing the parent state will cause a re-rendering and will resample our product, passing it the changed props.
