Introduction to React Components

Let's take a look at any website. It consists of a set of independent blocks: header, sidebars, footer, content. We can say that these blocks are components in the sense that is meant in React.

If you look at the same header, you can select a block with a logo, a block of contacts, a block with a menu, and so on. That is, components can consist of other subcomponents.

The situation is similar in React - a site is built from a set of components, which in turn can contain other components.

In React, each component is a separate module. Development usually starts with the main componentApp, which contains the rest.

Let's practice creating new components.

For example, let's say we need a component that displays product data. To do this, we need to create a file in the working folderProduct.jsand add the following code to it:

import React from 'react'; function Product() { return <p> product </p>; } export default Product;

As you can see, our component now returns a paragraph with text. In future tutorials, we'll modify this code to return product data. But for now, let's just start with a paragraph with some initial text.

Let's now output our created component in componentApp. Let it be ours nowApphas the following code:

import React from 'react'; function App() { return <div> text </div>; } export default App;

First, we need to import the product component we created:

import React from 'react'; import Product from './Product'; // импортируем продукт function App() { return <div> text </div>; } export default App;

After such import inside the componentAppyou can use the componentProduct.
