Features of returning multiple tags in JSX

ThroughreturnYou cannot return multiple tags at once. That is, the following code will not work:

function App() { return ( <div> <p>text1</p> <p>text2</p> </div> <div> <p>text3</p> <p>text4</p> </div> ); }

For the desired to work, we will have to take our tags into some common tag, for example, like this:

function App() { return ( <div> <div> <p>text1</p> <p>text2</p> </div> <div> <p>text3</p> <p>text4</p> </div> </div> ); }

This technique will work, however, it is not without its drawbacks: as a result of rendering, we will get an additional div, which we generally did not want and introduced solely for the correct operation of React. This div, for example, can break part of our layout.

To avoid such problems, React introduced a special empty tag that groups tags, but does not appear in the finished layout. Let's use this tag:

function App() { return ( <> <div> <p>text1</p> <p>text2</p> </div> <div> <p>text3</p> <p>text4</p> </div> </> ); }

The author of the following code wants to return two tags at onceul:

function App() { return <ul> <li>text1</li> <li>text2</li> <li>text3</li> </ul> <ul> <li>text1</li> <li>text2</li> <li>text3</li> </ul>; }

The code, however, doesn't work. Fix the code author's mistake. Suggest two ways to solve the problem.
