Problems with new ids in React

Suppose we received the following array of objects from the database:

const prods = [ {id: 1, name: 'product1', cost: 100}, {id: 2, name: 'product2', cost: 200}, {id: 3, name: 'product3', cost: 300}, ];

As you can see,idare numbered in order. However, we should take into account that numbers may have gaps, for example after3-go can go right away5-th or6-that one. Because of this, our client script cannot know what the next one will beid(in our case this is not necessary4).

Newidare created by the server database. This may cause some problems when running on the client. The point is this: imagine that using the form we added a new element to our array. However, we can’t just take and add data from the form - because we don’t know which oneidthe new element will have!

We will need to send a request to the server so that it gives us the following in orderid, and only then add the element to our array of objects. This will cause a delay in displaying the data on the screen: it will take some time until the data arrives to the server and until the server sends it back to us.
