One source of truth in React

For any changeable data in a React application there must be onesource of truth. What these words mean is that if you have two states and one state can be calculated through the other, then one of the states is redundant and needs to be removed.

Let's discuss this with an example. Let, for example, you have two inputs: the first one contains the temperature in degrees Fahrenheit, and the second - in degrees Celsius. At the same time, we want both inputs to be synchronized: when entering temperature into one input, it should change accordingly in the second.

The wrong approach would be to introduce two states: one for Fahrenheit temperature, and the second for Celsius. Why is it not correct: because one temperature is calculated from another, which means there should be one state as the data storage location.
