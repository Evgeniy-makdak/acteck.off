Types of Components in React

It is recommended to split components into two types in React:container components(smart, fat) andpresentational(stupid, thin).

Presentation components are designed to display data in HTML layout. Moreover, such components should not have states (except for states that regulate the operating mode of the component), and the data in them should be transferred externally using props. That is, such components simply receive data from their parent and display it in the required form.

Container components usually contain states with data, but do not display this data themselves, but use child presentation components for this. In this case, containers can be nested inside each other. That is, a container component can contain both presentational components and containers, in turn, containing presentational components.

The advantage of this approach is that presentational components can be easily reused, since such components do not know where the data comes from, but simply display it. This means that we can use the same component in different situations, passing data to it from different sources.
