Raising states in React

Often multiple components need to reflect the same changing data. In React, it is recommended to raise shared state to the nearest common ancestor. Let's look at an example.

Let's say we want to make a water temperature calculator. It will be an input in which the user will enter the temperature, and a paragraph in which the verdict will be displayed: liquid water, solid water, gaseous water.

Let our calculator be a container componentCalculator:

function Calculator() { return <div> </div>; }

Let's add an input for entering temperature into the componentTempInp, and the paragraph with the verdict - into the componentVerdict:

function Calculator() { return <div> <Verdict /> <TempInp /> </div>; }

It is easy to understand thatTempInp, AndVerdictmust have a state with temperature. At the same time, it is logical that this should be the same temperature: as data is entered into the input, the verdict should be displayed.

In this case, it is recommended to use a technique calledrise in fortune: state shared by two (or more) components is lifted up to their common parent.

In our case, it turns out that the state with temperature should belong to the componentCalculator, which will transmit it toTempInpAndVerdictas props:

function Calculator() { const [temp, setTemp] = useState(0); return <div> <Verdict temp={temp} /> <TempInp temp={temp} /> </div>; }

In componentTempInpthere must be an input to change the temperature. But, since this temperature is the props of this component, then directly inTempInpit cannot be changed. It would be correct to pass from the componentCalculatorspecial function for changing temperature:

function Calculator() { const [temp, setTemp] = useState(0); function changeTemp(event) { setTemp(event.target.value); } return <div> <Verdict temp={temp} /> <TempInp temp={temp} changeTemp={changeTemp} /> </div>; }

By the way, you can not introduce a new function, but pass the function to the child componentsetTemp:

function Calculator() { const [temp, setTemp] = useState(0); return <div> <Verdict temp={temp} /> <TempInp temp={temp} setTemp={setTemp} /> </div>; }
