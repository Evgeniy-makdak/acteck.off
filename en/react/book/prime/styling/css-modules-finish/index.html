Continued exploration of styling with CSS modules React

Let's continue working on our applicationbuttons. Now let's deal with the main componentApp, which has already been generated in the foldersrcinitially. In it we will have two divs, a title and a button.

Replace the file nameApp.cssonApp.module.cssBy convention, let's clean it up and create several classes in it with CSS styles for the tags:

.app { display: flex; flex-direction: column; width: 300px; border: 1px solid brown; padding: 10px; } .wrapper { display: flex; justify-content: space-around; } .title { text-align: center; margin-top: 10px; }

Now let's clear the contents of the fileApp.js. Then we import the React componentButtonsand styles fromApp.module.css. Let's write our tags and apply styles to them, and also place the React componentButtons:

import classes from "./App.module.css"; import Buttons from "./components/Buttons"; function App() { return ( <div class={classes.app}> <h2 class={classes.title}>CSS Module Buttons</h2> <div class={classes.wrapper}> <Buttons /> </div> </div> ); } export default App;

We used the wordclassesfor the name of the object containing the styles that we import fromApp.module.css. As in the previous case, you can name this object as you wish.

There are a couple of small steps left. Let's change the file nameindex.cssby agreement onindex.module.cssand in the fileindex.jsdon't forget to replace the import line:

Now we can see the result of our application in the browser.

If you open the generated layout in the developer panel in the browser, you will see that each component has its own unique classes generated. This way we no longer have to worry about conflicts between classes of individual components.

It is also important that CSS modules does not prohibit importing regular external CSS files.
