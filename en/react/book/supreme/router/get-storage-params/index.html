Loading page data using URL parameters from storage in React Router

Now we can add products to our application. Launch the application, add a couple of products using the button. If we now click on the products themselves in the list, we will see that each product still has the same static page, which is what we will deal with in this and subsequent lessons.

It is also necessary to note one more point. In general, the entire URL is divided intosegments- these are the parts of the URL between characters/. You probably remember that when creating this application we specified the path'products/:productId', so here it is:productIdcalled heredynamic segment. It is preceded by a symbol':'. The values ​​in this segment will change, they are the ones that go into the URL parameters (URL Params or params), which are passed to the loader under a certain key, in our case it will beparams.productId.

Take a look at the address bar in your browser when you click on products in the list. You will see that the last segment in the address bar changes, these are the values ​​​​that will end up in the bootloader. And our products in the warehouse have a unique generated by usid, this way we will be loaded with the product that we need.

And now, after a short digression, let's finally work with the product page, and more specifically, with loading data from the storage.

Let's repeat the familiar process. To start withforStorage.jslet's add a functiongetProductto retrieve specific product data from storage byid. Here we are already passing to the functionidand, accordingly, if the product is “cached”, it will be displayed without delay:

export async function getProduct(id) { await someNetwork(`product:${id}`); }

Next, we need to receive an array of products and among them find our product according to the passedid:

export async function getProduct(id) { await someNetwork(`product:${id}`); let products = await localforage.getItem('products'); let product = products.find((product) => product.id === id); return product ?? null; }
