Route without path in React Router

In this lesson, we'll look at another useful property of a route without a path.

You know that for every route object you can add a propertyerrorElementand set the value to the element that you want to display when an error occurs.

But what if we haveerrorElementwill it be the same for all routes? You can, of course, write the same thing for everyone, or you can use a route without a path and wrap the child routes that interest us in it. Then, if an error occurs on one of them, our route without a path will catch them and display itserrorElement.

Let's go tomain.jsxfile and wrap all our child routes in a route without a path, making them children of it (in turn, this route will become a child of the root one), don’t forget to add the propertyerrorElement:

const router = createBrowserRouter([ { path: '/', element: <Root />, errorElement: <ErrorPage404 />, loader: rootLoader, action: rootAction, children: [ { errorElement: <ErrorPage404 />, children: [ { index: true, element: <Index /> }, { path: 'products/:productId', element: <Product />, loader: productLoader, }, { path: 'products/:productId/edit', element: <EditProduct />, loader: editProductLoader, action: editAction, }, { path: 'products/:productId/delete', action: deleteAction, }, ], }, ], }, ]);

How can I test this code now? You can add a line throwing out the error (don’t forget to remove it later when you check it). Let's, for example, add it as the first line to the functionactionfor the product removal route, then when we try to remove the product we will see the error:

export async function action({ params }) { throw new Error('error'); await deleteProduct(params.productId); return redirect('/'); }

Now try to remove any product.

Take the application you created in the assignments for previous lessons. Using the tutorial, wrap your child routes in a route without a path with the propertyerrorElementand check that it works.
