Updating data from FormData in React Router

We have a functionupdateProductto update product data in the repository. Now let's move on to the data from our editing form.

When an HTML form is submitted, the browser creates an objectFormDatawith data and sends it in the body of the request to the server. Moreover, the value of each input is extracted into the object from the attributename(that's why we needed them in uniform, remember?). We now know that React Router sends requests not to the server, but to the methodactionour route, accordingly, will get thereFormData. Let's work with this.

First, let's open our fileedit.jsxand importupdateProduct:

import { updateProduct } from '../forStorage';

Then we will write a function foractionroute object, as we did before. Let's add it right after the functionloader. We will give ours to herrequestand URL parameters:

export async function action({ request, params }) {}

From the request we will getFormData, then extract data from it in the form of an object containing pairskey: valueand with the helpupdateProductLet's send this data to the storage:

export async function action({ request, params }) { const formData = await request.formData(); const updates = Object.fromEntries(formData); await updateProduct(params.productId, updates); return 1; }

All that remains is to go tomain.jsxand add a functionactionto the edit route object. We importaction:

import EditProduct, { loader as editProductLoader, action as editAction, } from './routes/edit';

And add it to the edit route object:

{ path: 'products/:productId/edit', element: <EditProduct />, loader: editProductLoader, action: editAction, },

Now launch the application, add a couple of products to the list, then click on them, fill out the form and click on the save button. Don't forget to go to the developer panel in the localforage section and update the repository. Now in the keyvaluespairs section we can see the objects in the arrayproductswith our entered data.
