Navigation state in React Router

If we click on products in the list, we see that the first time the page with the product loads with a delay, and the rest quickly, the same thing when adding and updating a product, since we have a cache on the client side.

In this lesson we will show the user what state our page is in. A hook can help us with thisuseNavigation. As a result of its work, this hook produces an object with a number of properties. We will be interested in one of them - the propertystate.

Propertystatemay have3values:'idle'(nothing happens)'submitting'(when submitting a form via POST, PUT, PATCH or DELETE is calledactionroute),'loading'(when the loader for the next route is called to render the next page). We will be interested in the last value.

Let's go to the fileroot.jsxand import this hook:

import { useNavigation } from 'react-router-dom';

Then, for the result returned by the hook, let's create a variablenavigationin functionRoot- beforereturn:

const { products } = useLoaderData(); const navigation = useNavigation();

Now let's use its propertystate, if this is the value'loading', then we will set the classloadingfor the div in which we draw product data:

<div id="product" className={navigation.state === 'loading' ? 'loading' : ''}> <Outlet /> </div>

All we have to do is add styles for the classloadingVindex.css. Let's just increase in case of loading stateopacity:

div.loading { opacity: 0.3; }

Now, if, after reloading the site, we click on products in the list or, for example, add a new product, we will see that the current page will fade for a moment before the next one appears.
