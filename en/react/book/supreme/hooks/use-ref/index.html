useRef hook for working with refs in React

In this lesson we will work with refs. For clarity, let's look at how the hook worksuseRefcompared to hookuseState.

Let's create a component with a button:

return ( <div> <button> state click </button> </div> );

Importing into a componentuseState:

import { useState } from 'react';

And let's start a statestate:

const [state, setState] = useState(0);

Now let’s make sure that by clicking on the button ourstatewould increase by1. We will display the state value directly in the button text:

<button onClick={handleStateClick}> state click: {state} </button>

Let's describe a function for processing a click on a buttonhandleStateClick:

function handleStateClick() { setState(state + 1); }

We press the button and see how the value of the state grows.

Now let's create the componentApp, but using not the state, but the ref.

First, let's import it into the componentuseRef:

import { useRef } from 'react';

And let's start a refref. The result is a hookuseRefreturns a ref object with a single propertycurrent, which will interest us in the future. Let's set its initial value to0:

const ref = useRef(0);

Let's attach a click handler to the button. Remember that we must read/modify not theref, and its propertycurrent:

<button onClick={handleRefClick}> ref click: {ref.current} </button>

Let's add a function to process a click on our button. Here we will increasecurrenton1, as in the previous example with the state. Unlike a state, where a function is requiredsetStateTo change its value, we work directly with the ref property:

function handleRefClick() { ref.current = ref.current + 1; }

Now let's press our button. So what do we see? When you click on it, as we had initially0, so it doesn’t change.

You may, of course, begin to doubt that the meaning changes at all. Let's check it out. To do this, add another line of code to the click handler function with the output of the valuecurrentto the console:

function handleRefClick() { ref.current = ref.current + 1; console.log('ref click: ' + ref.current); }

Now, having opened the console in the browser, click on the button again and make sure that the value really changes with each click and there is no deception.

Now we see that, unlike the state, changing the ref value does not cause the component to render. Therefore, every time we see the initial value in the button text0.

So, if your data is used for rendering, then store it in a state, and if you do not need rendering, then using refs can become more efficient.
