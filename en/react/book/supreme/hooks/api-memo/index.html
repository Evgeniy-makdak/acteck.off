API memo in React

Before we look at the next hook, there's a useful API to mention:memo, which helps us avoid re-rendering a component if its props remain unchanged.

Let's look at this with an example. Let's create a componentApp, in which there will be two inputs, into which, suppose, the first and last name are entered:

return ( <div> <label> name: <input value={name} onChange={(e) => setName(e.target.value)} /> </label> <br /> <label> surname: <input value={surn} onChange={(e) => setSurn(e.target.value)} /> </label> </div> );

Let's add states for them to the beginning of the component:

const [name, setName] = useState(''); const [surn, setSurn] = useState('');

Let's also create a child componentChild.js, have it print a greeting message that displays the name you entered. The console will also display the line'child render'every time this component is rendered:

function Child({ name }) { console.log('child render'); return <h3>Hello {name}!</h3>; }

Let's addChildinto the layout of the main component after the last input and we will pass its name as a prop:

<Child name={name} />

We import it into the main component:

import Child from './Child';

Now let’s open the console and enter the first and last name into the input fields. Here we will see that even when we enter characters in the last name field, our child component will be redrawn every time. No problem, because we have a small component. But imagine if this component displayed a large amount of data and at the same time was redrawn every time.

Now let's wrap the child component inmemo, and see how the rendering changes. First, let's import it intomemo:

import { memo } from 'react';

Then we will create a new variable and assign ourChild, wrapped inmemo. We will get the followingfunctional expression:

const Child = memo( function Child({ name }) { console.log('child render'); return <h3>Hello {name}!</h3>; });

Let's open the console and enter the first and last name into the fields again. Now we see that when we enter a last name, our child component is not redrawn.

It must be remembered that this will not work if, while the props are unchanged, the states involved by the component or the context that uses the changes change.

Take the component codeApp, which we did in this lesson, leave only the first input in it. Create a React ComponentTextcontaining a paragraph with text'long text', and place it inAppafter input.

In componentTextadd a lineconsole.log('text render');. Make sure that when entering characters into the input componentTextredraws every time.

Now wrap the componentTextVmemo. Make sure that when entering characters into the input componentTextis not redrawn again.
