UseMemo performance optimization hook in React

The first performance optimization hook we'll look at isuseMemo.

This hook helps cache the results of resource-intensive operations between screen redraws and, accordingly, can help avoid unnecessary large calculations. This caching is also calledmemoization.

Let's see how it works. Let's create a component with a button and a titleh3:

return ( <div> <h3>Text</h3> <button>click</button> </div> );

Now we’ll make sure that when you click on the title, its color changes from orange to green and back. First, let's create a stateisGreen:

const [isGreen, setIsGreen] = useState(false);

Add to attributestyleheader condition for changing the color of the header and attaching a click handler:

<h3 onClick={() => setIsGreen(!isGreen)} style={{ color: isGreen ? 'green' : 'orangered' }} >Text</h3>

Let us also have some value that will increase by one when we click on our button. Let's create a state for it:

const [num, setNum] = useState(0);

Let's add button click processing:

<button onClick={() => setNum(num + 1)}> clicks </button>

Let us also have a certain functionsquarewhich will return the square of the valuenum. We will write the result of the function call into a variableresult:

const result = square(num); function square(num) { return num * num; }

Let's deduceresultin the button text:

<button onClick={() => setNum(num + 1)}> clicks: {result} </button>

As a result, we got the following: when you click on the button, the value changesnum, which is then squared, and when you click on the title, the color of the title changes.

We have a very small component, everything works quickly, despite the fact that when you click on the title to change its color, the entire component is rendered anew, and accordingly, the calculations that are tied to the button occur again, even though we did not touch it. Now imagine if our calculations were voluminous and everything would be recalculated anew every time.

Let's make our function a little heavier, now it will think a little longer. This way we simulate long calculations:

function square(num) { let startTime = performance.now(); while (performance.now() - startTime < 500) { // Just do nothing ... } return num * num; }

Now click on the title. It turns out that now, due to the long operation of the functionsquare(and we don’t touch the button) we have to wait forever for the title to change color!

This is where the hook comes to our aiduseMemo. To do this, we need to pass as the first parameter a function that calculates the value that we want to cache; this function must be pure and not accept any parameters. And the second parameter is the dependencies in square brackets, in other words, all the reactive values ​​that participate in the function code. Thus, inresultWe will now write the following construction:

const result = useMemo(() => square(num), [num]);

Click on the title again. Now, if we do not touch the button with calculations and thus do not change the value of the statenum, then nothing is recalculated and React displays the cached value in the button, so our title quickly changes color.
