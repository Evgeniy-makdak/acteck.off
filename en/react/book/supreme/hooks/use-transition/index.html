UseTransition performance optimization hook in React

Performance optimization hookuseTransitionallows us to update states without blocking the GUI.

Using this hook, we can set the update of some state to low priority, which will allow the update of other states to have high priority and be completed without delay.

An example is filtering a drop-down list of products when the user enters characters in the search field. Of course, the list itself may be displayed with some delay, but we don't want characters to be entered into the search field with a delay.

Let's see how a hook can help us in such casesuseTransition. Let's say we have a piece of code for a main component. Let's analyze it:

import { useTransition } from 'react'; import { useState } from 'react'; function App() { const [isPending, startTransition] = useTransition(); const [filterTerm, setFilterTerm] = useState(''); const filteredProducts = filterProducts(filterTerm); function updateFilterHandler(event) { startTransition(() => { setFilterTerm(event.target.value); }); } return ( <div> <input type="text" onChange={updateFilterHandler} /> <ProductList products={filteredProducts} /> </div> ); }

Here we see two variables foruseTransition. VariableisPendingcontains a boolean value and indicates whether the update process is completed, andstartTransitiona function that allows you to reduce the update priority:

const [isPending, startTransition] = useTransition();

We also have statefilterTerm, which stores the expression and function entered in the input fieldsetFilterTermto install it:

const [filterTerm, setFilterTerm] = useState('');

When entering characters into the input, we call the function every timeupdateFilterHandler, in which the update takes placefilterTerm. This is where we apply our hookuseTransition, wrapping the install functionsetFilterTermVstartTransitionas follows:

function updateFilterHandler(event) { startTransition(() => { setFilterTerm(event.target.value); }); }

So we set to update the statefilterTermlow priority and our input field will remain responsive.

Using a variableisPendingwe can show the user that the list is still being updated. Let's display this information below the input field while the list is updated:

return ( <div> <input type="text" onChange={updateFilterHandler} /> {isPending && <p>updating ...</p>} <ProductList products={filteredProducts} /> </div> );
