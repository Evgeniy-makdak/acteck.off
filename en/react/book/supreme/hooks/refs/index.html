Refs in React

If we want to remember some information, but do not want its change to cause the component to render, then we will needrefsand accordingly the hookuseRefwho works with them.

A ref (or link) is a regular JavaScript object with a single propertycurrent, which you can read or change.

Like states, refs can store any type of data - it can be numbers, strings, objects, and even functions.

If your data is used for rendering, then store it in a state, and if you don't need rendering, then using refs can be more efficient. In addition, unlike the state, the value of the ref changes immediately.

In general, refs are used where you need to step away from React and interact with external APIs - often browser ones that do not affect the display of the component. This could be, for example, storageidtimer or storing an object for which there is no need to evaluate JSX, but most often it is interaction with DOM elements. They can be used to control focus, highlight text, or play media.
