Benefits of using createAsyncThunk in Redux

In the last lesson we used the functioncreateAsyncThunkfrom RTK to write our first thunk. In this tutorial I would like to dwell a little on the usefulness of this function when sending requests.

In general, the logic for obtaining data using the methodfetchlooks pretty formulaic in Redux. And, accordingly, to send the request status to thunk, we need to create at least three actions (withtypeAndpayload). The first should be sent when a request is sent, the second when the request is successful, and the third when it fails. They can then be sent usingdispatchin thunk code, for example in blockstry-catch. As a result, such code can look quite cumbersome and inconvenient (just look at the documentation). Each action can have its own action creator. In addition, you cannot confuse the sequence of their sending and the actions themselves in particular. Of course, this is not entirely mandatory, but it is a common practice when sending requests.

Fortunately, we no longer need to do all this manually - now the function will do itcreateAsyncThunkand will generate a thunk that will send all these actions automatically. We just need to transfer tocreateAsyncThunka callback that will make an asynchronous call and return a promise with the result. That's what we did in the last lesson.
