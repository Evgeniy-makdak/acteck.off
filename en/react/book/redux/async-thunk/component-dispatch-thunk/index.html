Sending a thunk from a component in Redux

In this tutorial we will send a thunk to get products from a React componentProductsList.

Let's now open our product application, and in it there is a fileProductsList.jsx. First, let's add thunk to the importfetchProducts, which we created:

import { selectAllProducts, fetchProducts } from './productsSlice'

We also need a hookuseDispatchto send our action creator and hookuseEffectbecause here we haveside effect, since we are dealing with a grid, and this is an external API:

import { useEffect } from 'react' import { useSelector, useDispatch } from 'react-redux'

Now the first line in the function codeProductsListbefore the announcementproductslet's add:

const dispatch = useDispatch()

Next, when receiving products from the state, we will also retrieve the request status:

const products = useSelector(selectAllProducts) const productStatus = useSelector((state) => state.products.status)

And immediately after that we will tell React using a hookuseEffect, that after rendering the component we need to get the products. This is where we need the request status, because it is important for us that such a request is sent only once:

useEffect(() => { if (productStatus === 'idle') { dispatch(fetchProducts()) } }, [productStatus, dispatch])

Now we can launch our application, click on'Products'in the menu on the left. We won’t see the list of products yet, but if we go to our Redux DevTools, then on the left side of the window we will have automatically generated actions (as I said in the last lesson)products/fetchProducts/pendingAndfulfilled. Now let's click on actionproducts/fetchProducts/fulfilledand across his fieldpayload- here is our list of products! Hurray, all our links in the chain - server, database, client and Redux - application work harmoniously and together. By the way, as you can see, there is also a propertymeta, in which we see the status of the request.

Let's pay attention to one more thing, it may happen to you thatpendingand thereforefulfilledwill be displayed twice. If so, don't worry, we'll talk about it later.
