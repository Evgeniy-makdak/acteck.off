Sending data using thunk in Redux

In the last lesson, we figured out how to process a POST request on our fake server. Let's now write a thunk function that will send a request to the server.

Let's open our application with products, and in it there is a fileproductsSlice.js. Now after thunkfetchProductswe are usingcreateAsyncThunklet's create a thunkaddProduct:

export const addProduct = createAsyncThunk()

The first parameter we will pass tocreateAsyncThunk 'products/addProduct', and the second is an asynchronous callback that will receive an object with new product data:

export const addProduct = createAsyncThunk( 'products/addProduct', async (newProduct) => {} )

In the code of this callback we will call the client, passing it the path and the object with the product as parameters, and then return the response data:

export const addProduct = createAsyncThunk( 'products/addProduct', async (newProduct) => { const response = await client.post('/fakeServer/products', newProduct) return response.data } )

Now let's look at the field below in the codereducersForproductsSlice. Previously, we added a new product using a reducerproductAdded, in which we had methodsreducerAndprepare. Now we generate the necessary data on the server and work with thunk, so we will completely remove the reducer hereproductAddedfrom code. And then we add in to the methodextraReducers(at the end of his code) another additional reducer, which, if the request is successful, will be added to the sliceproductsnew product straight frompayloadaction (remember that such code is only possible thanks to the uniquecreateSlice):

.addCase(addProduct.fulfilled, (state, action) => { state.products.push(action.payload) })

Oh, by the way, don’t forget to remove nanoid from the import lines, andproductAddedfrom the action export at the end of the file.
