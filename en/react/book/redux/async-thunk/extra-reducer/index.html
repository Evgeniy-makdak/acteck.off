Additional reducers in Redux

Let's open our application with products, and in it there is a fileproductsSlice.js. You probably noticed that we created thunkfetchProductsas a separate function? We did this becausecreateSlicedoes not support defining thunks. How can we force the slice reducer in this case?productsrespond to actions that are defined outsideproducts? After all, we just need to process the actions sent by thunkfetchProducts.

For such casescreateSlicethere is a propertyextraReducers, which allows you to addadditional reducers, which in turn will process actions that are not defined in this slice.

Let's now in the body of the functioncreateSliceafter propertyreducerswith all reducers we will add one more methodextraReducers:

extraReducers() {},

We must pass an object to this methodbuilder, which has methods with which you can add additional reducers:

extraReducers(builder) {},

We will use one of the methodsbuilder - addCase, which takes action creator as the first parameter, and reducer as the second. One of the actions that will be sent to usfetchProductswhen asked thisfetchProducts.pending, which tells us that the request has been sent. In this case, let's change the status to'in progress'(before that he was'idle'):

extraReducers(builder) { builder.addCase(fetchProducts.pending, (state) => { state.status = 'in progress' }) },

Now let's process the action that will be sent if the request is successful. Here we will change not only the status in the state to'success', but we will also take the products into a sliceproductsfrompayloadaction. To collect all the products we need a methodconcat:

extraReducers(builder) { builder .addCase(fetchProducts.pending, (state) => { state.status = 'in progress' }) .addCase(fetchProducts.fulfilled, (state, action) => { state.status = 'success' state.products = state.products.concat(action.payload) }) },

If the request is unsuccessful, we will change the status to'fail'and write an error message to the state:

.addCase(fetchProducts.rejected, (state, action) => { state.status = 'fail' state.error = action.error.message })

If we now launch our application and click on'Products', then in a couple of seconds (remember the delay we set on the server?) we will see a list of products.

We will also see changes in Redux DevTools. Now our products will appear in the state (look at the tab'State'), if you clicked on the actionproducts/fetchProducts/fulfilled. A new status will also appear there.'success'. Click on the action nowproducts/fetchProducts/pendingand see how the tab is now different'State'.
