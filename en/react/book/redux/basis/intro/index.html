Redux library

In this tutorial, we will be introduced to the very useful and valuable JavaScript library Redux. The name of the library comes from two words - the name of the functionreduceand architectureFluxapplications on React, which is based on the use of unidirectional data flow.

First and foremost, Redux is a predictable container or state store for application data. We have already encountered the concept of state orstate (state) while studying the React tutorial. With Redux, you can write client, server, or native applications that behave predictably and are easy to scale and test.

What else is attractive about Redux? It helps us manage the global state of the application. The tools and techniques used in Redux make it easier to understand where, why and how the state has changed, as well as how the application logic will behave in the event of such changes.

Redux is a standalone library, but it can be used in conjunction with such popular frameworks asReact, Angular, Vue, and alsopure JavaScript. Looking ahead a little, I will say that in our tutorial we will use a combination with React.

Despite the importance of this tool, when creating an application, you should wisely evaluate the need for its use. You need Redux if you have a large number of states needed in different parts of the application, or states change very often, or maybe your logic for updating states is too complex.
