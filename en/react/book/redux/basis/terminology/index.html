Basic terms in Redux

In this lesson, we will briefly go over the basic concepts and terms that are essential for further learning of Redux.

We'll start withactions (action). This is some event that describes what happened in our application. Technically this is just regular JavaScriptobjectcontaining the fieldtype, in which we enter the name of the action.

The second important field of an object with an action is the fieldpayload. It will containpayloadactions. Payload refers to some data that is passed by parameters to the action.

This object may also contain other fields with additional information.

Let's create an action as an example and call itaddOrderAction. Let it be responsible for adding an order. So in the propertytypewe will indicate that the order has been added (orderAdded) to orders (orders). And inpayloadLet us indicate in more detail what kind of order it is, for example, to paint a wall:

const addOrderAction = { type: 'orders/orderAdded', payload: 'Paint a wall' }

In order not to write an object with an action manually every time, we can use the so-calledaction creator- a function that will create and return an object with an action. Let's call heraddOrder. As a parameter we will pass it the text we need for the propertypayload:

const addOrder = text => { return { type: 'orders/orderAdded', payload: text } }

The next important concept in the chain isreducer(reducer) - a function that takes the current state and an object with an action. This function decides how to update the state and returns the already updated state if necessary.

The following concepts arestorage (store) is an object that stores the current global state of the Redux application. This object has a methodgetState, with which you can get the current state value.

The repository also has a methoddispatch. Calling it and passing an object with an action is the only way to change the state. As a result, the store will launch the reducer function and store the new state value.

And the last concept that we will look at in this lesson isselectors. Selectors are special functions that know how to extract information from a state located in the store. These features are especially useful as your application grows and help you avoid code repetition.

We will learn more about all these concepts and how to apply them in practice in the following sections of this textbook.
