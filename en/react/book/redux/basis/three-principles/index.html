Three Core Concepts of Redux

To wrap up this introductory theoretical part, we'll walk through three core Redux concepts that you need to follow when building applications.

The first of them states that there is a single source of truth. The global state of your application is stored in a tree of objects within a single store.

The second concept is that the state can only be read. The only way to change it should be to create an action - an object that describes what happened.

The third concept tells us that onlypure functions. That is, in order to specify how your state tree should change, you must write a pure reducer.
