Reducers in Redux

In the last lesson, using the functioncreateSlicecreated a slice for productsproductsand a reducer that will update this slice stored in the store. Let me remind you once again that in Redux, state data is changed using reducers.

We must remember that reducers are subject to a number of restrictions. The reducer updates the state only based on the received values ​​of the current state and objectaction, and then decides how to change the state and returns its new value.

We know that by default, objects and arrays in JavaScript are mutable (changeable). We also know that you cannot change state directly; in this case, you only need to work with copies. The reducer must also obey this rule. But writing such logic manually can be quite tedious and complex, and this is where the function comes to the rescuecreateSlicewhich uses the libraryImmer, which converts your 'mutable' code to 'non-mutable' code. Thus, you can sin and simplify life only by using the functionscreateSliceorcreateReducer.

The reducer's limitations also include the use of asynchronous logic, the calculation of random values, and other"side effects". Now, having gotten to know the reducer better, let’s return to our application with products.

Our application has a store that we created in previous lessons, but there is nothing in it yet. Let's open our filestore.jsand add some code to it. First, import the reducer function for products into it:

import productsReducer from '../parts/products/productsSlice'

Now we can specify the imported function as a reducerproductsReducer:

export default configureStore({ reducer: { products: productsReducer } })

With the above line we tell our store that now when an action occurs, all the data for the slicestate.productswill be updated by the reducerproductsReducer.

Now you can run the application (note that the warning about an invalid reducer has disappeared in the console!), open Redux DevTools in the browser and browse through its tabs. For example, by clicking on the State tab in the Inspector tab, we can see our two objects with products that were displayed as the initial state value when the application was first launched. On the Log monitor tab we see what slices the global state contains - this is a sliceproducts, including two objects. Also take a look at the Chart tab.
