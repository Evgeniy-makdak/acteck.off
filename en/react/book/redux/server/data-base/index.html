Database for server in Redux

In the last lesson, we installed msw, which will help us simulate the operation of our application with the server. And to begin with, we would like to load the data already available on the server when the application starts.

Let's open our application with products, and in it there is a fileserver.js. Here we already have a couple of lines of code from the last lesson.

And the first thing we have to ask ourselves is: where is the data usually stored? Yes, of course in the database, we will answer without hesitation. So let's use another cool tool that will allow us to create a model of our data and also make it look like we are working with a real SQL database.

The tool that will help us with this is the library.@mswjs/data. To install it, enter the following command in the terminal:

npm install @mswjs/data --save-dev

Now we import in the fileserver.jsthings we need:

import { factory, oneOf, manyOf, primaryKey } from '@mswjs/data'

And we will create our improvised database. For this we use the functionfactory. We will do this below after the lines with import and before exporting the worker:

export const db = factory({})

What data will we store in it? From the server we must receive products, sellers and user reactions. This means that in our database we must create three models that acceptfactoryin the form of objects:

export const db = factory({ product: {}, seller: {}, reaction: {}, })

As in a real SQL database, each of our models must haveprimary key. Traditionally, we will assign such a fieldid, which we will generate using the same nanoid library:

export const db = factory({ product: { id: primaryKey(nanoid), }, seller: { id: primaryKey(nanoid), }, reaction: { id: primaryKey(nanoid), }, })

Don't forget to add nanoid in the import lines at the beginning of the file:

import { nanoid } from '@reduxjs/toolkit'

In the next lesson we will add the necessary fields to our models.

Open your application with students. Once you've completed this tutorial, set @mswjs/data for your application.

Next create a database modeldbusing the functionfactory. Pass her three objects (student, teacher, vote) for your data, just as shown in the lesson.
