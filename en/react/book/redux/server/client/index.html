Client for an application in Redux

In the last lessons we almost completed the work with the server part for our application, we will add a little to it as needed in the following lessons. Now we need to create a client for the application that will send HTTP requests to the server.

Let's open our application with products and in the folderapilet's create a fileclient.js. First, we'll write the function itselfclient, which will work for both GET and POST requests. As parameters it will receive the path, type and body of the request (in the case of POST), if any:

export async function client(url, method, body = {}) {}

Next we need to create an objectoptions, in which we will pass various information for the request. This will be the request type and headers:

export async function client(url, method, body = {}) { const options = { method: method, headers: { 'Content-Type': 'application/json', }, } }

PropertybodyWe have it optional, we must pass it if we have a POST request. For this insideclientafter the code for the objectoptionslet's add it tooptionswith the request body in JSON format if the POST method is selected:

if (method === 'POST') { options.body = JSON.stringify(body) }

Then below in codeclientlet's organize a blocktry, in the first line of which, usingfetchWe will receive a response from the server. In case of failure, we will traditionally return the promise with an error:

let data try { const response = await window.fetch(url, options) } catch (err) { return Promise.reject(err.message ? err.message : data) }

Now let's add the code in the blocktry. After receivingresponse, we need to process it. Let's extract data from it in JSON and if the response status signals to us that everything is in order, then we will return an object with data and information: status, headers and address. If the status is not'ok', then we’ll throw an exception with the status text:

try { const response = await window.fetch(url, options) data = await response.json() if (response.ok) { return { status: response.status, headers: response.headers, url: response.url, data, } } throw new Error(response.statusText) }

Code for the functionclientWe have it ready, all that remains is to write how our GET and POST methods will work when called. Let's write the code for them after the functionclient. Methodclient.getwill take a path as a parameter and call the functionclient, passing it this path and GET request type:

client.get = (url) => client(url, 'GET')

Aclient.postmust transmit forclientpath, POST request type and request body:

client.post = (url, body) => client(url, 'POST', body)

That's all, our little client is ready to work with the server.
