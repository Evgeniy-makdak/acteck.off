Configuring Nginx for test sites

Let's use Nginx to bind domains to our test sites. To do this, open the configuration file, delete the current content from there and place the settings of our two sites there according to the following scheme:

server { настройки первого сайта } server { настройки второго сайта }

Let's make settings for the first site, indicating your domain name and your port on which the script of the first site is running:

server { listen 80; server_name test1.com; location / { proxy_pass http://localhost:3001; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } }

Below in the same file we will make the settings for the second site, also specifying the correct domain and port:

server { listen 80; server_name test2.com; location / { proxy_pass http://localhost:3002; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } }

After this, let’s run the command that checks the configuration for errors:

nginx -t

Let's restart Nginx:

service nginx reload

Restart Nginx:

service nginx restart

Let's check the status of Nginx (should be active, green text):

service nginx status

That's it, now you can access domains through your browser and they will go to your test sites. If this does not happen, make sure that the A-records of the domains point to the IP of your server, and also that you have not confused the ports of sites deployed on NodeJS.
