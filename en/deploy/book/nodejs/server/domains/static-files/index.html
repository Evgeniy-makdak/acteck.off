Static domain files via Nginx on a VPS server

To serve static files on NodeJS or Express, a static server is created. In fact, it is not recommended to do this because it is slow.

It is more optimal to serve static files via Nginx. To do this, a special setting is made for the domain, containing the regular sequence and the path to the folder with statics. Let, for example, in our domain, the folder with static content be the folderpublic. In this case, we will specify the following settings:

server_name test1.com; location ~* ^.+\.(jpg|jpeg|gif|png|ico|css|js|swf|txt|ico)$ { root /var/www/test1.com/public; }

At the same time, it is not very convenient that we have to write the domain name twice if this name coincides with the site folder (and for our convenience it does). The problem can be solved as follows:

server_name test1.com; location ~* ^.+\.(jpg|jpeg|gif|png|ico|css|js|swf|txt|ico)$ { root /var/www/$server_name/public; }

Let's put all our settings together and get the following:

server { listen 80; server_name test1.com; location ~* ^.+\.(jpg|jpeg|gif|png|ico|css|js|swf|txt|ico)$ { root /var/www/$server_name/public; } location / { proxy_pass http://localhost:3001; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } }

Configure static output for both test sites. Test it through your browser.
