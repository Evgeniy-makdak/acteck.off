Project logs via PM2 on VPS server

Scripts launched through PM2 do not outputconsole.logto the server console. Also, such scripts do not display errors that have occurred. However, PM2 allows you to configure the output of all this to files.

This is how we will force the execution results to be output to a fileconsole.log:

pm2 start app.js -o output.txt --time

And this is how we will force errors that have occurred to be output to a file:

pm2 start app.js -e error.txt --time

And this is how we will output both:

pm2 start app.js -o output.txt -e error.txt --time

Run your test script, outputting messages and errors to files.
