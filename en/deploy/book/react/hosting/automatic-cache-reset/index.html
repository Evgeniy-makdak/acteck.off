Automatic cache reset in React build

You should already know that there is a static cache problem in browsers. The essence of the problem is that to improve performance, the browser caches style files, scripts and images.

This means that if you change something in your scripts or styles on your hosting, the changes will only appear for new site visitors. Old visitors who have already been to the site before will see a cached version of the code. This is, of course, unacceptable.

Luckily, React solves this problem automatically. Her solution is that the build files, in addition to the name and extension, have a random string calledhash. This hash matches the contents of the file. This means that when the code in the file changes, the hash in its name will also change. Thus, the browser will think that this is a new file and download it for itself.

We can observe the hashes of files when they are connected toindex.html:

<script defer="defer" src="/static/js/main.3dd63bcb.js"></script> <link href="/static/css/main.f855e6bc.css" rel="stylesheet">

Build the project. Study what hashes the files in the assembly have.

Build without changing your project code. Make sure that the file hashes will not change.

Change your project code. Complete the build. Make sure the file hashes change.
