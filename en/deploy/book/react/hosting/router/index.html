React Router on PHP hosting

As you already know, after assembly, the React project is a regular static project that can be uploaded to any hosting, and it will immediately work there.

There is, however, a problem with React Router. The fact is that Router changes the URL of pages in the browser when running. In this case, the page does not actually reload, but only changes the URL using JavaScript.

Router will work on a hosted site. But, if you reload it while on any page of the site other than the main one, it will crash404error.

And this is logical, because the browser will search for the file at the given URL, as static sites usually work. But in our application there will be no such page, since we are only simulating them, but in fact our entire application runs onindex.html.

To solve the problem, you need to make sure that all URLs that do not lead to actual files are redirected toindex.html. This is done using a web server running on hosting.

Shared hosting runs on PHP. Typically, Apache is used as the server. There is a special file available in it.htaccess, with which you can set a redirection.

To do this, you need to place a file in the root of your site.htaccesswith the following content:

RewriteEngine On RewriteCond %{REQUEST_FILENAME} !-f RewriteRule ^ index.html [QSA,L]

Make a project with React Router.

Upload it to your hosting. Make sure that routing is disabled by default.

Fix the problem using the file.htaccess.
