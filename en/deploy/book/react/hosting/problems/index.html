Problems with deploying React to hosting

The React project code is regular JavaScript files. We, however, cannot simply upload a React project to hosting, like regular static stuff.

The fact is that in the process of developing a website in React, we need NodeJS. But NodeJS is only required to develop the site, and once the site is created and running, NodeJS is no longer required.

Therefore, we cannot simply upload our website files to hosting - they simply will not work without NodeJS.

The correct approach here is the following: when the project is ready for deployment, we will need to build it for production.

As a result of the assembly we will getbuild, which is bare static. This build needs to be uploaded to the hosting. In the following lessons we will discuss all the nuances of this process.
