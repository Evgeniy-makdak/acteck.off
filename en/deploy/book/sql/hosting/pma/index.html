Working with PMA on hosting

Typically available on virtual hostingPMA, in which all your databases will be visible.

For PMA, hosting usually initially creates a specific user with the appropriate rights. With these rights, the user will not be able to create new databases, but will be able to perform operations with tables: create new tables, new fields, and so on.

To log in to PMA, you need to go to your hosting control panel and find the PMA login link there. By clicking on a link or button, you will be transferred to PMA and you will already be authorized there under the default user.

This scheme is common, but not mandatory. It happens that you are redirected to PMA to the authorization form. And there you should already log in under your user. If the user is connected to only one database, then in the PMA you will see only this one database.
