Two types of servers for PHP hosting

There are two popular types of website server programs: Apache and Nginx. As a rule, on hosting PHP runs either in conjunction with Apache or in conjunction with Nginx.

For us, as PHP programmers, it makes no difference which server is used. With one important exception. The fact is that the htaccess file is a settings file for the Apache server. This means that if your server is running Nginx, then your htaccess will not work.

Nginx has a mechanism similar to how the htaccess file works. This mechanism consists of configuration files created for each site. Unfortunately, if you have shared hosting, you will not be able to access these files.

Considering that all fairly complex sites use htaccess, we need the Apache server. Fortunately, it is installed on almost all virtual hosting services.

It often happens that Apache is used in conjunction with Nginx. In this case, Nginx accepts requests to the site from the outside world and passes them to the Apache server. And PHP already works in conjunction with Apache and, accordingly, htaccess also works.

We will go deeper into these servers in other, more advanced tutorials. And in the next lesson we will simply check whether the htaccess file works on your hosting.
