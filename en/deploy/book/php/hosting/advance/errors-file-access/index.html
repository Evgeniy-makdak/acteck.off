Closing access to the log file on the hosting

Our log file has some problem. The fact is that it is accessible directly through the browser if you access it in the right way. In our case this:/logs/errors.log.

Thus, by guessing the path to the file, the hacker will be able to read the log with our errors. This should not be allowed.

To solve the problem in the folderlogsshould be put.htaccesswith the following content:

Deny from all

Check that the log file is accessible through the browser.

Close access to the log file. Make sure it is closed.
