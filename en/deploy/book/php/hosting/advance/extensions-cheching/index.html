Checking PHP modules on hosting

The PHP engine, in addition to the basic functions of the language, expands its functionality with the help of additional modules. These modules may or may not be installed. In addition, installed modules may be disabled and not work.

Quite a lot of familiar functionality is implemented through modules. For example, working with JSON, working with XML, the CURL library. Even working withmb_functions and it works through modules.

On LAN, as a rule, most of the modules we need will be installed out of the box. But some modules may not be available on the hosting. Or they can be disabled. This is done to save hosting resources.

To check if the module is enabled, you need to run the following command:

<?php phpinfo(); ?>

As a result, a page with the PHP configuration will be displayed in the browser. We can use a page search to find the module we need and see whether it is enabled or not.
