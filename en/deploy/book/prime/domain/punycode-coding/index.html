Encoding via punycode

Initially, only Latin letters were allowed in domain names. Then characters from national alphabets were allowed.

But everything is not so simple here! In fact, non-Latin names are built according to a cunning system. The user sees a beautiful non-Latin name in the browser, but inside various programs that use domain names for their work, such names are converted to Latin characters.

That is, each non-Latin domain corresponds to a special code consisting of Latin letters. This code is called Punycode.

For exampleмолоко.ruwill be converted to Punycodexn--j1abcibb.ru, Aмолоко.рфin Punycodexn--j1abcibb.xn--p1ai

Let's play around with Cyrillic names. Open the website in your browsermilk.rf. Then copy the address from your browser's address bar and paste it into the editor. And in return for the nameмолоко.рфthe editor will have his Punycode! A similar situation will happen if you copy the domain name into the messenger.

That is! It was stated above that Punycode is seen only by programs, and a person sees a beautiful domain name. In fact, as we see, this is not entirely true. This somewhat reduces the value of such domains. You should weigh the pros and cons when purchasing such a domain.
