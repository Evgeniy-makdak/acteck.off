Introduction to MongoDB Database

Now we will start studyingdatabases. They are special programs for storing website information. We'll start with the MongoDB database, which is often used in conjunction with NodeJS.

MongoDB is a so-called NoSQL database in which all data is stored in documents in JSON format. In the case of MongoDB, this format has some special features and is therefore called BSON.

Data storage structure

All data in MongoDB is stored indocuments. Documents are JSON containing data of some entity. For example, in a document we can store data of a certain user, like this:

{ "user": "john", "age": 25 }

As you can see, our user hasfields userAndage, with the corresponding data.

We can store the second user in a second document:

{ "user": "eric", "age": 26 }

And so on - for each user we will have our own document with his data.

Each document will be included incollectionwith users. Your database will have many collections - each entity has its own, for example, a collection with users or a collection with messages.

Document ID

To distinguish documents from each other, MongoDB adds a special field to each document_idcontaining a unique random string:

{ "_id": ObjectId("61057f32a6949aeafdcb8967"), "user": "john", "age": 25 }

With this field we will be able to retrieve and modify specific documents from our database.

Installing MongoDB

First, you need to install MongoDB. It will be a separate program on your computer. For installation information, seevia this link.

After installing MongoDB, you will also need a special NodeJS driver that will allow you to work with the database program. Install it with the following command:

npm install mongodb

Follow the instructions to install MongoDB and its driver.
