Introduction to Express

Express is a framework that allows you to significantly simplify the deployment of a NodeJS server, breaking the site layout into parts, templating, and routing. As a rule, currently working with NodeJS occurs through Express. Let's deal with him.

Installation and launch

To install the library, run the command:

npm install express

After this, you should import the library in your file:

import express from 'express';

Then you need to create an object with an Express application:

let app = express();

After this, using a special commandapp.getwe can process the requested URLs. For example, let’s give some text for the main page:

app.get('/', function(req, res) { res.send('hello world'); });

Let's deploy our application on some port:

app.listen(3000, function() { console.log('running'); });

The final code for our server will look like this:

import express from 'express'; let app = express(); app.get('/', function(req, res) { res.send('hello world'); }); app.listen(3000, function() { console.log('running'); });

Install Express and test it using the code above.

Processing many URLs

By usingapp.getYou can process any number of URLs. See example:

app.get('/1/', function(req, res) { res.send('page1'); }); app.get('/2/', function(req, res) { res.send('page2'); }); app.get('/3/', function(req, res) { res.send('page3'); });

Make five different URLs that will display some text.

Request and Response Objects

Look closely at the callback we pass toapp.get:

app.get('/', function(req, res) { });

As you can see, we have two variables. Express will put a special object containing the browser request data into the first variable, and a special object to control the server's response to this request into the second variable.

Of course, the names of these variables can be anything, but it is customary to call themreq(short for "request") andres(short for "response").

In the following lessons we will take a closer look at the capabilities of these special objects.
