Retrieving data from related tables in NodeJS

Let's make a query that will get all users along with their cities. To do this you will need a commandLEFT JOIN:

Its syntax is as follows:

SELECT поля FROM имя_таблицы LEFT JOIN имя_связанной_таблицы ON условие_связи WHERE условие_выборки

Let's look at the individual parts of the syntax for this command.

Fields

Since the selection comes from several tables, the selection of all fields via*won't work. The following query will select fields only from the main table, but not from the related one:

SELECT *

In order for data to be selected from all tables, you need to*specify the table name for the selection:

SELECT users.*, cities.*

Or we can list the fields we need, indicating the table name in front of them:

SELECT users.name, cities.name

These two methods have a problem. The fact is that if the fields in the tables have the same names, then a name conflict will occur in the array and only one field will win, and the second will not.

To solve the problem, you need to rename conflicting names using the commandas:

SELECT users.name, cities.name as city_name

Connection

After the commandONwe must indicate the fields from the two tables through which the connection is carried out. In our case this will be the fieldidfrom a table with cities and a fieldcity_idfrom the table with users:

ON cities.id=users.city_id

Request

As a result, the request that will retrieve users along with their cities will look like this:

SELECT users.name, cities.name as city_name FROM users LEFT JOIN cities ON cities.id=users.city_id
