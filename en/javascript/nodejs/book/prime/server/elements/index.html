Elements in a NodeJS website template

In the previous lesson, you learned how to store a website layout in a single template file. As you already know, only the common parts remain in this file: header, sidebars, footer. As a rule, this is convenient, however, the HTML code of these parts can be quite large, which interferes with the perception of the overall structure of the site.

It would be convenient to put these parts in separate files, so to speak, “out of sight”, so that you can access them as needed, and the rest of the time so that they do not interfere with reading other code.

Let's come up with a special command for this{% get element '' %}, where the name of the element will be written in quotes. In this case, our template file will turn into something like this:

<!DOCTYPE html> <html> <head> <title>{% get title %}</title> </head> <body> <div id="wrapper"> <header> {% get element 'header' %} </header> <main> {% get content %} </main> <aside> {% get element 'aside' %} </aside> <footer> {% get element 'footer' %} </footer> </div> </body> </html>

Let's store the layout elements in a folderelems. At the user's request, we will need to read the file with the site template, then find all the commands for inserting elements and replace them with the texts of the corresponding files.

Let's write code that implements what has been described, omitting for brevity the work with the title and content, as well as site resources:

http.createServer(async (request, response) => { let lpath = 'layout.html'; let layout = await fs.promises.readFile(lpath, 'utf8'); let reg = /\{% get element '(.+?)' %\}/g; layout = layout.replace(reg, async (match0, match1) => { return await fs.promises.readFile('elems/' + match1 + '.html', 'utf8'); }); response.writeHead(200, {'Content-Type': 'text/html'}); response.write(layout); response.end(); }).listen(3000);
