Implementing a static server on NodeJS

Let's now make sure that the URL on our site is searched for the corresponding HTML file. For example, if you are asked/page.html, then we must give the same file, and if requested/dir/test.html, then we have to give the filetest.htmlfrom folderdir.

Let's place all our HTML files in a folderroot. This will be the root folder of our site and we will start searching for HTML files relative to this folder.

Let's implement the described:

http.createServer(async (request, response) => { if (request.url != '/favicon.ico') { let path = 'root' + request.url'; // преобразуем URL в путь к файлу let text = await fs.promises.readFile(path, 'utf8'); response.writeHead(200, {'Content-Type': 'text/html'}); response.write(text); response.end(); } }).listen(3000);

Let's now process a URL like/dir/sub/. As you can see, this address does not contain the file name and its extension. On the Internet it is generally accepted that such an address refers to a fileindex.htmllocated in this folder. That is, our URL should be interpreted as/dir/sub/index.html.

Let's modify our code:

http.createServer(async (request, response) => { if (request.url != '/favicon.ico') { let path = 'root' + request.url'; if ((await fs.promises.stat(path)).isDirectory()) { path += 'index.html'; } let text = await fs.promises.readFile(path, 'utf8'); response.writeHead(200, {'Content-Type': 'text/html'}); response.write(text); response.end(); } }).listen(3000);

If the requested URL does not match a file on our site, it is obvious that we must serve404error. Let's do this:

http.createServer(async (request, response) => { if (request.url != '/favicon.ico') { let text; let status; let path = 'root' + request.url; try { if ((await fs.promises.stat(path)).isDirectory()) { path += '/index.html'; } status = 200; text = await fs.promises.readFile(path, 'utf8'); } catch (err) { status = 404; text = 'page not found'; } response.writeHead(status, {'Content-Type': 'text/html'}); response.write(text); response.end(); } }).listen(3000);

Issuing resources

Let's now modify our server so that in addition to HTML files, the requested resources are also automatically provided.

First, let's make a function that will take a path to a file and, based on the extension of this file, return its mime type:

function getMimeType(path) { let mimes = { html: 'text/html', jpeg: 'image/jpeg', jpg: 'image/jpeg', png: 'image/png', svg: 'image/svg+xml', json: 'application/json', js: 'text/javascript', css: 'text/css', ico: 'image/x-icon', }; let exts = Object.keys(mimes); let extReg = new RegExp('\\.(' + exts.join('|') + ')$'); let ext = path.match(extReg)[1]; if (ext) { return mimes[ext]; } else { return 'text/plain'; } }

Having this function, it is easy to adapt our server to serve files of any type:

http.createServer(async (request, response) => { let text; let status; let path = 'root' + request.url; try { if ((await fs.promises.stat(path)).isDirectory()) { path += '/index.html'; } status = 200; text = await fs.promises.readFile(path, 'utf8'); } catch (err) { status = 404; text = 'page not found'; } response.writeHead(status, {'Content-Type': getMimeType(path)}); // изменение response.write(text); response.end(); }).listen(3000);

Removing the extension from the URL

We implemented our server so that all URLs (except folder addresses) end with the extension.html. However, in the modern world, the presence of a file extension in an address is considered a sign of bad taste and old-fashionedness.

For beauty, it is necessary to make sure that the address looks like/page/matched the fileroot/page.html, address like/dir/page/- fileroot/dir/page.html, and the home page address/- fileroot/index.html.

In this case, requests to resources should work as they did.
