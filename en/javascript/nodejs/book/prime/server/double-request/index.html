Removing double HTTP server request on NodeJS

When displaying the site URL, you may have noticed that in addition to the requested address, the console also displays'/favicon.ico'.

The fact is that when you access the site, the browser automatically requests a favicon. This results in two requests being sent to the server.

The most correct action here would be to give a file with it when you request a favicon. However, you don’t know how to do this yet, so we’ll just make a condition that cuts off double output to the console:

http.createServer((request, response) => { if (request.url != '/favicon.ico'){ console.log(request.url); // теперь выполнится один раз response.writeHead(200, {'Content-Type': 'text/html'}); response.write('text'); response.end(); } }).listen(3000);

This step does not eliminate the double request (it will always be double), but the annoying double output to the console will disappear.

In the future, we will correct our code so that the favicon is actually sent to the browser.

Make sure you don't have double output to the console.
