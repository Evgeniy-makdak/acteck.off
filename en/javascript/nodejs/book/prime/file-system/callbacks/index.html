Asynchronous work with fs through callbacks in NodeJS

Let's now look at asynchronous work with the file system. Typically, in NodeJS, all module methodsfsexist in two versions: synchronous and asynchronous.

For example, to read a file synchronously, use the methodreadFileSync, and for asynchronous -readFile. Similarly, for writing a file there is a pairwriteFileSyncAndwriteFile.

Asynchronous file reading

MethodreadFilethe first parameter takes the name or path to the file, the second parameter is the encoding, and the third is a callback that will be executed after reading the file.

Two parameters should be passed to the callback. The first parameter will contain the object with the error if it occurs, and the second will contain the text of the read file.

Let's read the text of a file as an example:

fs.readFile('readme.txt', 'utf8', function(err, data) { console.log(data); });

Asynchrony check

You can make sure that the file is read asynchronously. To do this, let's output something to the console after working with the methodreadFile:

fs.readFile('readme.txt', 'utf8', function(err, data) { console.log(data); }); console.log('!!!');

As you already know, the callback will be executed when the file is read. While the file is being read, the script code will continue to be executed. This means that the result of the second will appear in the console first.console.log, and then the first.

Check the code after the methodreadFilewill be executed before the file is read.

Exception Handling

Since our code is asynchronous, exceptions cannot be caught throughtry-catch. The first parameter is intended for handling exceptions in the callback. This parameter will containnull, if no exception occurred, or an error object if an exception occurred.

Let's add the callback code so that it handles exception situations:

fs.readFile('readme.txt', 'utf8', function(err, data) { if (!err) { console.log(data); } else { console.log('ошибка', err); } });

Asynchronous file writing

Asynchronously writing text to a file is done in the same way:

fs.writeFile('readme.txt', 'text', function(err) { if (err) { console.log('ошибка'); } });

Asynchronous reading of multiple files

Let's say we have two files with numbers. Let's find the product of these numbers. Obviously, to do this we need to read both of these files.

But, since the code is asynchronous, we need to read the second file in the callback of the first:

fs.readFile('readme1.txt', 'utf8', function(err, data1) { if (!err) { fs.readFile('readme2.txt', 'utf8', function(err, data2) { if (!err) { console.log(data1 * data2); } else { console.log('ошибка чтения файла readme2'); } }); } else { console.log('ошибка чтения файла readme1'); } });

Asynchronous file reading and writing

Suppose we need to read a file, perform a text operation on it and write it back to this or another file. In this case, writing to the file will need to be done in the read callback:

fs.readFile('readme.txt', 'utf8', function(err, data) { if (!err) { fs.writeFile('readme.txt', data + '!', function(err) { if (err) { console.log('ошибка записи файла'); } }); } else { console.log('ошибка чтения файла'); } });

Arrow functions

As a rule, callbacks in NodeJS are done using arrow functions. This shortens the code, but makes it somewhat difficult to understand if you're not used to it.

Let's rework the previous code into arrow functions:

fs.readFile('readme.txt', 'utf8', (err, data) => { if (!err) { fs.writeFile('readme.txt', data + '!', err => { if (err) { console.log('ошибка записи файла'); } }); } else { console.log('ошибка чтения файла'); } });

Code given:

fs.readFile('readme1.txt', 'utf8', function(err, data1) { if (!err) { fs.readFile('readme2.txt', 'utf8', function(err, data2) { if (!err) { fs.writeFile('readme.txt', data1 + data2, function(err) { if (err) { console.log('ошибка записи файла'); } }); } else { console.log('ошибка чтения файла readme2'); } }); } else { console.log('ошибка чтения файла readme1'); } });
