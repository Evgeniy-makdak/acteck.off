Connecting the fs module in NodeJS

Now we will learn how to work with the file system. A built-in module is responsible for this in NodeJSfswhich needs to be imported:

import fs from 'fs';

Each module methodfsexists in two versions: synchronous and asynchronous.

Synchronous option

The synchronous version is simpler, but it blocks the entire script for the duration of its execution. For example, if you are reading a file, then all other code will wait until the file is read.

In this case, when reading a file, the hard drive is mainly loaded, not the processor. That is, the processor could continue to execute our code, but is forced to wait for the file to load.

This matters when you have a website deployed on the Internet using NodeJS that visitors visit.

Let's assume that every time any user visits our site, we must open a certain file. If we do this synchronously, then when opening one fileall other usersour site will wait until this file opens and only then will the script continue for them.

In this regard, the synchronous version of work is used very rarely, mainly in cases where you are creating some kind of local script that is deployed on the user’s computer.

Asynchronous option

Working with the asynchronous option is more difficult, but its advantage is that it does not block the script.

This means that while a file is being downloaded from the hard drive, our script will be able to perform other operations, which significantly increases the performance of the script.

The asynchronous style is characteristic of NodeJS in general, not only when working with files. This approach can significantly improve the performance of scripts.
