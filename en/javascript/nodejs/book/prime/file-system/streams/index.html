Reading and writing threads in NodeJS

Let's imagine that you have a fairly large file, say the size of100megabyte. Let's say we want to do something with the data in this file. Obviously, to do this you need to read the contents of this file into a variable:

let data = await fs.promises.readFile('readme.txt', 'utf8'); console.log(data);

The problem is that in this case everything100megabytes of our file will end up in the server’s RAM. Therefore, an attempt to simultaneously read a dozen or two such files will easily exhaust the server memory available to us, which will lead to its crash.

To solve this problem, they inventedstreams. Streams allow you to read a file piece by piece. Let's see how it's done. First, let's create a reading stream using the methodcreateReadStream:

let readableStream = fs.createReadStream('readme.txt', 'utf8');

Now let's read our file piece by piece:

readableStream.on('data', function(chunk) { console.log(chunk); });

Make the file large enough. Read it piece by piece and print each piece to the console.

Recording streams

You can also create recording streams. This is done using the methodcreateWriteStream:

let writeableStream = fs.createWriteStream('writeme.txt');

Let's write something to our stream using the methodwrite:

writeableStream.write('text1\n'); writeableStream.write('text2\n'); writeableStream.write('text3\n');

Let's complete the recording procedure using the methodend:

writeableStream.end();

Write a column of numbers from one to a million to a file.

Read and write

You can read one file while writing its data piece by piece to another file:

let readableStream = fs.createReadStream('readme.txt', 'utf8'); let writeableStream = fs.createWriteStream('writeme.txt'); readableStream.on('data', function(chunk) { writeableStream.write(chunk); });

The file is large in size. Write code that will make three copies of this file.
