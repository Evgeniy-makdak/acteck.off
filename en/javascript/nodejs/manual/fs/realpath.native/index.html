realpath.native method

Methodrealpath.nativedetermines the actual location of the path. In the first parameter we indicate the source path to the file or folder, in the second optional parameter - the encoding. In the third optional parameter we specify the callback. Despite the similarity with the methodrealpathour method works asynchronously only with the callback, i.e. Promises cannot be used in this method.

Syntax

fs.realpath.native(путь к файлу, [кодировка], [коллбэк]);

Example

Let's say we have the following file structure:

  • /dir/
    • file.txt

Let's asynchronously find out the location of the file path through a callback:

import fs from 'fs'; fs.realpath.native('./dir/file.txt', (err, resolvedPath) => { if (!err) { console.log(resolvedPath); } else { console.log(err); } });

Example

Now let's synchronously find out the full path to the file viarealpathSync.native:

import fs from 'fs'; try { let data = fs.realpathSync.native('./dir/file.txt'); console.log(data); } catch (err) { console.log(err); }
