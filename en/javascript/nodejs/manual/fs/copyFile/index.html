copyFile method

MethodcopyFilecopies the contents of the selected file to a new document. The first parameter specifies the source file, the second parameter specifies where to copy it, and the third optional parameter specifies the copy mode. In the fourth optional parameter we set the callback.

Syntax

fs.copyFile(исходный файл, куда копируем, [модификаторы], [коллбэк]);

Copy modes

Mode Description
fs.constants.COPYFILE_EXCL Indicates that if the destination file name already exists, the copy operation will fail.
fs.constants.COPYFILE_FICLONE Indicates that the copy will attempt to create a copy-write link. If the platform does not support copy-on-write, then the backup mechanism will be used.
fs.constants.COPYFILE_FICLONE_FORCE Indicates that the copy will attempt to create a copy-write link. If the platform does not support copy-on-write, the operation will fail.

Example

Let's copy the file asynchronously viaawait:

import fs from 'fs/promises'; try { let data = await fs.copyFile('file1.txt', 'file2.txt'); console.log(data); } catch (err) { console.log(err); }

Example

Let's copy the file asynchronously viathen:

import fs from 'fs/promises'; fs.copyFile('file1.txt', 'file2.txt').then(data => { console.log(data); }).catch(err => { console.log(err); });

Example

Let's copy the file asynchronously via a callback:

import fs from 'fs'; fs.copyFile('file1.txt', 'file2.txt', function(err, data) { if (!err) { console.log(data); } else { console.log(err); } });

Example

Now let's synchronously copy the file viacopyFileSync:

import fs from 'fs'; try { let text = fs.copyFileSync('file1.txt', 'file2.txt'); console.log(text); } catch (err) { console.log(err); }
