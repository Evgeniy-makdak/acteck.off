Enum as a type in TypeScript

Each enum creates its own data type. For example, let's assign the type to a variable storing the current seasonSeason:

let current: Season;

Let's write the season number into our variable:

let current: Season = Season.Winter; console.log(current); // 0

You can specify the season number manually:

let current: Season = 3;

But if you try to write data of a different type, for example, a string, you will get an error:

let current: Season = 'str'; // будет ошибка

Unfortunately, the range of values ​​is not tracked and it is possible to write down a number that is not in our enumeration:

let current: Season = 7; // ошибки не будет

When checking through an operatortypeofour variable will give a numeric type:

let current: Season = 3; console.log(typeof current); // "number"

To summarize, we can say that this type is not checked very strictly and this makes its value questionable.
