Limitations of Generics in TypeScript

In one of the previous lessons, we created a generic function that was supposed to print the length of the parameter passed to it. However, during compilation we received an error because not all types can have a length:

function myFunc < T > (data: T): T { console.log(data.length); // ошибка return data; }

In that case, we corrected the situation by changing the generic type to a generic array type. But TypeScript has the ability to create generic constraints. To do this, in angle brackets, after the variableTwrite the keywordextends. After it, we write a new variable in curly bracesstrstring type. The rest of the function code will remain unchanged:

function myFunc <T extends {str: string}> (data: T): void { console.log(data.str.length); }

It turns out that the generic type inherits the type of the variablestr, indicated in curly braces. Now let's test our function and create a variablemyStr, referring to the typestr, and give it a value in curly braces:

let myStr: {str: string} = {str: 'abcde'};

Let's call the functionmyFunc, not forgetting to specify its type again and pass the variablemyStras parameter:

myFunc <{str: string}> (myStr);
