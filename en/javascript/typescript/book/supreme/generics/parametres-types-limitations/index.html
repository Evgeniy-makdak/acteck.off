Using Generic Parameters in Generic Constraints in TypeScript

TypeScript also allows you to declare one type parameter that will be constrained by another type parameter. This is necessary when we have to copy properties from one object to another, while ensuring that we do not transfer an extra property.

Let's look at the following example. Let's go back to the functionmyFunc, which determines the length of the variable:

function myFunc <T> (data: T): T { console.log(data.length); // ошибка return data; }

However, now we want this function to only work on types that have the propertylength. For this we will create an interfaceILength, to which we will set the propertylengthnumeric type:

interface ILength { length: number; }

Next, in our function we will define a generic type that inherits fromILengthvia keywordextends:

function myFunc <T extends ILength> (data: T): T { console.log(data.length); return data; }

This is what our complete code looks like:

interface ILength { length: number; } function myFunc <T extends ILength> (data: T): T { console.log(data.length); return data; }

Let's test our function and find the length of the string:

console.log(myFunc('abcde'));

Result of the executed code:

5 'abcde'

Now let’s pass an array to the parameter:

console.log(myFunc(['a', 'b', 'c']));

Result of the executed code:

3 [ 'a', 'b', 'c' ]
