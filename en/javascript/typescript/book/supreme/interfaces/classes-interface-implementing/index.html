Interfaces for OOP classes in TypeScript

Interfaces in TypeScript are also widely used as templates for classes. In order to apply the desired interface to a class, a keyword is written after the class nameimplementsand then the interface name.

In this case, interfaces describe only public properties and methods of a class. The private part is specified in the class itself and is not described in the interface.

Let's try it in practice. Let's say we have the following interface that specifies a property and a method:

interface IUser { name: string; greet(): string; }

Let's create a class that implements this interface. In this class we must implement the propertynameand methodgreet:

class User implements IUser { name: string; constructor(name: string) { this.name = name; } greet() { return `hello, ${this.name}!`; } }
