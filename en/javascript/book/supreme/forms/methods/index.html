Form submission methods in JavaScript

Forms can be submitted in two ways: the GET method or the POST method. The method of submitting the form is controlled by the attributemethodforms.

For example, let's specify the GET submit method for the form:

<form action="/handler/" method="GET"> <input name="test1"> <input name="test2"> <input type="submit"> </form>

And now the POST method:

<form action="/handler/" method="POST"> <input name="test1"> <input name="test2"> <input type="submit"> </form>

Let us now see what is the difference between the two sending methods. In the case of the GET method, the form data will be visible in the browser as the so-calledparameter strings (query string), which are pairs of the formkey-value, where the key is the name of the form element, and the value is the data entered into it. In this case, pairs of values ​​will be separated by ampersands.

Data sent using the GET method will go to the property on our servergetobject with data:

export default { '/handler/': function(data) { console.log(data.get); // выведется в консоль сервера return 'form data received'; } }

And the data sent by the POST method will go to the property on our serverpostobject with data:

export default { '/handler/': function(data) { console.log(data.post); // выведется в консоль сервера return 'form data received'; } }

For brevity, we can destructure to get our data into a separate variable:

export default { '/handler/': function({get, post}) { console.log(get); console.log(post); return 'form data received'; } }
