Full interception of AJAX errors in JavaScript

A promise that returnsfetch, fails only if a network error occurs. If the server returned a response with the status404or500, then the promise will be completed successfully, but the statusokwill be installed infalse.

Let's catch both types of errors:

button.addEventListener('click', function() { let promise = fetch('/ajax.html') .then( response => { if (response.ok) { return response.text(); } else { console.log('плохой статус ответа'); return ''; } }, ).then( text => { console.log(text); } ).catch( error => { console.log(error); } ); });

Let's make sure that an error associated with a bad HTTP response status is also caught by the blockcatch. To do this, let’s pass it further throughthrow:

button.addEventListener('click', function() { let promise = fetch('/ajax.html') .then( response => { if (response.ok) { return response.text(); } else { throw new Error('плохой статус ответа'); } }, ).then( text => { console.log(text); } ).catch( error => { console.log(error); } ); });

Print the page text if the request was successful and an error if something went wrong.
