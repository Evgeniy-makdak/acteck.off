Choosing the optimal algorithm in JavaScript

Sometimes a code problem is due to the fact that not the most optimal solution to the problem was chosen.

Let, for example, we are faced with the task of finding the sum of integers from1to1000000. Let's solve the problem:

let sum = 0; for (let i = 1; i <= 1000000; i++) { sum += i; } console.log(sum);

What's wrong? The fact is that to solve the problem at hand there ismathematical solution.

This solution requires virtually no resources! Let's program it:

let n = 1000000; let sum = n * (n + 1) / 2; console.log(sum);

Moral: before solving a resource-intensive problem, be sure to check whether there is a ready-made mathematical formula or a ready-made mathematical approach for solving it.
