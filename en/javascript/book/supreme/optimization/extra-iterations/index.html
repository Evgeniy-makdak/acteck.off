Optimizing extra loop passes in JavaScript

In the following code, a programmer checks whether there is a number in the array3:

let arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]; let exists = false; for (let elem of arr) { if (elem === 3) { exists = true; } } console.log(exists);

What's wrong with his decision? It seems that no unnecessary operations are performed in the loop. The problem, however, is that once it is determined that the number3is in the array, the loop still continues to spin until the end of the array.

It will be most irrational if the number3will be found somewhere at the beginning of the array, and the array itself will be of length, say,1000elements. This will result in a thousand useless extra iterations of the loop! Not optimal.

Let's optimize the code by stopping the loop in time:

let arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]; let exists = false; for (let elem of arr) { if (elem === 3) { exists = true; break; } } console.log(exists);

let arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]; let sum = 0; let i = 1; for (let elem of arr) { sum += elem; if (sum <= 10) { i++; } } console.log(i);
