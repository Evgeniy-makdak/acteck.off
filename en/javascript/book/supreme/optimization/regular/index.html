Optimizing the use of regular expressions in JavaScript

Regular expressions are a very heavy and relatively slow-working thing. When there is an alternative solution, it is advisable to use it.

Let's look at an example. Suppose we need to check whether a certain line begins with an exclamation mark. A certain programmer solved this problem using regular expression:

let str = '!123'; if (/^!/.test(str)) { console.log('+++'); } else { console.log('---'); }

However, this problem has a much faster solution:

let str = '!123'; if (str[0] == '!') { console.log('+++'); } else { console.log('---'); }

The following code checks for the presence of a substring in a string'33'. Perform optimization:

let str = '123345'; if (/33/.test(str)) { console.log('+++'); } else { console.log('---'); }

The following code checks if the string ends with'.html'. Perform optimization:

let str = 'index.html'; if (/\.html$/.test(str)) { console.log('+++'); } else { console.log('---'); }

The following code trims trailing spaces. Perform optimization:

let str = ' text '; let res = str.replace(/^\s+|\s+$/g, ''); console.log(res);
