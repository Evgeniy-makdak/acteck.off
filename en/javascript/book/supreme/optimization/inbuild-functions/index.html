Optimization through built-in functions in JavaScript

Let a programmer check whether there is a number in the array3:

let arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]; let exists = false; for (let elem of arr) { if (elem === 3) { exists = true; break; } } console.log(exists);

I submit that there is something wrong with this code. What's wrong, we exit the loop after we've found the number3? The point is thatfunctions built into JavaScript always work faster than similar home-written code.

In our case there is a functionincludes, which solves the problem, and you need to use exactly this function:

let arr = [1, 2, 3, 4, 5, 6, 7, 8, 9]; console.log(arr.includes(3));

Moral: before solving a problem, be sure to check if there is a built-in JavaScript function to solve it.
