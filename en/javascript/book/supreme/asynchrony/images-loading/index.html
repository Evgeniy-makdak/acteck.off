Asynchronous image loading in JavaScript

Images that are dynamically created via JavaScript are also loaded asynchronously. Consider the following code as an example:

let image = document.createElement('img'); image.src = 'img.png'; document.body.appendChild(image);

As you can see, here we are creating a tagimg, write it insrcpath to the image and place this image inbody. The picture, however, will not appear on the page immediately. The point is that when we wrote insrcpath to the image - the browser starts downloading this image from the site. Once the image is downloaded, only then will the browser be able to display it.

If the picture is large enough, and the Internet speed is low enough, then the site user will be able to “admire” the empty picture for some time - until it loads.

Actually at the tagimgthere is an eventload, which is triggered when the image is loaded:

let image = document.createElement('img'); image.src = 'img.png'; image.addEventListener('load', function() { // сработает по загрузке картинки });

We can use this to place an image on the page only when that image is loaded:

let image = document.createElement('img'); image.src = 'img.png'; image.addEventListener('load', function() { document.body.appendChild(image); // размещаем по загрузке });

The picture will not necessarily load: it may be that the path to the picture is incorrect, or there will be an Internet outage, a breakdown of the server with the site, or something similar. In other words -exceptional situation. In this case, it is not the event that will fireload, and the eventerror:

let image = document.createElement('img'); image.src = 'img.png'; image.addEventListener('load', function() { document.body.appendChild(image); }); image.addEventListener('error', function() { // ошибка загрузки картинки });

Make a button that, when clicked, will load an image. Show the picture when it is loaded. Display a message if there are problems loading the image.
