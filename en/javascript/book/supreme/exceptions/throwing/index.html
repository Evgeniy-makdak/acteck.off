Throwing Exceptions in JavaScript

In previous lessons, we looked at two places where JavaScript throws an exception when something goes wrong.

There may be other situations in your project, however, that are exceptional for you but not for JavaScript. In this case, we can create and throw our own custom exceptions.

Let's break down the syntax required for this. First, you need to create an exception using the commandnew Error, passing the exception text as a parameter:

new Error('текст исключения');

This exception must then be thrown using the commandthrow:

throw new Error('текст исключения');

Throwing an exception causes JavaScript to believe that an exception has occurred. This means that such an exception can be caught using the constructtry-catchand process as needed.

Let's look at an example of how to use this. Let's say we have a function that divides one number by another:

function div(a, b) { return a / b; }

Let's assume that dividing by zero is prohibited and an exception should occur when attempting to do so. To do this, we will check in the function whether there is an attempt to divide by0. If not, we will divide, and if yes, we will throw an exception:

function div(a, b) { if (b !== 0) { return a / b; } else { throw new Error('ошибка деления на ноль'); } }

Let's just try dividing by0without catching the exception:

alert( div(3, 0) );

In this case, the execution of the script will be interrupted and an error with the text will appear in the console'ошибка деления на ноль'(check). Let's now intercept our error and somehow process it:

try { alert( div(3, 0) ); } catch (error) { alert('вы пытаетесь делить на 0, что запрещено'); }

In JavaScript, trying to take the root of a negative number does not throw an exception:

let result = Math.sqrt(-1); console.log(result); // выведет NaN

Write your own function that will extract the root of a number and at the same time throw an exception if the root is taken from a negative number.
