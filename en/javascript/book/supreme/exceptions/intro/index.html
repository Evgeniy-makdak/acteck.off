Introduction to Exceptions in JavaScript

Now we will look at exception situations in JavaScript. To begin with, I will make some introduction, which will contain examples of exception situations in general for any programming languages, and then we will figure out how things work in JavaScript.

When we write our programs, we implicitly expect that all the software and hardware mechanisms we use will work correctly.

This, however, is not always the case. When transmitting data over the network, the connection is broken and the data comes to us in an incorrect form or does not arrive at all. When writing a file, it turns out that the space allocated to us on the hard drive has run out and the file cannot be written. When reading the file, it turns out that such a file does not exist and we have no place to read from. When printing data on the printer, the cable connecting the printer and the computer breaks.

All the situations described have a common essence: some kind of failure occurs, which makes it impossible or pointless to complete the planned operation.

There are also situations in which some error occurs that is not a failure. For example, you ask a user for his email, and he enters the email in an incorrect format. It is clear that our program cannot continue processing the email, since it is not correct. But, nevertheless, this is not an exceptional situation. Our program can correct the situation itself: it will display an error message and the user will repeat his input.

In fact, the difference between crashing and not crashing is quite vague. An event that one program may treat as an exception may be treated by another program as some kind of error that it can handle.

The criterion here is this: if, when a problem occurs, your program can continue to do what it was designed to do, then it is not an exception, and if it cannot, then yes, it is an exception.

For example, we have a program that needs to ask for the user's email. If the user entered an email in an incorrect format, this is not a failure. This is an expected problem and our program will ask the user for an email so many times until he enters it correctly.

Let our program, which asks for an email, also have to send this correct email over the Internet. It turns out that the Internet does not work. This is already a problem: the program will not be able to send data via the Internet if the Internet does not work. The program, however, can continue to execute: it can display information about the problem, retry sending after a while, and so on. But these actions are no longer exactly what the program was intended for, since the program cannot do the main action - sending an email.

Very often, whether an exception occurs or not depends on the programming language. In many languages, if a division by zero suddenly occurs, this is considered an exception (since you cannot divide by zero), but in JavaScript it is not considered (in JavaScript you can divide by zero).
