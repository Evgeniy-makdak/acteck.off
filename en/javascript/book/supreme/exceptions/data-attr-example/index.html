An example of an exception with data attributes in JavaScript

Let the server create HTML code when loading the page, which stores the name, price and quantity of the purchased product:

<div id="product" data-product="яблоко" data-price="1000" data-amount="5"></div>

Let's make a function that will take a link to an item with a product and find the full cost of the product (price multiplied by quantity):

function getCost(elem) { return elem.dataset.price * elem.dataset.amount; }

Let's find the cost of our product:

let product = document.querySelector('#product'); let cost = getCost(product); alert(cost);

Now let's assume the following situation: due to some kind of server failure, he sent us a product that does not have a price or quantity (or both at once), for example, like this:

<div id="product" data-product="яблоко" data-price="1000"></div>

If you now try to calculate the cost of the product, the result will be displayed as NaN. Agree, not very informative.

It turns out that we need to somehow protect ourselves from the fact that the attributes we need will be missing. This can be done in two ways. The first way is to say that this is normal behavior and simply check if the attributes we need are present:

function getCost(elem) { if (elem.dataset.price !== undefined && elem.dataset.amount !== undefined) { return elem.dataset.price * elem.dataset.amount; } else { return 0; // вернем что-нибудь, например, 0 или null или false } }

The second option is to say that the absence of an attributedata-priceordata-amount- an exceptional situation. In this case we will throw an exception:

function getCost(elem) { if (elem.dataset.price !== undefined && elem.dataset.amount !== undefined) { return elem.dataset.price * elem.dataset.amount; } else { throw { name: 'ProductCostError', message: 'отсутствует цена или количество у продукта' }; } }

Which of the two options is more appropriate to use here is the choice of the programmer. He may consider the problem to be normal script operation or an exceptional situation.

Let us decide that the situation is exceptional. Then the code for getting the cost of the product will look like this:

let product = document.querySelector('#product'); try { let cost = getCost(product); alert(cost); } catch (error) { // как-то реагируем на исключение }

Change my code so that the functiongetCostthrew two types of exceptions: if the price is missing and if the quantity is missing. Think carefully about the names of these exceptions. In the blockcatchPrint different error messages for different types of exceptions.
