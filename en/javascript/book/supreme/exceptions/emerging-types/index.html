Types of situations that arise in JavaScript

There are very few situations in JavaScript in which exceptions are thrown. Firstly, simply because there are very few places for them to arise.

Secondly, because the language itself is “forgiving”: it turns a blind eye to many things, for example, division by zero or incorrect HTML code. Even if you specify the wrong path to the image you are going to upload, JavaScript will forgive you and will not consider this an exception.

However, there are exceptional situations. We will analyze the two simplest ones and use their example to study working with exceptions in JavaScript.

The first exception occurs when we want to parse invalid JSON:

let data = JSON.parse('{1,2,3,4,5}'); // данный json некорректен

And the second exception occurs when the local storage allocated for our site is full (more5megabyte). Let's artificially throw an exception like this:

let str = ''; for (let i = 1; i <= 6 * 10 ** 6; i++) { // формируем строку более 5 мб str += '+'; } localStorage.setItem('key', str); // пытаемся записать в хранилище

How does JavaScript react to such exception situations? It simply throws an error into the console and stops further execution of the script.

Our task, as programmers, is to catch such a situation and somehow cope with it,preventing the program from completely stopping execution. There is a special design for thistry-catch, which we will examine in the following lessons.

Create a string that is large enough and try to write it to local storage. Make sure there is an error in the console.

Try parsing incorrect JSON. Make sure there is an error in the console.
