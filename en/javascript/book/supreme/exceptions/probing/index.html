Forwarding exceptions in JavaScript

Consider the blockcatchtasks with product JSON:

catch (error) { if (error.name == 'SyntaxError') { alert('Некорректный JSON продукта'); } else if (error.name == 'ProductCostError') { alert('У продукта отсутствует цена или количество'); } }

As you can see, we catch the two exceptions we planned and react to it somehow. But what happens if an exception of a different type that we did not anticipate occurs? In this case, it will also end up in the blockcatch, but no reaction to this will follow, since an exception with a different type simply will not get into any of our ifs.

When I say that there will be no reaction, I mean that there really is no reaction: there will not even be an error thrown into the console. Our code simply won't work silently.

Therefore, the following rule is: your code should only catch exceptions that it knows how to handle. If the exception is not known, then it is neededforwardfurther with the helpthrow. In this case, someone more knowledgeable will catch it above, or the exception will appear as an error in the console.

Let's fix our code:

catch (error) { if (error.name == 'SyntaxError') { alert('Некорректный JSON продукта'); } else if (error.name == 'ProductCostError') { alert('У продукта отсутствует цена или количество'); } else { throw error; // пробрасываем исключение далее } }
