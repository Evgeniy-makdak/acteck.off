Using Exception Types in JavaScript

Let us have a function that takes a JSON with a data array as a parameter and writes each element of this array to local storage:

function saveData(json) { let arr = JSON.parse(json); for (let i = 0; i < arr.length; i++) { localStorage.setItem(i, arr[i]); } }

In this function, an exception can occur in two places: when parsing JSON and when trying to save data to local storage.

Suppose, for example, we decided to display some message about problems as error handling:

try { saveData('{1,2,3,4,5}'); } catch (error) { alert('какие-то проблемы'); }

Our message is, of course, good, but it does not differentiate the problems that have arisen. It would be more logical to display a message indicating exactly what problem occurred.

To do this, we can distinguish the errors that have arisen by name:

try { saveData('{1,2,3,4,5}'); } catch (error) { if (error.name == 'QuotaExceededError') { alert('закончилось место в хранилище'); } if (error.name == 'SyntaxError') { alert('некорректный JSON'); } }
