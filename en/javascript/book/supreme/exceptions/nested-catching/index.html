Catching exceptions in nested code in JavaScript

The peculiarity and convenience of exceptions is that they can be caught at any level of code nesting. Let's look at an example. Let's say we have a function that saves data to local storage:

function save(str) { localStorage.setItem('key', str); }

As you already know, when the storage is full, the methodsetItemwill throw an exception. It is not necessary, however, to catch this exception inside the functionsave. Can be wrapped intryeach call to the function itself:

try { save('некая строка'); } catch (error) { alert('закончилось место в локальном хранилище!'); }

Given a function that converts JSON to an array:

function getArr(json) { return JSON.parse(json); }

The following code gets an array from JSON:

let arr = getArr('[1,2,3,4,5]'); console.log(arr);

Wrap the function call in a constructtry-catch.
