Working with URLSearchParams in JavaScript

As you already know, when working with forms and GET requests in JavaScript, lines like this appear:

let paramsString = 'a=1&b=2&c=3';

Sometimes you need to programmatically change the value of some parameter from this line. Writing code that does this is not very convenient. That's why JavaScript has a built-in classURLSearchParamsto work with such strings.

Let's create an object of this class by passing our string with parameters as a parameter:

let paramsString = 'a=1&b=2&c=3'; let searchParams = new URLSearchParams(paramsString);

Let's see what methods the created object has.

Getting the value of a parameter

Using the methodgetyou can get the parameter value:

let res = searchParams.get('a'); console.log(res);

Let's try to get the value of a non-existent parameter:

let res = searchParams.get('x'); console.log(res); // выведет null

Checking the presence of a parameter

Using the methodhasyou can check the presence of the parameter. Let's check the existing parameter:

let res = searchParams.has('a'); console.log(res); // выведет true

Let's check for a non-existent one:

let res = searchParams.has('x'); console.log(res); // выведет false

Convert to string

Using the methodtoStringwe can convert our object back to a string:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&b=2&c=3'

Changing settings

Using the methodsetyou can change the parameter value:

searchParams.set('b', 'x');

Let's check the changes:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&b=x&c=3'

Adding parameters

Using the methodsetyou can also add a new parameter:

searchParams.set('d', '4');

Let's check the changes:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&b=2&c=3&d=4'

Adding parameters

Using the methodappendyou can also add a new parameter:

searchParams.append('d', '4');

Let's check the changes:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&b=2&c=3&d=4'

Let's try adding an already existing parameter:

searchParams.append('a', '4');

In this case, our method will simply add the parameter of the same name to the end:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&b=2&c=3&a=4'

This shows the difference between the methodssetAndappend. The first one will change the existing parameter, and the second one will simply add its double to the end.

Removing settings

Using the methoddeleteyou can delete parameters:

searchParams.delete('b');

Let's check the changes:

let res = searchParams.toString(); console.log(res); // выведет 'a=1&c=3'

Looping through parameters

You can cycle through the parameters:

for (let p of searchParams) { console.log(p); }

Several identical parameters

Let our line contain several identical parameters:

let paramsString = 'a=1&a=2b=2&c=3';

Using the methodgetAllyou can get an array of values ​​for these parameters:

let res = searchParams.getAll('a'); console.dir(res); // выведет [1, 2]

Practical problems

Let the following line with parameters be given:

let paramsString = 'test1=param1&test2=param2&test3=param3';

Add another parameter to this linetest3.

Remove the parameter from this linetest2.

Change the settingtest1to a new meaning.
