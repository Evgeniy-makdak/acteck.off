Iterators work once in JavaScript

Each iterator can loop only once. This is a little unexpected, but logical - after all, the traversed iterator establishes itsdoneVfalse, which means he can no longer move.

Let's make sure of this. Let us have the following generator:

function *func() { yield 1; yield 2; yield 3; }

Let's make an iterator:

let iter = func();

Let's go through the iterator in one loop:

for (let elem of iter) { console.log(elem); // 1, 2, 3 }

Now let’s iterate through the iterator in two loops:

for (let elem of iter) { console.log(elem); // 1, 2, 3 } for (let elem of iter) { console.log(elem); // не работает }

To solve the problem, you need to create your own iterator for each new loop:

let iter1 = func(); for (let elem of iter1) { console.log(elem); // 1, 2, 3 } let iter2 = func(); for (let elem of iter2) { console.log(elem); // 1, 2, 3 }

We can simplify:

for (let elem of func()) { console.log(elem); // 1, 2, 3 } for (let elem of func()) { console.log(elem); // 1, 2, 3 }

Correct the error in the following code:

function *func() { for (let i = 1; i <= 3; i++) { yield i; } } let elems = func(); for (let elem of elems) { console.log(elem); } for (let elem of elems) { console.log(elem); }
