Iterator Calls in JavaScript

Each successive call to an iterator must return an object of a special structure. Let's look at it. Let's say we have an array:

let arr = [1, 2, 3];

Let's get the array iterator into a variable:

let iter = arr[Symbol.iterator]();

This variable now contains an object with a methodnext. This method will return an object with the keyvalue, containing the value of the element, and the keydone, containing information about whether the search has ended or not:

console.log( iter.next() ); // {value: 1, done: false}

Sequential method callnextwill produce the following element every time:

console.log( iter.next() ); // {value: 1, done: false} console.log( iter.next() ); // {value: 2, done: false} console.log( iter.next() ); // {value: 3, done: false}

And as soon as the elements run out, we will see the following values:

console.log( iter.next() ); // {value: undefined, done: true}
