Exceptions in asynchronous callbacks in JavaScript

Let, if the parametermakethe number of a non-existent array element is passed - this is an exceptional situation. As you already know, exceptions thrown inside an asynchronous function cannot be caught viatry-catch. In our case, the exception that occurred insidemakeor callback will not be caught:

try { make(10, function(res) { console.log(res); }); } catch(err) { // не поймается }

In the callback approach, exceptions are handled as follows: the result is sent to the first parameter of the callback, and the error is sent to the second parameter. In this case, error handling occurs as follows:

make(10, function(res, err) { if (!err) { console.log(res); // ошибки не возникло, выведем результат } else { console.log(err); // ошибка возникла, выведем ее текст } });

Let's rewrite our function codemakeas described:

function make(num, callback) { setTimeout(function() { let arr = [1, 2, 3, 4, 5]; let err; if (arr[num] === undefined) { err = 'elem not exists'; // текст ошибки } else { err = null; // ошибки нет } callback(arr[num], err); }, 3000); }
