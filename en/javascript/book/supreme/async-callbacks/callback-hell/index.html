Callback hell problem in JavaScript

Let us want using the functionloadImageupload three pictures:

loadImage('img1.png', function(image, err) { document.body.append(image); }); loadImage('img2.png', function(image, err) { document.body.append(image); }); loadImage('img3.png', function(image, err) { document.body.append(image); });

There is something wrong with this code. The point is that the pictures will be added tobodyas they are downloaded. That is, no one guarantees us that the pictures will be added in exactly the order we need.

There's one more thing. Let's say we want to do something when all three images are loaded. In our code we simply cannot catch this moment, sinceall three images are loaded independently.

Okay, let's change the code:

loadImage('img1.png', function(image1, err1) { document.body.append(image1); loadImage('img2.png', function(image2, err2) { document.body.append(image2); loadImage('img3.png', function(image3, err3) { document.body.append(image3); console.log('все картинки загружены'); }); }); });

We have solved both problems described. However, they received another one in return. It’s not very visible yet, but imagine what our code will look like if it loads not three, but, say, ten pictures, plus exception handling is added. As a result, the code will become extremely unreadable:The complexity of the code increases like an avalanche when friend callbacks are nested. This situation is calledcallback hell- hell of callbacks.
