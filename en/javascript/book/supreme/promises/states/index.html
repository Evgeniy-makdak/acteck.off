Promise states in JavaScript

A promise can be in one of three states. When created, the promise is pending (pending), and then can become executed (fulfilled), returning the result, or rejected (rejected), returning the reason for refusal. You can watch the transition from one state to another by running the following code:

let promise = new Promise(function(resolve, reject) { setTimeout(function() { let isError = false; // поставьте либо true, либо false if (!isError) { resolve([1, 2, 3, 4, 5]); } else { reject('error in promise'); } }, 3000); }); setInterval(function() { console.log(promise); // каждую секунду выводим промис в консоль }, 1000);

Please note that statesfulfilledAndrejectedare immutable: if a promise has passed into one of these states, then it will no longer be able to move to another. Let's look at an example. In the following code the callrejectwill happen earlier, so the callresolvewill be ignored:

let promise = new Promise(function(resolve, reject) { setTimeout(() => reject(new Error('error')), 1000); setTimeout(() => resolve('ignored'), 2000); });
