Exceptions in Promises in JavaScript

Let's now learn how to handle exceptions that occur inside a promise. If such a situation arises, we must reject the promise using a special reject function, which automatically ends up in the second parameter of the promise function:

let promise = new Promise(function(resolve, reject) { setTimeout(function() { ... }, 3000); });

Inside the promise function we must callresolve, if everything went smoothly, orrejectif an exception occurs:

let promise = new Promise(function(resolve, reject) { setTimeout(function() { let isError = false; if (!isError) { resolve([1, 2, 3, 4, 5]); // данные промиса } else { reject('error in promise'); // ваш текст ошибки } }, 3000); });

Then in the methodthenYou must pass not one, but two functions as parameters: the first one will work if the promise worked normally (resolved), and the second - if it worked with an error (resigned):

promise.then( function(result) { console.log(result); // выведет результат промиса }, function(error) { console.log(error); // выведет текст ошибки } );

As a rule, the above code is written more compactly, like this:

promise.then(function(result) { console.log(result); // выведет результат промиса }, function(error) { console.log(error); // выведет текст ошибки });

Make an asynchronous code that will generate random numbers from0to5. Wrap it all in a promise. Let the promise as its result return the result of dividing one by the generated number. Let the promise be executed with an error if division by zero occurred, and with success in all other cases.
