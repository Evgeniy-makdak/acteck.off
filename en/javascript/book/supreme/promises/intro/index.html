Introduction to Promises in JavaScript

You already know that using the asynchronous callback model easily leads to a callback hell situation. Therefore, a new model was introduced into JavaScript calledpromises (promise). Let's study this model.

A promise is an object into which a function is passed as a parameter, inside which we need to place our asynchronous code:

let promise = new Promise(function() { // асинхронный код });

As you can see, I wrote an object with a promise to a variablepromise. Somewhere else in the code I can apply a method to this variablethen, passing into it a function with code that should be executed upon completion of the asynchronous code written when creating this promise:

promise.then(function() { // выполнится при завершении асинхронного кода });

This sounds confusing, so let's look at an example. Let me have this asynchronous code:

setTimeout(function() { let result = [1, 2, 3, 4, 5]; }, 3000);

Let me want to solve our main asynchrony problem for it: execute some code after the timer fires. At the same time, I don’t want to place this code in the timer itself and I want the result that I wrote in the variable to somehow get into this coderesult. In general, we solved this problem in previous lessons through callbacks and subscriptions. Let's now see how to do this through promises.

First we need to wrap our asynchronous code in a promise:

let promise = new Promise(function() { setTimeout(function() { let result = [1, 2, 3, 4, 5]; }, 3000); });

This, however, is not enough. We mustexplicitly indicate that our asynchronous code has completed. A special completion function will help us with this, automatically going into the first parameter of the function if it is specified:

let promise = new Promise(function(resolve) { // указываем параметр setTimeout(function() { let result = [1, 2, 3, 4, 5]; }, 3000); });

With the completion function, we can explicitly indicate to a promise that asynchronous code has completed. To do this, we must call this function in the place we need:

let promise = new Promise(function(resolve) { setTimeout(function() { let result = [1, 2, 3, 4, 5]; resolve(); // завершаем промис }, 3000); });

At the same time, if we want to pass some result of asynchronous code externally, we can pass it as a parameter to our completion function:

let promise = new Promise(function(resolve) { setTimeout(function() { let result = [1, 2, 3, 4, 5]; resolve(result); // передаем результат }, 3000); });

You can, of course, get rid of the intermediate variable:

let promise = new Promise(function(resolve) { setTimeout(function() { resolve([1, 2, 3, 4, 5]); }, 3000); });

Now in any other place we can call the methodthenof our promise:

promise.then(function() { // сработает по завершению промиса });

The result of the promise will end up in the first parameter of the function if we wish to specify it:

promise.then(function(result) { console.log(result); // выведет массив с результатом });
