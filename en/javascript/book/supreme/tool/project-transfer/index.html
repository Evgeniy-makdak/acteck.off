Porting an npm project to JavaScript

Foldernode_modulestakes up a lot of space and contains many small files. Therefore, when you upload your project to hosting or give the project to a colleague, you should not give away the folder with libraries.

The fact is that your colleague can easily install the libraries written in the filepackage.json. Let's see how it's done. Place the following file in an empty folder:

{ "dependencies": { "jquery": "^3.6.1", "lodash": "^4.17.21" } }

Now run a terminal command which will install all the projects described inpackage.json:

npm install

Let them send you the next onepackage.json:

{ "dependencies": { "jquery": "^3.6.1", "underscore": "^1.13.6" }, "devDependencies": { "webpack": "^5.74.0" } }

Place it in an empty folder and run the installation of libraries from this file.
