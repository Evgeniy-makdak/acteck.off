Dependency version formats in npm in JavaScript

In filepackage.jsonVersions of installed packages can be specified in different formats. Let's explore these formats using the imaginary somelib library as an example.

The following example requires the exact version2.4.2:

{ "dependencies": { "somelib": "2.4.2" } }

The following example requires the latest version compatible with2.4.2:

{ "dependencies": { "somelib": "^2.4.2" } }

In the following example, the project works with versions2.4.2, 2.4.3, 2.4.4and so on:

{ "dependencies": { "somelib": "~2.4.2" } }

In the following example, the project works with versions2.4, 2.5, 2.6and so on:

{ "dependencies": { "somelib": "~2.4" } }

In the following example, the project works with any patch version of the package2.4:

{ "dependencies": { "somelib": "2.4.x" } }

In the following example, the project works with any minor version of the version2:

{ "dependencies": { "somelib": "2.x" } }

In the following example, the project works with version2.4and higher:

{ "dependencies": { "somelib": ">=2.4" } }

In the following example, the project works with any version ranging from2.4.2to3.1.1:

{ "dependencies": { "somelib": "2.4.2 3.1.1" } }
