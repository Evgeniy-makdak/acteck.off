JavaScript package.json file settings

When a programmer wants to make his library available to everyone, he must place it inregister(i.e. in the repository) npm. To do this, he must register the filepackage.jsonthis library in a special way, adding some important settings.

These settings apply specifically to public libraries that you download from npm. In your projects for private use, you don't have to install them.

So, here is a list of the most popular settings:

Meaning Description
name project name, must be written in lowercase letters
version version number, according to semantic versioning
description project description
main project input file
keywords this is an array that helps you find a module in the npm registry
author this field accepts an object with keysname, emailAndurl, allowing people to easily contact the author of the project.
license accepts the license name using the SPDX identifier. The default license is ISC, another popular choice is MIT. You can also use UNLICENSED for private and closed source projects.
private if this key is set to true, the project will not be able to be public in the npm registry. This helps prevent accidental publication of the project.
homepage Project home page URL
bugs A URL where you can report problems and bugs. This is often the URL of the project's Github page.
