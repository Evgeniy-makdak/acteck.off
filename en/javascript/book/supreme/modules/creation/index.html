Creating an ES module in JavaScript

Let's now start studying the modules. Each module must be a separate file. The variables and functions that we will place in this file will not be accessible from outside the file. This is the advantage of modules - we can create any variables and functions without fear of conflict with names from other libraries.

Those variables and functions that we want to make visible externally, we must export using the commandexport.

Let's try it with an example. Let's make a modulemath, which will provide a library of functions for mathematical operations.

Let's place our module in the filemath.jsand make several functions in it:

function root2(num) { return round(num ** (1 / 2)); } function root3(num) { return round(num ** (1 / 3)); } function round(num) { return num.toFixed(2); }

Let the functions for extracting roots be the main ones and will be exported outside, and the function for rounding will be auxiliary and will not be exported. Let's write the command to the necessary functionsexport:

export function root2(num) { return round(num ** (1 / 2)); } export function root3(num) { return round(num ** (1 / 3)); } function round(num) { return num.toFixed(2); }

Make a module containing a functionpow2to raise a number to the second power,pow3for construction into the third andpow4for construction into the fourth.
