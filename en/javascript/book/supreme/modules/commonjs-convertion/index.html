Converting CommonJS modules in ES to JavaScript

Until now, in various documentation you can find imports in the CommonJS style of modules. You need to learn how to convert these imports into standard ES modules.

In CommonJS, imports are done using the commandrequire, whose parameter specifies the path to the module. The result of the command is written to a variable:

let math = require('./math');

Let's rewrite this code in ES style. First, please note that CommonJS does not include the extension.jsfor files, and in ES it is installed.

Next, everything depends on whether the module exports by default or normal. There is no difference in CommonJS, but there is in ES modules. As a rule, the difference is visible in the code examples, or you can simply try both import options.

In our case, the above import will be rewritten either in this form:

import math from './math.js';

Or like this:

import * as math from './math.js';

Rewrite the following code in ES module style:

let {square, cube} = require('./math');

Rewrite the following code in ES module style:

let math = require('./math'); let res = math.square(2) + math.cube(3);

Rewrite the following code in ES module style:

let sum = require('./sum'); let res = sum([1, 2, 3]);
