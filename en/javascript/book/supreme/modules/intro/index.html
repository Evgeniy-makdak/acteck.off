Introduction to Modern Modules in JavaScript

Have you already worked withmodules via closures. In modern JavaScript, more convenient modules have appeared that allow you to place each module in a separate file, and then, if necessary, connect to other files, exporting only the necessary variables and functions.

Initially, such modules were not enshrined in the JavaScript standard and were simply developed by the community. Accordingly, many different modular systems have appeared: AMD, CommonJS, UMD and others.

Then the JavaScript standard introduced its own modules. They are called ES modules. Because of this, community-developed modules are rapidly becoming outdated and should not be used.

It’s worth paying attention only to CommonJS modules, since NodeJS originally worked on them, and now it works on them by default for compatibility reasons.

In addition, often in instructions for connecting various libraries you will find a description of the CommonJS option, and not ES. Therefore, you should be aware of how to rewrite CommonJS code in ES.

There is another nuance. Although modules are now enshrined in the JavaScript standard, they simply won't work in the browser (yet). To do this, you need to use one of the popular build tools.
