Escaping special characters in JavaScript regular expressions

Let's assume that we want to make a special character represent itself. To do this, you need to escape it using a backslash. Let's look at examples.

Example

In the following example, the author of the regex wanted the search template to look like this:letter'a', then plus'+', then the letter'x'. However, the author of the code did not escape the symbol'+'and so the search template actually looks like this:letter'a'one or more times, then a letter'x':

let str = 'a+x ax aax aaax'; let res = str.replace(/a+x/g, '!');

As a result, the following will be written to the variable:

'a+x ! ! !'

Example

And now the author has escaped the plus with a backslash. Now the search template looks like this:letter'a', then plus'+', then the letter'x'.

let str = 'a+x ax aax aaax'; let res = str.replace(/a\+x/g, '!');

As a result, the following will be written to the variable:

'! ax aax aaax'

Example

In this example, the template looks like this:letter'a', then dot'.', then the letter'x':

let str = 'a.x abx azx'; let res = str.replace(/a\.x/g, '!');

As a result, the following will be written to the variable:

'! abx azx'

Example

In the following example, the author forgot to escape the slash and all substrings fell under the regular expression, since an unescaped dot denotes any character:

let str = 'a.x abx azx'; let res = str.replace(/a.x/g, '!');

As a result, the following will be written to the variable:

'! ! !'

Comment

Note that if you forget the backslash for a dot (when it should denote itself), you might not even notice it:

'a.x'.replace(/a.x/g, '!'); // вернет '!', как мы и хотели

Visually it works correctly (since the dot denotes any symbol, including a regular dot'.'). But if we change the line in which the replacements occur, we will see our error:

'a.x abx azx'.replace(/a.x/g, '!'); // вернет '! ! !', а ожидалось '! abx azx'

List of special and regular characters

If you escape an ordinary symbol, nothing bad will happen - it will still represent itself. The exception is numbers; they cannot be escaped.

There is often doubt as to whether a given character is special. Some go so far as to escape all suspicious characters in a row. However, this is bad practice (it clutters up the regex with backslashes).

They are special characters:$ ^ . * + ? \ / {} [] () |

They are not special characters:@ : , ' " ; - _ = < > % # ~ `& !

Practical problems

Given the line:

let str = 'a.a aba aea';

Write a regular expression that will find the string'a.a'without capturing the rest.

Given the line:

let str = '2+3 223 2223';

Write a regular expression that will find the string'2+3'without capturing the rest.

Given the line:

let str = '23 2+3 2++3 2+++3 345 567';

Write a regular expression that finds strings'2+3', '2++3', '2+++3', without capturing the rest (+ can be any number).

Given the line:

let str = '23 2+3 2++3 2+++3 445 677';

Write a regular expression that finds strings'23', '2+3', '2++3', '2+++3'without capturing the rest.

Given the line:

let str = '*+ *q+ *qq+ *qqq+ *qqq qqq+';

Write a regular expression that finds strings'*q+', '*qq+', '*qqq+'without capturing the rest.

Given the line:

let str = '[abc] {abc} abc (abc) [abc]';

Write a regular expression that finds strings in square brackets and replaces them with'!'.
