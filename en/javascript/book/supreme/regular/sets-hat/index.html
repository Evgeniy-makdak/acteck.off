The cap character inside sets in JavaScript regular expressions

As you know, the hat is inside[ ]makes a negation when written at the beginning of parentheses. This means that it is a special character inside these brackets. To get a hat as a symbol, you need to either screen it or remove it from the first place.

Example

In the following example, the search pattern is:the first character is everything except'd', then two letters'x'.

let str = 'axx bxx ^xx dxx'; let res = str.replace(/[^d]xx/g, '!');

As a result, the following will be written to the variable:

'! ! ! dxx'

Example

And now the search template is like this:the first character is'd'or'^', then two letters'x':

let str = 'axx bxx ^xx dxx'; let res = str.replace(/[d^]xx/g, '!');

As a result, the following will be written to the variable:

'axx bxx ! !'

Example

You can not remove the hat from the first place, but simply escape it using a backslash, and it will begin to represent itself:

let str = 'axx bxx ^xx dxx'; let res = str.replace(/[\^d]xx/g, '!');

As a result, the following will be written to the variable:

'axx bxx ! !'

Practical problems

Given the line:

let str = '^xx axx ^zz bkk @ss';

Write a regular expression that finds strings using the pattern:a hat or a dog, and then two Latin letters.

Given the line:

let str = '^xx axx ^zz bkk @ss';

Write a regular expression that finds strings using the pattern:NOT a hat and not a dog, and then two Latin letters.

Given the line:

let str = '^xx axx ^zz bkk';

Write a regular expression that finds strings using the pattern:not a cap and not a space, and then two Latin letters.
