Pockets in the match method in JavaScript regulars

As you already know, found through the methodmatchthe substring ends up in the zero element of the array with the result. The remaining elements of this array include the so-calledpockets.

Pockets are a way to break down what you find into separate parts. To use them, you need to enclose part of the regular expression in parentheses. In this case, in the array with the result, in addition to the found string, the one that falls under the regular expression in these brackets will also appear.

Let's try it in practice. Let us have a line like this with time:

let str = '12:34';

Let's put the hours and minutes of this time into separate pockets:

let res = str.match(/(\d+):(\d+)/);

Let's look at the result:

console.log(res[0]); // '12:34' - найденное console.log(res[1]); // '12' - 1 карман console.log(res[2]); // '34' - 2 карман

Given a string containing the domain:

let str = 'sss domain.ru zzz';

Find that domain and put its name in the first pocket and the zone in the second.

Given a string containing a date:

let str = '31.12.2025';

Put the day in the first pocket, the month in the second, and the year in the third.
