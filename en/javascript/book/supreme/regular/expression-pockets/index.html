Pockets in the regex itself in JavaScript

The contents of pockets are available not only in the replacement line, but also in the regex itself: we can put something in a pocket, and then directly in the regex say that the contents of this pocket should go here.

The contents of pockets are identified by their numbers, preceded by a backslash. For example, the first pocket will be accessible like this:\1, second pocket like this -\2, third -\3and so on.

I am sure that everything written above is still very vague for you. This is not surprising, since pockets are the most obscure place in regulars. Let's look at examples.

Example

Let's say we have this line:

let str = 'aa bb cd ef';

Let's find all the places in it where any two identical letters appear in a row. To solve the problem, we will look for any letter, put it in a pocket, and then check whether the next symbol matches the contents of this pocket:

let res = str.replace(/([a-z])\1/g, '!');

As a result, the following will be written to the variable:

'! ! cd ef'

Example

Let's say we have this line:

let str = 'asxca buzxb csgd';

Let's find all the words in it in which the first and last letters are the same. To solve the problem, we will write the following template:a letter, then one or more letters, and then the same letter as the first:

let res = str.replace(/([a-z])[a-z]+\1/g, '!');

As a result, the following will be written to the variable:

'! ! csgd'

Practical problems

Given the line:

let str = 'aaa bbb ccc xyz';

Find all substrings that contain three identical letters in a row.

Given the line:

let str = 'a aa aaa aaaa aaaaa';

Find all substrings that contain two or more identical letters in a row.

Given the line:

let str = 'aaa aaa bbb bbb ccc ddd';

Find all substrings that contain two identical words in a row.
