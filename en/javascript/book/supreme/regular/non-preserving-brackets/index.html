Non-preserving parentheses in JavaScript regexes

Parentheses( )perform two functions - symbol grouping and pocket function. What if we need to group it, but not put it in our pocket? To solve this problem, specialnon-preserving parentheses (?: )- they group, but do not put in a pocket.

Example

In the following example, we need the first brackets for grouping, and the second ones for pocketing. However, both parentheses save data to the pocket:

let str = 'abab123'; let res = str.match(/(ab)+([1-9]+)/);

As a result, our pockets will have the following:

console.log(res[0]); // выведет 'abab123' console.log(res[1]); // выведет 'ab' console.log(res[2]); // выведет '123'

Example

Let's make sure that the first pair of brackets only groups, but does not put in a pocket:

let str = 'abab123'; let res = str.match(/(?:ab)+([1-9]+)/);

As a result, the first pocket will contain our number:

console.log(res[1]); // выведет '123'
