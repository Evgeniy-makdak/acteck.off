The Inconvenience of Classes in JavaScript

Let's say we have several classes for styling messages:

.success { color: green; } .warning { color: orange; } .error { color: red; }

Let us display a successful message in some element and assign it the appropriate class for success:

elem.classList.add('success');

Let us now display an error message in the same element and assign it the appropriate class for the error:

elem.classList.add('error');

As a result, the element will have two classes that conflict with each other:

<div id="elem" class="success error"> text </div>

It turns out that before adding a new class, we will have to first remove the previous one:

elem.classList.remove('success'); elem.classList.add('error');

This is not very convenient, since we may not know what exactly the previous class was and we will have to delete all classes in a row:

elem.classList.remove('success'); elem.classList.remove('warning'); elem.classList.add('error');
