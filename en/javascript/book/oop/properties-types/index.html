Accessor properties in OOP in JavaScript

There are two types of object properties.

The first type is data properties (data properties). We already know how to work with them. All the properties we have used up to now have been data properties.

We have not yet considered the second type of properties. These are accessor properties (accessor properties). At their core, these are functions that are used to assign and get a value, but in external code they look like ordinary object properties.

Accessor properties are created using keywordsgetAndsetwritten when declaring methods:

class User { get name() { } set name(name) { } }

As a result of such a declaration, our class will have the propertyname. In this case, when trying to read this property, the method declared viaget:

let name = user.name;

When you try to write, the method declared viaset:

user.name = 'john';

Given the following class:

class Employee { get name() { console.log('get'); } set name(name) { console.log('set'); } }

Create an object of this class and try to write and read data from the propertyname.
