Calling an overridden method in OOP in JavaScript

When overridden, the child loses access to the parent's overridden method. However, you can still access it. This is done using the keywordsuper, which points to the parent class.

Let's look at an example where we might need access to a parent method. Let us have the following parent class:

class User { setName(name) { this.name = name; } getName() { return this.name; } }

Let us override the parent method in the descendant class:

class Student extends User { setName(name) { if (name.length > 0) { this.name = name; } else { throw new Error('student name error'); } } }

You may notice that in an overridden method, when the condition is met, the code of the parent method is actually executed. This results in unnecessary code duplication.

You can get rid of it by calling the parent method. Let's do this:

class Student extends User { setName(name) { if (name.length > 0) { super.setName(name); // метод родителя } else { throw new Error('student name error'); } } }
