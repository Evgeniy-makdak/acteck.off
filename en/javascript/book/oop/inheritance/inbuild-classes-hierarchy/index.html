Hierarchy of built-in classes in OOP in JavaScript

Built-in classes also have a hierarchy. Let's look at an example paragraph:

<p>text</p>

Let's get a link to a paragraph into a variable:

let elem = document.querySelector('p');

Let's look at our paragraph in the console:

console.dir(elem);

As you already know, in the property[[Prototype]]contains the name of the paragraph class. ThisHTMLParagraphElement. If you expand the list of properties and methods of this class, then at the bottom you will again find the property[[Prototype]], which will already contain the parent class and this will beHTMLElement. You can also find a parent for it, and so on.

Explore the class hierarchy for this tag:

<div>text</div> let elem = document.querySelector('div'); console.dir(elem);

Explore the class hierarchy for this tag:

<input> let elem = document.querySelector('input'); console.dir(elem);

Explore the class hierarchy for this collection:

<div>text</div> <div>text</div> <div>text</div> let elems = document.querySelectorAll('div'); console.dir(elems);

Explore the class hierarchy for this collection:

<div> <p>text</p> <p>text</p> <p>text</p> </div> let elem = document.querySelector('div'); let elems = elem.children; console.dir(elems);
