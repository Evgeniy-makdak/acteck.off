Benefits of Setters and Getters in OOP in JavaScript

In the previous lesson, we made a getter and a setter for each property. One might wonder why such complexity is necessary, since in fact the same effect can be achieved by making properties public rather than private.

The point is that getters and setters have the advantage that you can perform some checks before accessing the property. For example, in our case, when writing a first and last name, we can check that the new value is not an empty string:

class User { #name; #surn; setName(name) { if (name.length > 0) { this.#name = name; } else { throw new Error('name is incorrect'); } } setSurn(surn) { if (surn.length > 0) { this.#surn = surn; } else { throw new Error('surn is incorrect'); } } getName() { return this.#name; } getSurn() { return this.#surn; } }

Let's check how it works. First, let's create a class object:

let user = new User;

Now let's try to write down the correct value:

user.setName('john');

Now let’s try to write down the incorrect one:

user.setName(''); // ошибка
