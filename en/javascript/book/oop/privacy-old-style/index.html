Old style privacy in OOP in JavaScript

Declaring private properties and methods using a symbol#appeared in JavaScript quite recently. Before this, I had to use a special tricky technique.

The essence of this technique is that there is an agreement that the names of private properties and methods must begin with an underscore character. In reality, such names will not be private, but we agree that we will not use them outside the class.

Let's try it. Let's create a private property in this style:

class User { constructor(name) { this._name = name; } show() { return this._name; } }

Let's create a class object:

let user = new User('john', 'smit');

Let's call the method that displays the value of our property:

console.log(user.show());

Trying to access a property from outside the class will not produce an error:

console.log(user._name); // ошибки нет

In the following code, make the properties private:

class Employee { constructor(name, salary) { this.name = name; this.salary = salary; } getName() { return this.name; } getSalary() { return this.salary; } }

In the following code, make the helper method private:

class Employee { constructor(name, salary) { this.name = name; this.salary = salary; } getSalary() { return this.addSign(this.salary); } addSign(num) { return num + '$'; } }
