Editing purchases in the grocery calculator

Let's now implement cell editing. According to the condition, only the first ones can be edited3cells.

Here they are:

createCell(tr, name.value, 'name'); createCell(tr, price.value, 'price'); createCell(tr, amount.value, 'amount');

Obviously, the editing code will be the same for each cell: by double clicking, an input should appear instead of the cell text, and by clickingEnterin the input, its contents should become the new cell text.

I propose to make some function that will take a table cell as a parameter and bind editing capabilities to it:

function allowEdit(td) { td.addEventListener('dblclick', function() { }); }

Then we can apply our function like this:

allowEdit(createCell(tr, name.value, 'name')); allowEdit(createCell(tr, price.value, 'price')); allowEdit(createCell(tr, amount.value, 'amount'));

An attentive reader may notice that not all cells are the same: after all, when changing a cell with a price or a cell with a quantity, the purchase cost in the column should changecost, and the total amount was recalculated.

We will leave this nuance as the next subtask to solve in the next lesson. To begin with, we’ll just make it possible to edit our three cells, without recalculation.
