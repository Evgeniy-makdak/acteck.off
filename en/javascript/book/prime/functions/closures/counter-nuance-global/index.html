Global Counter Variable in JavaScript

Let's take out the variablenumfor functions, thereby making it global:

let num = 1; // глобальная переменная function test() { return function() { console.log(num); num++; }; }

In this case, all functions returned will change this global variable and the counters will work independently of each other:

let num = 1; function test() { return function() { console.log(num); num++; }; } let func1 = test(); // первый счетчик func1(); // выведет 1 func1(); // выведет 2 let func2 = test(); // второй счетчик func2(); // выведет 3 func2(); // выведет 4

Why did our previous code make independent counters? Let me remind you of this code:

function test() { let num = 1; return function() { console.log(num); num++; }; };

The point is that the variablenum- local inside the functiontest. Therefore every calltestcreates its own local variable.

Therefore, the returned functions will each refer to their local function variabletest. This is how work independence is achieved.

If you donumglobal variable - this will also be a closure. It's just that the lexical environments of the functions returned refer to the same variablenum- any changes to this variable will be visible in all functions.
