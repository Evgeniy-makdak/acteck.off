Counter on closures in JavaScript

Let's rewrite the code we looked at so that the returned function increments the value of the variable each timenumper unit:

function test() { let num = 1; return function() { console.log(num); num++; // прибавляем единицу } } let func = test();

It turns out that each function callfuncwill output the new value to the console:

function test() { let num = 1; return function() { console.log(num); num++; } } let func = test(); func(); //выведет 1 func(); //выведет 2 func(); //выведет 3 func(); //выведет 4 func(); //выведет 5

It turns out that we have implemented a function call counter using a closure (more precisely, using a variablenumfrom the closure of our function).

Please note that each function calltestwill return a new function that will have its own closure. That is, different counters will work independently:

function test() { let num = 1; return function() { console.log(num); num++; }; } let func1 = test(); // первый счетчик func1(); //выведет 1 func1(); //выведет 2 let func2 = test(); // второй счетчик func2(); //выведет 1 func2(); //выведет 2

It turns out that the same variablenumwill have different meanings for different functions!

That is, if we call the functiontesttwice, then the functions derived from it will work independently and each of these functions will have its own independent variablenum.
