Semicolon for safety in IIFE

Let's now let our in-place function be called not with a plus at the beginning, but wrapped in parentheses, like this:

(function() { console.log(1); // выведет 1 })();

Let the variablenumis specified outside the function:

let num = 1; // точка с запятой стоит (function() { console.log(num); // выведет 1 })();

Suppose now we forgot to put a semicolon:

let num = 1 (function() { console.log(num); //!! выдаст ошибку })();

It turns out that such code will generate an error, since JavaScript perceives our function as a continuation of the first line command.

To avoid such problems, you should always use a semicolon before calling a function in place, like this:

let num = 1 ;(function() { console.log(num); // выведет 1 })();

It seems that the problem is somewhat far-fetched. In fact, this is not true. There may be a situation where you have several scripts from separate files connected to your page. In this case, a semicolon may be omitted at the end of one file and this will automatically cause a problem if the second file begins with an in-place function call.

Therefore, always, always put a semicolon before calling a function in place, even if you are currently confident that there will be no problems. They may arise later.

Let's apply the above and call the function in place, prefixed with a semicolon:

;(function() { console.log(1); // выведет 1 })();

Determine, without running the code, what will be output to the console:

let str = 'str'; (function() { console.log(1); })();

Determine, without running the code, what will be output to the console:

let str = 'str' (function() { console.log(1); })();
