The fine point of return in JavaScript

After the instruction is executedreturn- the function will finish its work and no further code will be executed. See example:

function func(num) { return num ** 2; console.log('!'); // этот код никогда не выполнится } let res = func(3);

This does not mean that there should be one in the functionreturn. But only one of them will be executed. In the following example, depending on the value of the parameter, either the first or the second will be executedreturn:

function func(num) { if (num >= 0) { return '+++'; } else { return '---'; } } console.log(func( 3)); // выведет '+++' console.log(func(-3)); // выведет '---'

What will be output to the console as a result of executing the following code:

function func(num) { return num; let res = num ** 2; return res; } console.log( func(3) );
