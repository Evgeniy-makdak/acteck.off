Named Function Expressions in JavaScript

Let us have this functional expression:

let test = function() { console.log('!'); }; test(); // выведет '!'

Let's give our function a namefunc:

let test = function func() { console.log('!'); }; test();

As you already know, you cannot access a function by this name:

let test = function func() { console.log('!'); }; test(); // выведет '!' func(); // выдаст ошибку

Why give a function name if it won't be available? The point is that this name will not be available outside the function, but is available inside this function.

Let's check:

let test = function func() { console.log(func); // функция выведет свой же исходный код }; test(); // вызываем функцию

Let's call our function inside itself:

let test = function func() { console.log('!'); // выводим '!' func(); // вызываем сами себя }; test();

If you run this code, an infinite number of outputs will be printed to the console.

In fact, our function can be called internally not only asfunc, but also howtest:

let test = function func() { console.log('!'); test(); // вызываем сами себя }; test();

What's the difference? The difference is that the nametest- it's just a variable. While the script is running, our function may be written to another variable or passed a parameter - in this case, the connection between the variabletestand the function may disappear.

Namefuncit is strictly tied to the function specifically so that we can access our function within it itself.

Such Function Expression with the name of the function are callednamed function expressions.
