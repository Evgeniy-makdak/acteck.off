The nuances of function expressions in JavaScript

The name Function Expression is given for a reason. What it really means is that such functions are part of some expression.

For example, we can add some string and an unnamed function:

let str = 'str' + function() {return 3;}; console.log(str); // выведет 'strfunction() {return 3;}'

Why do we see such a strange result and not a number?3? Because the second term is not the result of the function, but its source code (after all, we did not call this function, but simply wrote it).

That is the namefunctional expressionmeans that such a function takes part in some expression.

Assignment to a variable is also an expression:

let func = function() { console.log('!'); };

You can also, for example, pass a function as a parameter toconsole.logand it will output its source code to the console - this will also be considered an expression:

console.log(function() {return 3;});

Why is this important: because the difference between Function Declaration and Function Expression is not at all that the first function is created with a name, and the second does not initially have a name. This is wrong.

Example. Here we have a function without a name, but it does not participate in any expression (that is, no actions are performed with it, to put it simply):

/* Данная функция будет Function Declaration, но с синтаксической ошибкой: */ function() { console.log('!'); }

This code will generally generate an error! Why: since the function is not involved in any expression, the browser considers it a Function Declaration, but does not find its name and throws an error.

For the error to disappear, you need to force the function to become part of an expression. For example, let's write the operation in front of it+:

+function() { // такой код корректен console.log('!'); };

How it works: the operation itself+doesn't do anything, it's like writing a number instead3number+3- acceptable, but does not change anything.

But in the case of a function, it changes. Now our function is no longer just written, but participates in the expression. Therefore, there will be no error now. There will be no result from executing the function either, because we just wrote it, but did not call it.

Instead of+you can write anything you want. For example:

-function() { // такой код корректен console.log('!'); }; !function() { // такой код корректен console.log('!'); };

You can also put our function in parentheses, in which case it also becomes a function expression:

(function() { // такой код корректен console.log('!'); });
