Function with name but Function Expression in JavaScript

Let's now make a function that will have a name, but will also be a Function Expression because it participates in the expression:

+function func() { console.log('!'); }

I wonder what by namefuncwe will not be able to access our function, this will result in an error:

+function func() { console.log('!'); } func(); //!! выдаст ошибку

To be able to call our function, it needs to be assigned to some variable:

let test = function func() { console.log('!'); }; test(); // выведет '!'

Once again: a function that is a function expression cannot be called by its name. Such a function can only be called using the variable in which the function was written.

But nevertheless, a function expression can have the name of a function, this will be syntactically correct. Why this is necessary, we will understand in the following lessons.

To summarize: functions are Function Declaration or Function Expression not because they have a name or not, but because they are participants in expressions or are not.

As you saw above, a function without a name can be thought of as a Function Declaration, while a function with a name can be a Function Expression.
