Problem with optional curly braces in if-else

Although JavaScript allows the curly brace abbreviations shown above, I generally don't recommend doing so, as it is a breeding ground for subtle errors. For example, let's say you have the following code:

let test = 3; if (test > 0) console.log(test);

Let us now decide, if the condition is met, to also display a second message with the text'+++':

let test = 3; if (test > 0) console.log(test); console.log('+++');

However, without the curly braces, only the first message is inside the condition. That is, the first line will fire if the condition is true, and the second line will always fire.

In fact, our code is equivalent to this:

let test = 3; if (test > 0) { console.log(test); } console.log('+++'); // эта строка оказалась снаружи условия

And we would like this code:

let test = 3; if (test > 0) { console.log(test); console.log('+++'); // эта строка внутри условия }

This is why it is recommended to always place curly braces in order to avoid such errors.
