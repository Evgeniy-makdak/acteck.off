Nuances of variable scope in if-else constructs in JavaScript

The scope of variables has some nuance. Let's look at it with an example. Let's declare a variable outside our conditionreswith one value, and inside the condition we change this value to another:

let res = 1; if (true) { res = 2; } console.log(res); // выведет 2

As you can see, the variablereschanged within the conditions. Everything, however, will change if you also declare a variable inside the conditionresthroughlet:

let res = 1; if (true) { let res = 2; // объявим переменную через let } console.log(res); // выведет 1, а не 2!

The whole point here is that declaring a change throughletinside the condition created a local variableres.

That is, there is one variable inside the conditionres, but outside the conditions are different. You can verify this by outputting the value of the variable to the console inside the condition:

let res = 1; if (true) { let res = 2; console.log(res); // выведет 2 } console.log(res); // выведет 1
