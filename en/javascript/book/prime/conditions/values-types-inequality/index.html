Inequality by value and type in JavaScript

Except the operator!=there is also an operator!==, taking into account the type when comparing. Let's look at the differences between them with examples.

Let using the operator!=two numbers are compared3. This operator compares values ​​to ensure that they are NOT equal. Since our values ​​are exactly equal, the condition will be false:

if (3 != 3) { console.log('+++'); } else { console.log('---'); // сработает это }

Now let one of our values ​​be in quotes. In this case the operator!=will still consider them equal (since the value matches, and the type is not important for this operator) and again the condition will be false:

if ('3' != 3) { console.log('+++'); } else { console.log('---'); // сработает это }

Let's now compare the two numbers3using the operator!==. He will also consider them equal:

if (3 !== 3) { console.log('+++'); } else { console.log('---'); // сработает это }

But if we now put one of the triplets in quotation marks, then the operator!==will consider our triplets unequal, since, although their values ​​​​are the same, they have a different type:

if ('3' !== 3) { console.log('+++'); // сработает это } else { console.log('---'); }
