Automatically adjusting dates in JavaScript

JavaScript has a very interesting and useful feature: if, when creating an objectDateAn incorrect time point was specified - it will be automatically recalculated to the correct one. Let's look at an example.

As you know the dates35January doesn't exist. The maximum possible day in January is31. It turns out that our date35January has4extra days. JavaScript in this case will simply add these extra4days to next month:

let date = new Date(2018, 0, 35); console.log(date); // получится 4 февраля

You can specify not only extra days, but also months. It should be remembered that months start from zero, which means the last correct month is11-ty. If you specify12-th month, then it turns out to be January of the next year:

let date = new Date(2018, 12, 1); // указываем 12-тый месяц console.log(date); // получится 1 января 2019 года

The described adjustment also works to a lesser extent. As you know, the minimum day of the month is the day with the number1. Therefore, if you specify a day with a number0, then you get the last day of the previous month:

let date = new Date(2018, 1, 0); // указываем нулевой день console.log(date); // получится 31 января

You can also specify negative days and months. In this case, you need to remember that the minimum day has a number1, and the minimum month is the number0. This means that minus the first day falls into2day from the end of the previous month, and minus the first month simply falls into the last month of the previous year:

let date = new Date(2018, 1, -1); // указываем -1 день console.log(date); // получится 30 января
let date = new Date(2018, -1, 1); // указываем -1 месяц console.log(date); // получится 1 декабря 2017 года

All of the above works similarly with hours, minutes and seconds:

let date = new Date(2018, 0, 1, 24, 0, 0); // указываем 24-тый час console.log(date); // получится 2 января, 0 часов
let date = new Date(2018, 0, 1, 25, 0, 0); // указываем 25-тый час console.log(date); // получится 2 января, 1 час
