Unbound Function Context in JavaScript

Let's see what happens if we specify in the functionthis, but do not bind it to any element, like this:

function func() { console.log(this); } func();

In this case, the result depends onstrictwhether we have a regime or not. If the regime is not strict, thenthisa link towindow:

function func() { console.log(this); // ссылка на окно window } func();

And if the regime is strict, thenthiswill be storedundefined:

"use strict"; function func() { console.log(this); // undefined } func();

From the above it may seem that in strict modethiswill always beundefined. No matter how it is! If you just withdrawthisoutside the function, it will contain a reference towindowregardless of mode:

"use strict"; console.log(this); // в this ссылка на window

Conclusions

In any function you can writethis, this will not cause any JavaScript errors. But what exactly will lie in thisthisis not defined until the function is called. Moreover, for different function callsthiscan take on different meanings. It all depends on the context in which the function was called.
