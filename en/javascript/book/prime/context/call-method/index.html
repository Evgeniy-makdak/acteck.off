Bind context via call method in JavaScript

So, we figured out how it actually worksthis. Let's now look at methods that force you to specify in what context a function is called (that is, force you to say what the function is equal to)this).

The first method we'll look at is calledcall. Let's see how it works with an example. Let us have an input:

<input id="elem" value="text">

Let's get a link to this input and write it to a variableelem:

let elem = document.querySelector('#elem');

Let's now make a functionfunc, inside which we will outputthis.value:

function func() { console.log(this.value); }

Our function doesn't know what it's referencing yetthis. Now, if we tied her throughaddEventListener, then yes. But we won't do that. Instead, we'll simply call our function, telling it thatthismust be equal to elem.

This is done like this:func.call(elem). This code is equivalent to a simple function callfunclike this:func(), only with the condition thatthisequalselem.

So, the syntax of the call method is:функция.call(объект). Let's put it all together:

let elem = document.querySelector('#elem'); function func() { console.log(this.value); // выведет value инпута } func.call(elem);

Given function:

function func() { console.log(this.value); }

Three inputs are given:

<input id="elem1" value="text1"> <input id="elem2" value="text2"> <input id="elem3" value="text3">

Using the methodcalland functionsfuncdisplayvalueeach of the inputs.
