Losing Context in JavaScript

Let's say we have a function within a function. Let's call the external functionparent, and internal -child:

function parent() { function child() { } }

Fromprevious lessonsyou know that if you set any variable in the outer function, it will be available in the inner one:

function parent() { let str = 'abcde'; function child() { console.log(str); // выведет 'abcde' } child(); // вызываем внутреннюю функцию } parent(); // вызываем внешнюю функцию

There is, however, a caveat: the inner function has access to all the variables of the outer one, but does not have accessthis. That is: if an external function is bound to some DOM element, thenthisit will point to this element, butthisthere will be no internal function!

What then will it indicate?thisinternal function? Answer: it will be equalundefined(in strict mode), since the function is not bound to anything.

Let's check it in practice. Let us be given an input:

<input id="elem" value="text">

Let's bind a function to this inputparent, which will be called when focus is lost with the input:

"use strict"; let elem = document.querySelector('#elem'); elem.addEventListener('blur', parent); // Вызовется по потери фокуса: function parent() { // тут будет какой-то код function child() { // тут будет какой-то код } child(); // вызываем дочернюю функцию }

Let's output the contentsthisto the console in two places: inside a functionparentand inside the functionchild:

"use strict"; let elem = document.querySelector('#elem'); elem.addEventListener('blur', parent); function parent() { console.log(this); // выведет ссылку на наш инпут function child() { console.log(this); // выведет undefined } child(); }

Run this code, lose focus on the input and look at the console - you will see that the firstconsole.logwill display a link to our input in the console, and the second one will simplyundefined. This is the situation whenthisin an unexpected way for us, indicates something that is not what we need, is calledloss of context.

Let us now output in one and the second functionvalueinput. Determine what is printed in the commented lines of code:

"use strict"; let elem = document.querySelector('#elem'); elem.addEventListener('blur', parent); function parent() { console.log(this.value); // что выведет? function child() { console.log(this.value); // что выведет? } child(); }
