Basics of working with context in JavaScript

Let us have some functionfunc, inside which it is usedthis:

function func() { console.log(this.value); }

What does it indicate?thisin this function? But we don't know. And it doesn't know JavaScript. And the function itself does not know. That is, at the time the function is created, what exactly does it point to?this, not defined. And this will be determined only when this function is called.

Let us have some kind of input:

<input id="elem" value="text">

Let's bind our function to this inputfuncso that it is executed when the input loses focus. Now at the time the function is executedthiswill point to our input:

let elem = document.querySelector('#elem'); elem.addEventListener('blur', func); function func() { console.log(this.value); // по потери фокуса выведет 'text' }

But we can have not one input, but several:

<input id="elem1" value="text1"> <input id="elem2" value="text2">

And to each of these inputs we can bind our functionfunc. In this case, for the first elementthisat the time of calling the function will point to it, and for the second - to it.

In practice this means thatthisinside the function depends on which input we lost focus on:

let elem1 = document.querySelector('#elem1'); elem1.addEventListener('blur', func); let elem2 = document.querySelector('#elem2'); elem2.addEventListener('blur', func); function func() { console.log(this.value); // выведет или 'text1', или 'text2' }

In general, this featurethisvery convenient - we create just one function and bind it to any number of inputs. Ifthisdidn’t point to the element in which the event occurred, then we wouldn’t be able to do this - we’d have to create our own function with the same code for each input!
