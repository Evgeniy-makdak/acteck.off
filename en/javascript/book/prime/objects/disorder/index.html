Object Disorder in JavaScript

As you already know, elements in arrays are arranged in a strict order, because the position of each element determines its key. In objects, we assign the keys ourselves, so the order of the elements does not matter. That is, arrays are ordered lists, but objects are not.

For example, consider this object:

let obj = {1: 'a', 2: 'b', 3: 'c'}; console.log(obj[1]); // выведет 'a' console.log(obj[2]); // выведет 'b' console.log(obj[3]); // выведет 'c'

If we rearrange the elements of this object in any order (along with their keys, of course), then nothing will change in the operation of our script:

let obj = {3: 'c', 1: 'a', 2: 'b'}; console.log(obj[1]); // выведет 'a' console.log(obj[2]); // выведет 'b' console.log(obj[3]); // выведет 'c'

Also, numeric keys don't have to have all the values ​​without holes, like an array. We can have arbitrary numbers and it won't cause any problems (likesparsity of arrays). Therefore the following object is correct:

let obj = {7: 'a', 50: 'b', 23: 'c'};
