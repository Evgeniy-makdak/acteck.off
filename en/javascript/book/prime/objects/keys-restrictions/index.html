Restrictions on Object Keys in JavaScript

We can write without quotes not all object keys, but only those that satisfy the following restrictions: they cannot begin with a number and cannot contain a hyphen, a space, and the like.

If a string violates a constraint, it must be quoted. In the following example, some of the keys do not satisfy the conditions and are therefore in quotes:

let obj = {'1key': 'a', 'key-2': 'b', key3: 'c'};

Such keys can only be accessed using square brackets:

console.log(obj['1key']); console.log(obj['key-2']);

Also, accessing such a name through an object property will result in an error:

console.log(obj.1key); console.log(obj.key-2);

But the third key is valid, and we can access it in both ways:

console.log(obj['key3']); console.log(obj.key3);

Given object:

let obj = { '1a': 1, 'b2': 2, 'c-c': 3, 'd 4': 4, 'e5': 5 };

For which keys of a given object are quotes required and for which are they not?

Correct the errors in the following code:

let obj = { '1a': 1, 'b2': 2, 'c-c': 3, 'd 4': 4, 'e5': 5 }; console.log(obj.1a); console.log(obj.b2); console.log(obj.c-c); console.log(obj.d 4); console.log(obj.e5);
