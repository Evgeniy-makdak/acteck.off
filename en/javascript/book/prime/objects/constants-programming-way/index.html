Constant Programming Approach in JavaScript

Recently, a new approach to declaring variables has emerged in JavaScript. The essence of the approach is that instead ofletwe use it everywhereconst, except in cases where we know for sure that the value will be variable.

See example:

const a = 1; const b = 2; const c = a + b; console.log(c);

This approach is very popular and you can find it in various third-party sources.

I am, however, against this fashionable approach. The fact is that constants were invented to store values ​​like the number Pi. That is, for such values ​​that are predefined in your program. Declaring all variables as constants contradicts this idea.

In addition, the behavior of arrays and objects is not very logical - we declare them constants, but we can easily change their properties. What are these constants if they can be changed?

Often declaring objects as constants is used to prevent us from changing the data type - writing a primitive instead of an object. But if we need type control, which is not available by default in JavaScript, then it is better to use its dialectTypeScript.

In general, you can use this approach, since this is the fashion, but I am against it and further in the tutorial, variables will still be declared usinglet, as intended by the authors of the language.

Rewrite the following code using the described approach:

let arr = [1, 2, 3, 4, 5]; let res = arr[1] + arr[2]; console.log(res);
