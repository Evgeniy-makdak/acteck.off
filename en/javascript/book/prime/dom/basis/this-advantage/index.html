Advantage of this in JavaScript

From what has been said before, a particular advantage is not yet obviousthis. After all, inside the handler function our element will already be available - after all, the variableelemwill be global for our functionfunc:

let elem = document.querySelector('#elem'); elem.addEventListener('click', func); function func() { // здесь доступна переменная elem с нашим элементом }

And it’s easy to imagine that the contentsthisand the contents of the variableelemin our case they are equal:

let elem = document.querySelector('#elem'); elem.addEventListener('click', func); function func() { console.log(elem === this); // выведет true }

What is the special advantagethis? It appears when we have several elements, and the same function is attached to each.

Let's look at an example. Let us have3buttons:

<input id="button1" type="submit" value="text1"> <input id="button2" type="submit" value="text2"> <input id="button3" type="submit" value="text3">

Let's get references to them in variables:

let button1 = document.querySelector('#button1'); let button2 = document.querySelector('#button2'); let button3 = document.querySelector('#button3');

Let's bind the same function to these buttons:

button1.addEventListener('click', func); button2.addEventListener('click', func); button3.addEventListener('click', func);

And inside the function we will outputthis.value:

function func() { console.log(this.value); }

It turns out that we have three buttons. Clicking each button will call a functionfunc. Moreover, with each clickthiswill contain a link to the button in which the event occurred.

That is, each click will output to the consolevaluethe button on which the click occurred, but it will be done by the same functionfunc! This is the advantage of usingthis.
