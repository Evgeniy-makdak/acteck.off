Advantages and disadvantages of chaining in JavaScript

Despite the fact that chains shorten the code, in most cases introducing a variable is still more convenient. Compare two examples - now I have entered a variableelemand can record any number of attributes, whilequerySelectorcalled only once:

let elem = document.querySelector('#elem'); elem.value = 'www'; elem.type = 'submit';

And now I don't introduce a new variable and so I have to callquerySelectortwice:

document.querySelector('#elem').value = 'www'; document.querySelector('#elem').type = 'submit';

In my opinion, this code has become more complex, although it takes one less line. Also, if I want to change the valueidWith'elem'for something else, I will have to do it in many places, which is not very convenient.

There is another problem - the load on the browser. Searching for elements on the page, which is what the method doesquerySelector, is a rather slow operation (and in general any work with page elements is a slow operation- remember this).

In our case, if we use each timequerySelector, then the browser will process the HTML page each time and look for an element with the givenidseveral times (no matter what)idare the same - the browser will perform all the actions several times), performing useless operations that can slow down the browser.

If we use the variableelem- no search on the page occurs (the element has already been found and the link to it is in the variable).
