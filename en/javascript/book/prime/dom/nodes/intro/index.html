Working with Nodes in JavaScript

As you know, tags can contain other tags or, in JavaScript terms, DOM elements can contain other DOM elements. In addition, however, they may contain comments and plain text. Comments, texts and DOM elements are united by one name -nodes.

You are already familiar with the propertiesfirstElementChild, lastElementChild, nextElementSibling, previousElementSibling. These properties work specifically on DOM elements, ignoring other nodes. As a rule, this is exactly what we need.

However, there are also propertiesfirstChild, lastChild, nextSibling, previousSibling. These properties work in a similar way, however, taking into account all nodes. Let's see the difference between these properties with an example. Let us have a div containing three nodes:

<div id="elem"><!--com-->text<span>tag</span></div>

Let's get a link to this div into a variable:

let elem = document.querySelector('#elem');

Now let's see what the properties containfirstChildAndfirstElementChild:

console.log(elem.firstChild); // комментарий console.log(elem.firstElementChild); // тег span
