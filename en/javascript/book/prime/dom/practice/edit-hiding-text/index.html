Hiding text when editing an element in JavaScript

Let's now make sure that when editing, the input appears in the paragraph itself - instead of the text of this paragraph. Let then, when editing is completed, the input will be removed, and the text of the paragraph will appear in its place again.

Let's start implementation.

First, let's make sure that when you click on a paragraph, an input with the text of this paragraph is added to the end:

let elem = document.querySelector('#elem'); elem.addEventListener('click', function() { let input = document.createElement('input'); input.value = elem.textContent; elem.appendChild(input); });

Our code, however, is very imperfect - every time you click on a paragraph, a new input will be added to it.

Moreover, if the first input contains the text of a paragraph, then the second input will already contain the text of the paragraph along with the first input, and the third input will already contain the text of the paragraph along with two inputs, and so on.

Please also note that clicking on the added input will be regarded as a click on the paragraph: the fact is that the input is located in the paragraph, and clicking on the input will simply pop up higher to this paragraph.

This will lead to the fact that after the first input appears, when we try to click on it in order to start editing, we will automatically click on the paragraph with all the ensuing consequences.

So, the problem is outlined. Let's fix it now.

To do this, simply when the input appears, we will unbind the click handler from the paragraph. In this case, only the first click on the paragraph will result in the input appearing, and other clicks made after the input appears will be ignored.

We implement:

let elem = document.querySelector('#elem'); elem.addEventListener('click', function func() { let input = document.createElement('input'); input.value = elem.textContent; elem.appendChild(input); elem.removeEventListener('click', func); // отвяжем событие });

Let's now make sure that when the input appears, the text of the paragraph itself disappears. To do this, before inserting the inputtextContentassign the paragraph to an empty line:

let elem = document.querySelector('#elem'); elem.addEventListener('click', function func() { let input = document.createElement('input'); input.value = elem.textContent; // сначала записываем текст абзаца в инпут elem.textContent = ''; // затем убираем текст абзаца elem.appendChild(input); // затем вставляем инпут elem.removeEventListener('click', func); });

Let's now make sure that when an input loses focus, the text of that input is written into a paragraph:

let elem = document.querySelector('#elem'); elem.addEventListener('click', function func() { let input = document.createElement('input'); input.value = elem.textContent; elem.textContent = ''; elem.appendChild(input); input.addEventListener('blur', function() { elem.textContent = this.value; }); elem.removeEventListener('click', func); });

Please note that we don't need to delete the input - it deletes itself when it writes its text into a paragraph: since the input is part of the text of the paragraph, writing some text into this paragraph simply deletes our input, that's all.

We have one more problem, however: the paragraph text will only be edited the first time. After the first edit, clicking on the paragraph text again will do nothing.

The fact is that at the moment the input appears, we untied the event from the paragraph for the reasons described above. Now we need to bind the event back when editing is completed.

Let's do this:

let elem = document.querySelector('#elem'); elem.addEventListener('click', function func() { let input = document.createElement('input'); input.value = elem.textContent; elem.textContent = ''; elem.appendChild(input); input.addEventListener('blur', function() { elem.textContent = this.value; elem.addEventListener('click', func); // повесим событие обратно }); elem.removeEventListener('click', func); });
