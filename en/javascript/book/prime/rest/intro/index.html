rest operator in JavaScript

Let's start this lesson with some explanation. All the previous lessons in this section, I did not tell you the whole truth about the operator...

In fact, this operator, depending on its place in the code, can be called either the spread operator or the rest operator.

As you already know, when the spread operator appears before an array or string, it splits them into individual values. The rest operator, although it has the same syntax in the form..., but intended for something else.

Now we will figure out why. Let us have a function that takes two parameters:

function func(a, b) { console.log(a); console.log(b); }

You should already know that if you pass unnecessary parameters to a function, this will not lead to an error:

func(1, 2, 3, 4, 5);

Operatorrestallows you to get all the extra parameters as an array. To do this, another parameter is added to the function (necessarily the last one), preceded by3points:

function func(a, b, ...rest) { }

These three dots are the rest operator, and the variable name can have any name, but it is usually also customary to call itrest.

Let's call our function with an excessive number of parameters and see what ends up in the parametersaAndb, and what - the restrest:

func(1, 2, 3, 4, 5); function func(a, b, ...rest) { console.log(a); // выведет 1 console.log(b); // выведет 2 console.log(rest); // выведет [3, 4, 5] }

If there are no extra parameters, then into the variablerestan empty array will be written:

func(1, 2); function func(a, b, ...rest) { console.log(a); // выведет 1 console.log(b); // выведет 2 console.log(rest); // выведет [] }

A function may have no parameters at all, but you can still addrest. In this case, the function can be called with any number of parameters and all of them will end up in the arrayrest:

function func(...rest) { console.log(rest); }
