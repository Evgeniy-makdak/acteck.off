Prefix and postfix type in JavaScript

In fact, increment and decrement operations can be written in two ways. Let's look at these methods with an example++, and for--everything will be the same.

In the first method, the operation is written after the variable name, like this:a++, and in the second method - before the variable name, like this:++a.

The first form is calledpostfix, and the second -prefix. Let's see in what cases the difference between these two forms appears.

Let us have a codealert(a++)and codealert(++a).

In casealert(a++)the variable will first be output and then increased by one, and in the casealert(++a)the variable will first increase by one and then be output.

In the following example the firstalertwill output0, since the output to the screen will work first, and only then the variable will be increased:

let num = 0; alert(num++); // выведет 0 alert(num); // выведет 1

And now the variable will first increase, and only then will it be displayed on the screen:

let num = 0; alert(++num); // выведет 1 - переменная увеличилась сразу

This behavior is true not only for display, but also for assignment:

let num1 = 0; let num2 = num1++; // в переменную num2 запишется 0 alert(num2); // выведет 0 alert(num1); // выведет 1 - переменная num1 поменялась после записи в num2

Now let’s change the postfix form to the prefix form:

let num1 = 0; let num2 = ++num1; // в переменную num2 запишется 1 alert(num2); // выведет 1

If our operation is performed on a separate line, then there is no difference between the prefix and postfix forms:

let num = 0; ++num; num++; alert(num); // выведет 2
