Imprecise calculations in JavaScript

Fractions are stored in computers in such a way that they can often be represented inaccurately. In this case, you may be in for some surprise when a trivial operation produces a strange result. Example:

let a = 0.1 + 0.2; alert(a); // выведет 0.30000000000000004

To combat this behavior, you can use a special methodtoFixed, which rounds to a specified sign in the fractional part. Let's round up our result:

let a = 0.1 + 0.2; alert(a.toFixed(2)); // выведет '0.30'

MethodtoFixedreturns the result as a string. You can convert its result to a number:

let a = 0.1 + 0.2; alert(+a.toFixed(2)); // выведет 0.3

Check what will be the result of the following operation:

alert(0.1 * 0.2);

Check what will be the result of the following operation:

alert(0.3 - 0.1);
