Browser caching of JavaScript files

The browser can cache connected JavaScript files. This means that it stores them to improve the loading speed of the site.

In practice, this leads to the fact that at some point when editing code, the browser will use the previous version of the code, rather than your current one.

To combat this behavior, you can clear your browser cache, or use a clever trick. Its essence lies in the fact that when connecting a script, after the file name we put a question, an equal sign and the version number of your script. When the browser caches the file, you will simply need to increase the number by one.

See example:

<!DOCTYPE html> <html> <head> <meta charset="utf-8"> <title></title> <script src="script.js?v=1"></script> </head> <body> </body> </html>

Correct your file connection so that the file name is followed by its version number.
