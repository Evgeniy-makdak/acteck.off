Variable Conflicts in JavaScript

Let's say we have an HTML pageindex.html, on which in the tagscriptyou create a variablestrand display it on the screen:

<html> <head> <script> let str = 'script text'; alert(str); // выведет 'script text' </script> </head> <body> ... </body> </html>

Let us also have a filescript.js, which also sets the variablestr:

let str = 'file text';

Let now our filescript.jsconnects to the pageindex.htmlas follows:

<html> <head> <script> let str = 'script text'; </script> <script src="script.js"></script> <script> alert(str); </script> </head> <body> ... </body> </html>

Since the variablestris also in the fileindex.html, and in the filescript.js, then there will be a conflict in which the variable that is written below, that is, the variable from the file, will winscript.js. That is, our code will output'file text', not'script text', as we expect.

The problem is actually very serious. On a real website, you will most often have several files with your scripts, in addition, you will connect some third-party plugins. In this case, variables and functions in one file may conflict with variables and functions in another file.
