The problem with optional parentheses in loops in JavaScript

Although it is possible to omit curly braces in loops, I highly recommend not doing so, as such code often leads to errors.

Let's look at an example. Let's say we have the following code:

for (let i = 0; i <= 9; i++) console.log(i); // выведет числа от 0 до 9

I’ll make a small correction to the above code (find which one) - and it will stop working:

for (let i = 0; i <= 9; i++); console.log(i); // выведет ошибку

So what did I fix?

The problem arose because I placed a semicolon after the parenthesis)from the cycle. In this case, the so-calledcycle without body: it will simply scroll inside, and the next line will no longer apply to it. Therefore, to avoid problems, I always recommend putting curly braces in loops.
