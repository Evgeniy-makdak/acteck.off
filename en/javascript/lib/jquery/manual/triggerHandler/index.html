triggerHandler method

MethodtriggerHandlerallows you to run all event handlers bound to an element. It works similar to the methodtrigger, but there are differences between them: The event handler will fire, but the standard event will not be called. For exampletriggerHandler('submit'), when triggered, will not causesubmit. MethodtriggerHandleronly affects the first element in the set. Events triggeredtriggerHandlerdo not float down the hierarchy of DOM elements.triggerHandlerdoes not return a jQuery object, it returns the value received by the last handler called. If no handlers are run, then returnsundefined.

Syntax

As the first parameter we can pass the event type as a string, the second optional is an array or object containing additional parameters to be passed to the handler:

$(селектор).triggerHandler(тип события, [дополнительные параметры]);

You can pass the event as an object as the first parameterjQuery.Event:

$(селектор).trigger(событие, [дополнительные параметры]);

Example

Let's make it so that when you click on the button with#test1our input will receive focus and be displayed in the console'Focused', in other words, both the handler function and the focus event will work. When you press the second button with#test2Only the handler function will work, but the input will not receive focus:

<button id="test1">button1</button> <button id="test2">button2</button> <input type="text"> $('#test1').click(function() { $('input').trigger('focus'); }); $('#test2').click(function() { $('input').triggerHandler('focus'); }); $('input').focus(function() { console.log('Focused') });
