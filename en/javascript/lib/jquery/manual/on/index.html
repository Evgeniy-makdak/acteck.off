on method

Methodonallows you to add an event handler to an element. To remove a handler, you can use the methodoffso that the event fires only once, and then the handler deletes itself - use the methodone. The elements to which we bind the handler must exist at the time of the callon.

Syntax

So we add an event handler to the element, one or more events separated by spaces are passed as the first parameter as a string, the second parameter is an additional filtering selector of descendants inside the element, the third is additional data that is passed to the handler in the propertyevent.datawhen the event is triggered. The second and third parameters are optional. Fourth, we pass the handler function, to which the event object and optional additional parameters are passed. If instead of a handler function we passfalse, then the function will simply returnfalse:

$(селектор).on(события, [селектор], [данные], функция-обработчик(объект события, [дополнительные параметры]));

You can use the methodondifferently, then the first parameter is passed a JavaScript object, where the keys are the event type, and the values ​​are the handler functions called for the events:

$(селектор).on({'тип события': handler}, [селектор], [данные]);

If we do not pass an additional selector, then the event fires on the element to which we attach the handler, otherwise on the descendant element that matches this selector (delegated events). The same event handler can be attached to an element multiple times.

Example

Let's inalert, display the text of the paragraph with#testwhen you click on it, clicking on other paragraphs will lead to nothing:

<p>text1</p> <p id="test">text2</p> <p>text3</p> $('#test').on('click', function() { alert( $(this).text() ); });

Example

Let's click on a paragraph to display the data that we passed to the methodon. Let's use the handler functiontestFunc, which we created:

<p>click</p> function testFunc(event) { alert(event.data.text); } $('p').on('click', {text: 'aaa'}, testFunc);
