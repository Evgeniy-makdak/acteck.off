Method one

Methodoneallows you to add an event handler to an element. After the event has been processed once, the handler will be automatically deleted.

Syntax

This is how we attach an event handler to an element, one or more events separated by spaces are passed as the first parameter as a string, and as the second parameter we pass data in the form of a JavaScript object, which is passed to the handler in the propertyevent.datawhen the event is triggered. The second parameter is optional. To the third we pass the handler function, to which the event object is passed, or we passfalse:

$(селектор).one(события, [данные], функция-обработчик(объект события));

You can pass an additional filtering selector of descendants inside the element as the second parameter, and additional data that is passed to the handler in the property as the third parameter.event.datawhen the event is triggered. The second and third parameters are optional. Fourth, we pass the handler function, to which the event object and optional additional parameters are passed. If instead of a handler function we passfalse, then the function will simply returnfalse:

$(селектор).one(события, [селектор], [данные], функция-обработчик(объект события));

You can use the methodonedifferently, then the first parameter is passed a JavaScript object, where the keys are the event type, and the values ​​are the handler function that we added to the element:

$(селектор).one({'тип события': handler}, [селектор], [данные]);

If we do not pass an additional selector, then the event fires on the element to which we attach the handler, otherwise on the descendant element that matches this selector (delegated events).

Example

Let's click on a paragraph to display the data that we passed to the methodone. Nothing will happen on subsequent clicks. Let's use the handler functiontestFunc, which we created:

<p>click</p> function testFunc(event) { alert(event.data.text); } $('p').one('click', {text: 'aaa' }, testFunc);
