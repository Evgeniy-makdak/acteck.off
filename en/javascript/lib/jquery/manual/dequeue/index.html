Dequeue method

Methoddequeueallows us to execute the next function from the function queue. When a method is called, the function is removed from the queue and then executed. A function must call this method so that the function chain can continue. See also methodqueue, which allows us to work with a queue of functions bound to an element.

Syntax

This way we can continue the work of the next functions in the queue. You can pass the queue name as a string as an optional parameter. If you do not pass any parameters, the method will work with the queuefx:

.dequeue([имя очереди]);

Example

In this example, using the methodqueue, we'll add a custom function that will recolor the square green by adding the classnewcolor. As you can see, it is then called heredequeueto remove the function from the queue and allow the next one to execute:

<button id="animate">start</button> <div></div> div { position: absolute; margin: 3px; width: 50px; height: 50px; left: 0px; top: 30px; background: red; display: none; } div.newcolor { background: green; } $('#animate').click(function() { $('div') .show('slow') .animate({left: '+=200'}, 2000) .queue(function() { $(this).addClass('newcolor').dequeue(); }) .animate({left: '-=200'}, 500) .queue(function() { $(this).removeClass('newcolor').dequeue(); }) .slideUp(); });
