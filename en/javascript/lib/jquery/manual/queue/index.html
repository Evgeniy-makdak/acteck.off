queue method

Methodqueueallows us to work with a queue of functions bound to an element. See also methodclearQueue, which allows us to remove from the queue all functions that have not been run.

Syntax

This is how we show the queue of executing functions bound to the element. You can pass the queue name as a string as an optional parameter (by defaultfx- standard effects queue):

.queue([имя очереди]);

We can manipulate the queue. Executes once for each element. To do this, you can pass as the second parameter an array of functions that will replace the contents of the current queue:

.queue([имя очереди], новая очередь );

Or you can use the second parameter to pass a new callback function to be added to the queue. This function in turn receives another function as a parameter. This allows you to automatically remove the next element from the queue and move the queue:

.queue([имя очереди], callback-функция(next) { next(); });

Each element can have one or more queues. Many applications use only one (fx). Queues allow you to carry out a sequence of actions with an element asynchronously, without interrupting program execution. Callqueuec callback function allows us to put a new function at the end of the queue. The callback function is executed once for each element in the set. When adding a function usingqueue, we need to make sure that the method is then calleddequeue, so that the next function in the chain works.

Example

Let's add a custom function. First, after clicking on#animate, we will show a red square that will move to the right for2с, then usingqueuewe will add a custom function that will recolor the square green by adding the classnewcolor. As you can see, it is then called heredequeueto remove a function from the queue. Then our square will move to the left for half a second and recolor back to red - using the second custom function we remove the classnewcolor. At the end of the animation, we collapse our square:

<button id="animate">start</button> <div></div> div { position: absolute; margin: 3px; width: 50px; height: 50px; left: 0px; top: 30px; background: red; display: none; } div.newcolor { background: green; } $('#animate').click(function() { $('div') .show('slow') .animate({left: '+=200'}, 2000) .queue(function() { $(this).addClass('newcolor').dequeue(); }) .animate({left: '-=200'}, 500) .queue(function() { $(this).removeClass('newcolor').dequeue(); }) .slideUp(); });

Example

Let's add a queue as an array to remove the previous one. When you press the button#startwe will see the animation from the previous example. When you press the button#stopwe will stop the animation by adding an empty array. When you press again#startthe animation will start again:

<button id="start">start</button> <button id="stop">stop</button> <div></div> div { position: absolute; margin: 3px; width: 50px; height: 50px; left: 0px; top: 30px; background: red; display: none; } div.newcolor { background: green; } $('#start').click(function() { $('div') .show('slow') .animate({left: '+=200'}, 5000) .queue(function() { $(this).addClass('newcolor').dequeue(); }) .animate({left: '-=200'}, 1500) .queue(function() { $(this).removeClass('newcolor').dequeue(); }) .slideUp(); }); $('#stop').click(function() { $('div').queue('fx', []).stop(); });
