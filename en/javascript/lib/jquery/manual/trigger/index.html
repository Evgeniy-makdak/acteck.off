Trigger method

Methodtriggerallows you to run all event handlers bound to an element for events of a given type. It calls the handler functions in the order in which they would be called by the user. If you want to call an event handler, but not execute a standard event, then use the methodtriggerHandler.

Syntax

As the first parameter we can pass the event type as a string, the second (optional) is an array or object containing additional parameters to be passed to the handler:

$(селектор).trigger(тип события, [дополнительные параметры]);

You can pass the event as an object as the first parameterjQuery.Event:

$(селектор).trigger(событие, [дополнительные параметры]);

There is a difference between passing additional parameters to a methodtriggerand data into the methodon. In the first case, the data must be determined during the execution of the event, and in the second, already at the time of binding the handler function.

Example

When you press the button#firstthe value will increase only in the first span, when the button is pressed#second- in both, while simulating the user’s first click. Functionincreasewill increase by1the number passed to it as a parameter. Using the methodtriggerwe also force the click event to fire in the first span, although we click on the button#second:

<button id="first">button1</button> <button id="second">button2</button> <div><span>0</span></div> <div><span>0</span></div> button { margin: 10px; } div { margin-left: 10px; font-weight: bold; } span { color: green; } $('#first').click(function() { increase($('span').first()); }); $('#second').click(function() { $('#first').trigger('click'); increase($('span').last()); }); function increase(str) { let num = parseInt(str.text(), 10); num.text(num + 1); }
