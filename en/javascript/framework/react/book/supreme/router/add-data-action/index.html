Adding data with action method in React Router

In this lesson we will get acquainted with the methodactionroute object. This method is called when the application sendsHTTP requesttype POST, PUT, PATCH or DELETE (except GET) to your route.

First, let's import the function for creating a product that we wrote in the last lesson into ourroot.jsx:

import { createProduct } from '../forStorage';

Now we need to create a functionaction, which React Router will call when you click on the add product button, place it after the functionloader:

export async function action() { const product = await createProduct(); return { product }; }

Let's openmain.jsxand import action:

import Root, { loader as rootLoader, action as rootAction } from './routes/root';

And set it as the value of the methodactionfor a route object. ComponentFormwill prevent the browser from sending requests to the server when the button is pressed, but will call the methodactionour route. This is how client-side routing works interestingly with React Router:

const router = createBrowserRouter([ { path: '/', element: <Root />, errorElement: <ErrorPage404 />, loader: rootLoader, action: rootAction, children: [ { path: 'products/:productId', element: <Product />, }, ], }, ]);

Let's launch our application. Let's open the tab'Приложение'(Application for Chrome) on the developer panel, among the different types of storage, click on indexedDB, here we are interested in localforage. Now, clicking on the add products button in our application and updating the localforage storage after the click, we see how in keyvaluepairs into an arrayproductsobjects with differentid. Victory!!! This means that records are being created in our storage! Of course, the list in our application is also updated. Hint: do not forget to clear the saved data of this page later on the tab'Storage'in Application.
