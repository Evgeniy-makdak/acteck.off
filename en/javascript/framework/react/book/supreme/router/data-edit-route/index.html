Creating a Data Editing Route in React Router

In the last lesson we created a form for editing in a separate React componentEditProduct.

Let's now write a loader function for the new route just after the import and before the functionEditProduct. It will be the same as inproduct.jsx. URL parameters will be passed to the function and then we will receive the product by itsid:

export async function loader({ params }) { const product = await getProduct(params.productId); return { product }; }

We importgetProductfromforStorage.js:

import { getProduct } from '../forStorage';

And now we will open ourmain.jsxand import the component we createdEditProductand bootloader:

import EditProduct, { loader as editProductLoader } from './routes/edit';

Then add it to the arraychildrenroute object to edit immediately after the route for the product:

children: [ { path: 'products/:productId', element: <Product />, loader: productLoader, }, { path: 'products/:productId/edit', element: <EditProduct />, loader: editProductLoader, }, ],

Now let’s launch the application, create a product, click on it, and then on the edit button and see our form.

The only thing we still need to do is to ensure that the current data is loaded in the form fields before editing. To do this we import intoedit.jsxhookuseLoaderData:

import { Form, useLoaderData } from 'react-router-dom';

And use it to receive data from the loader in a functionEditProduct. Let's add it before the commandreturn:

const { product } = useLoaderData();

Now let’s add an input attribute to each tagdefaultValuewith corresponding values ​​for each input. For example, the first input will now look like this:

<input placeholder="name" type="text" name="name" defaultValue={product.name} />

For the two remaining inputs the values ​​will be respectively'product.cost'And'product.amount'. However, we don’t have any data in the form yet.
