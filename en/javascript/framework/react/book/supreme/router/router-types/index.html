Routers in React Router

First of all, to create routing you need to start withrouter (Router). This is the top-level component, thanks to which all other components and hooks work.

There are several types of routers. Most web applications useBrowserRouter. If for some reason you cannot use the full URL, thenHashRouter. Very convenient for testing componentsMemoryRouter. RouterNativeRouter, as the name suggests, is used for native development. SelectStaticRouter, if you want to use a data router for rendering on your server, for example onNodeJS.

To use one of these routers you just need to wrap the code in the selected component like this, for example:HashRouterThis:

function App() { return ( <HashRouter> /* Ваш код */ </HashRouter> ); }

True, this syntax is already considered a little outdated. In order for us to have access to new APIs in the future (for example, some hooks and various useful properties), which are constantly updated and added, it is recommended to use a more modern syntax for creating routers, that is, instead of the above components, we need to usecreateBrowserRouter, createMemoryRouter, createHashRouterAndcreateStaticRouter.
