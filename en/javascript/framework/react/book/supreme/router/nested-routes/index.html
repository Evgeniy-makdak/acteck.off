Nested routes in React Router

In this lesson we will get acquainted withnested routes. Launch the application we made in the last lesson. By clicking on the links, we are taken to a separate page with the product. Suppose we are uncomfortable with this, and we would like the list of products and the product page to be displayed side by side on the same screen for greater clarity. Let's do this.

To do this, let's open the filemain.jsxand simply make our route for the product page a child route of the root route or, in other words, “nested” it into the root route. To do this, we will use one more property of the route objectchildren:

const router = createBrowserRouter([ { path: '/', element: <Root />, errorElement: <ErrorPage404 />, children: [ { path: 'products/:productId', element: <Product />, }, ], }, ]);

We click on the links again, but we don’t see anything to the right of the product list. This is because we didn't tell the root route where we want its child routes to appear.

Just for rendering elements of child routes in the parent one, the component will help usOutlet. Let's import it from the library into a file for the root route:

import { Outlet } from 'react-router-dom';

Then we wrap it in a div#productand add it to the layout for the root route at the end of the layout after the closing tagnav, and we’ll wrap the whole structure in one more div#main:

return ( <div id="main"> <nav> <a href={`/products/1`}>Product1</a> <a href={`/products/2`}>Product2</a> </nav> <div id="product"> <Outlet /> </div> </div> );

By clicking on the links, we now see both a list of products and a product page.

Let's add some styles to ourindex.css:

h1, h2 { margin: 0; } div#main { display: flex; margin: 20px; } div#product { margin-left: 40px; }

Take the application you created in the assignment for the last lesson. Using the lesson materials, make the route for the student’s page nested in the root one.

Now make sure that when you click on the links, both the list of students and the student’s page are displayed on the screen.
