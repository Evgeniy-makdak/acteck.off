Client Side Routing in React Router

In this lesson, we'll look at an important component of Single Page Applications (SPA) - client-side routing. This routing allows us to update the URL in the browser without additional document requests from the server.

Launch the application we created in the last lesson. Open the developer panel and in it the tab'Сеть'(Network for Chrome). After each click on our links, we see a request for document. That is, every time we ask the server to load us document.

Let's now take advantage of React Router to get rid of such requests. For this we will openroot.jsxour application and replace the tagamarkings per componentLink. First, let's add its import:

import { Outlet, Link } from 'react-router-dom';

Now let's replace the tagsawith attributehref. Instead of a piece of code:

<a href={`/products/1`}>Product1</a> <a href={`/products/2`}>Product2</a>

We will now have the following code:

<Link to={`/products/1`}>Product1</Link> <Link to={`/products/2`}>Product2</Link>

Open the tab again'Сеть'on the developer panel, set the root path in the address bar and refresh the page. Now we click on the links and see that the document was requested from us only once during the initial download and with subsequent clicks there is no longer a request to the server.

Take the application you created in the assignments for previous lessons. Using the lesson materials, implement client-side routing in it by modifying your links as described in the lesson.
