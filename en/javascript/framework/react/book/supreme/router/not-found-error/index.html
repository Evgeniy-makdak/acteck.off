Handling Not Found error in React Router

When creating an application, you need to pay attention to the appearance of various errors during its further use. In this lesson we will learn how to work with an error'Not Found'. This error can occur, for example, if the user clicks on a link leading to a non-existent page.

First, let's open our application, which we did in previous lessons, and replace the layout of the componentRoot. Now, when we launch the application, we will display a list of two products:

function Root() { return ( <nav> <a href={`/products/1`}>Product1</a> <a href={`/products/2`}>Product2</a> </nav> ); } export default Root;

Let's add some styles toindex.css:

body { font-size: 18px; line-height: 1.5; } nav { display: flex; flex-direction: column; } a { text-decoration: none; color: blue; }

Now click on the links. Since we don't have page layouts for our products, the links will only take us to an error screen'404 Not Found'and a reproach that we could come up with something more beautiful and interesting ourselves than what React Router displayed to us by default.

Let's make our own page that will be displayed if a 404 error occurs. To do this, in the foldersrclet's create a fileerror-page-404.jsx.

Inside a React componentErrorPage404we will use a hookuseRouteErrorto catch the error that we will print to the console:

import { useRouteError } from 'react-router-dom'; function ErrorPage404() { const error = useRouteError(); console.error(error); } export default ErrorPage404;

Next in the component layoutErrorPage404we will display a couple of headers indicating the error and display the following properties of the objecterror, HowstatusTextAnddata(objecterrorand you can also see its properties if you open the console and start clicking on our links in the application).

return ( <div> <h1>Hi! It is an Error Page</h1> <h2>404 Not Found Error</h2> <p> <i>{error.statusText}</i> </p> <p> <i>{error.data}</i> </p> </div> );

Be sure to remember to open the filemain.jsxand add another import line withErrorPage404:

import ErrorPage404 from './error-page-404';

And also add one more propertyerrorElementto objectRouter- this element is displayed when an error occurs on this route. The value of the error element will be our componentErrorPage404:

const router = createBrowserRouter([ { path: '/', element: <Root />, errorElement: <ErrorPage404 />, }, ]);
