Retrieving data from storage in React Router

You see that our application is static for now. In this and subsequent lessons we will begin to revive it. In the next lessons we will be able to add products to the list using React Router.

In this lesson, we will first look at retrieving data from the storage. To store data locally, we will use localForage storage (you can read about its use and benefits indocumentation) - analogueLocalStorage, which is supported by all modern browsers. In fact, it is a combination of several storage technologies. With relatively easy use, it allows you to store large amounts of data, of different types, including pictures. Let's install it in our project. Type in the terminal:

npm i localforage

Let's restart our application. In a foldersrcwe will create a fileforStorage.js, which will contain functions for working with storage.

So, let's start with the function of loading products from storage. Add toforStorage.jsimport the library and write a functiongetProductsto receive products:

import localforage from 'localforage'; export async function getProducts() { await someNetwork(); let products = await localforage.getItem('products'); if (!products) products = []; return products; }

Below after the function code we will also add the code forsomeNetwork, simulating network delays (up to 0.7 seconds) for normal operation of functions. If our product is “cached”, then there will be no delay in the functions for working with the storage; if not, then there will be a delay; you will see later how this works:

let someCache = {}; async function someNetwork(key) { if (!key) { someCache = {}; } if (someCache[key]) { return; } someCache[key] = true; return new Promise((res) => { setTimeout(res, Math.random() * 700); }); }
