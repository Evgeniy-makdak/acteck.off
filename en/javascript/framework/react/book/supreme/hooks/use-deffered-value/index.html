UseDeferredValue performance optimization hook in React

Performance optimization hookuseDeferredValue, as well asuseTransition, helps speed up the GUI, but does it a little differently.

Take a look at the pastlessonand see how it is withuseTransitionwe wrapped the update functionsetFilterTerm. So, the state update code may not always be available. It may be hidden, for example, in a library that you have included. In this case we can use the hookuseDeferredValue. Then we need to wrap either the state itself or the result calculated using this state, thereby reducing its priority.

Let's analyze the following piece of child component codeProductList:

import {useDeferredValue} from 'react' function ProductList({ products }) { const deferredProducts = useDeferredValue(products); return ( <ul> {deferredProducts.map((product) => ( <li>{product}</li> ))} </ul> ); }

Note that here we have wrapped inuseDeferredValuethe variable itselfproducts, which we took from the component props. We then wrote the result indeferredProducts, with which we will work further:

const deferredProducts = useDeferredValue(products);

Now, if in the piece of code of the main component that we looked at in the last lesson, remove the line:

const [isPending, startTransition] = useTransition();

And a line usingisPending:

{isPending && <p>updating ...</p>}

And also remove the wrapper from the functionsetFilterTerm, then we will get the same result as in the last lesson:

function updateFilterHandler(event) { setFilterTerm(event.target.value); }
