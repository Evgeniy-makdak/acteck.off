Context in React

Before we move on to the next hook, we need to look at the conceptcontextin React.

Typically we would pass data from parent components to children usingprops. This method is good if you have few components and you don’t need to pass props from the parent to the component you need through its children.2-3once.

This method can also become extremely inconvenient and lead to various problems if these props are not used at all in intermediate components (this problem is calledprop drilling), or you need to pass the same data to many components.

In these cases it can help uscontext, which allows you to make the data of a parent component available to any child, regardless of its location in the component tree, without passing it through props. Accordingly, only those components that need it will accept data.

In order to use a context, you need to create it. First, let's create a fileMyContext.jsand import the function into itcreateContext:

import { createContext } from 'react';

Now let's create a context object and write it to a variableMyContext, which we won’t forget to export. In our case, we installed increateContextinitial valuenull, since it’s not important to us (we could have just left empty parentheses). The given default value will appear when reading the context if no others are found. This value can be of any type:

export const MyContext = createContext(null);

In the next tutorial we will create an application from several components located in separate files. If all the components were in one file, then we would not create a separate file to create the context and would not export it.

Let's now importMyContext.jsto an empty componentApp- to the component from which we are going to transfer data:

import { MyContext } from './MyContext.js';
