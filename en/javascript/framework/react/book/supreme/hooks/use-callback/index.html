UseCallback performance optimization hook in React

In this tutorial we will look at the following performance optimization hookuseCallback.

HookuseCallbacksimilar to APIuseMemo, the difference is that the first caches the value between the moments the screen is redrawn, and the second -callback. This allows us to avoid restarting resource-intensive functions when not required and can be used whenfunction transferto child components.

Let's take a closer look with an example. First, let's create a componentAppand let's create a state in itnum:

const [num, setNum] = useState(0);

Let us have a button that, when clicked,numincreases by1, and the paragraph in which we will display the valuenum:

return ( <div> <button onClick={() => setNum(num + 1)}>click</button> <p>clicks: {num}</p> </div> );

Now, let's assume that we haveAppanother list with elements is displayed, which we will supplement by pressing another button. To store the elements of this list we will create a stateitems:

const [items, setItems] = useState([]);

And then we'll write a functionaddItemto add them:

function addItem() { setItems([...items, 'new item']); }

Now let's write the code to display the list items and put it in a child componentItems, which in the form of props will receive an array of elements and a function for adding them. Let's not forget to add output to the console so we can see when ourItemswill be redrawn:

function Items({ items, addItem }) { const result = items.map((item, index) => { return <p key={index}>{item}</p>; }); console.log('Items render'); return ( <div> <h3>Our items</h3> {result} <button onClick={addItem}>add item</button> </div> ); } export default Items;

Let's postItemsat the end of the componentAppand we will pass it an arrayitemsand a function to add elementsaddItem:

return ( <> <div> <button onClick={() => setNum(num + 1)}>click</button> <p>clicks: {num}</p> <br /> </div> <Items items={items} addItem={addItem} /> </> );

Now let’s press the buttons and make sure thatnumgrows and new elements are added to the list. And when we open the console, we will see that our list is redrawn every time, even if we click on the button that increasesnum.

If we have a small list, then everything is fine, but what if it is supposed to be voluminous and there is a lot more in there? It doesn’t matter, you say, because in the last lesson we looked at the APImemo, just to avoid unnecessary redrawing of the component.

So let's wrap our componentItemsVmemoand that's all. By the way, this can be done directly during export.Items:

export default memo(Items);

Let's not forget to importmemo:

import { memo } from 'react';

Now let’s open the console and press the buttons. All efforts are in vain! We memoized the component, but when the button is clicked'click'componentItemsit still gets redrawn every time.

The thing is that when the parent component is redrawn, its functions are recreated anew - this also applies to our functionaddItem, which we transfer toItems.

It is at this moment that the hook will help ususeCallback. Let's apply it. First, let's import it intoApp:

import { useCallback } from 'react';

Then we'll redo a simple function declarationaddItemV Function Expression, we indicate as the first parameter foruseCallbackour function as a callback. The second parameter in square brackets will indicate the dependencies - all reactive variables participating in the function, in our case this is an arrayitems:

const addItem = useCallback(() => { setItems(() => [...items, 'New item']); }, [items]);

Ready! This way we cached the function. We press the buttons again and see that now when you press the button'click'our child component is not redrawn.
