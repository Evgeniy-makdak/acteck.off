Introduction to the Component Approach in React

Let us have a website. On this site we can highlight some blocks: header, content, sidebar, footer, and so on. Each block can be divided into smaller subblocks. For example, in the header you can usually highlight a logo, menu, contact block, and so on.

In React, each such block is calledcomponent. Each component can contain smaller components, which in turn are even smaller, and so on.

Each component in React corresponds to an ES6 module located in the foldersrc. The name of the file with the module is written with a capital letter and must correspond to the function that is located in the code of this module.

For example, a file calledApp.jsmust contain a function inside itApp:

import React from 'react'; function App() { // код компонента } export default App;

One of the components must be the main one - the one to which the remaining components are added. In React, by default this will be the componentApp. Other components will connect to this component. We will discuss how this is done later in the tutorial.
