Passing states to child components in React

Let us be given an array with products:

const initProds = [ {id: id(), name: 'product1', cost: 100}, {id: id(), name: 'product2', cost: 200}, {id: id(), name: 'product3', cost: 300}, ];

Let this array be located in the componentProducts. Let's write these products into the component state:

function Products() { const [prods, setProds] = useState(initProds); }

Now let's go through the products in a loop and display them in some layout:

function Products() { const [prods, setProds] = useState(initProds); const items = prods.map(prod => { return <div key={prod.id}> name: <span>{prod.name}</span>, cost: <span>{prod.cost}</span> </div>; }); return <div> {items} </div>; }

As you can see, the contents of our loopmapquite complex, especially if the product layout grows in the future. This complexity makes the code difficult to read, understand, and maintain.

It would be better to move the code responsible for displaying the product into a separate component. Let's call himProduct. Here is the code for our component:

function Product({ name, cost }) { return <div> name: <span>{name}</span>, cost: <span>{cost}</span> </div>; }

Let's now inside the componentProductsin a loopmapwe will use child componentsProduct:

function Products() { const [prods, setProds] = useState(initProds); const items = prods.map(prod => { return <Product key ={prod.id} name={prod.name} cost={prod.cost} />; }); return <div> {items} </div>; }

As you can see, the loop code is now simplified and easier to understand. In addition, now a separate component is responsible for displaying the product, in which we can create and then edit the layout of products.

Technically, it turns out that the parent component has a state with data, and the child components receive this data in the form of props and display them in the way we need.
