Component Props in React

In the previous lesson, we displayed several instances of products:

function App() { return <div> <Product /> <Product /> <Product /> </div>; }

So far, each call to a tag with a product displays the same thing on the screen. Let's now make each product unique.

Let, for example, each product have a name and price. Let's make sure that each call to a tag with a product displays a product with its own name and price, formatted in the layout we need.

To do this we need to take several steps.

To begin with, when calling a tag with a product, we will write two attributes into this tag: attributenamewith product name and attributecostwith the price, like this:

function App() { return <div> <Product name="product1" cost="100" /> <Product name="product2" cost="200" /> <Product name="product3" cost="300" /> </div>; }

The names and values ​​of these attributes will appear as an object in the first parameter of our component’s constructor functionProduct:

function Product(props) { console.log(props); // объект с ключами name и cost return <p> product </p>; }

The parameter name can be anything, but in React it is common to call itprops. In fact, it is not just a parameter name, but an important concept in React.

The essence of this concept is as follows: when you call a component tag, you can write attributes with data to this tag. This data will go topropscomponent. The component can then use this data, for example, to create the desired layout.

Let's do this:

function Product(props) { return <p> name: <span>{props.name}</span>, cost: <span>{props.cost}</span> </p>; }

It is more convenient and more accepted not to use the objectprops, and immediately executedestructurizationprops directly in the function parameters:

function Product({ name, cost }) { return <p> name: <span>{name}</span>, cost: <span>{cost}</span> </p>; }
