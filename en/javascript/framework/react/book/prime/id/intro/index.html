Discussion about unique ids in React

As you already know, an array of objects must contain uniqueid. Let's figure out where they come from.

Arrays of objects can have two origins: either they are sent to us by the server, or they are generated on the client (that is, in the browser).

The data sent to us by the server was usually stored there in a database. Databases (DBs) come in the form of SQL (for example, mySQL, PostgreSQL) or NoSQL (for example, MongoDB).

SQL databases are usually numericid, automatically arranged by the database in ascending order.

NoSQL databases typically haveid, which are random unique strings. These strings are assumed to have no matches (collisions) for two array elements.

Uniquenessidis achieved due to a sufficiently large length of random strings - so large that the probability of matching two strings will be close to zero.

Moreover, the more data in the database, the greater the likelihood of a collision. The programmer's task is to estimate the amount of data in advance and determine the optimal length of random strings so that the probability of collisions is minimal (small enough to be considered acceptable).
