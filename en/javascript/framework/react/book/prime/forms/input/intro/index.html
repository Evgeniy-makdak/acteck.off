Working with inputs in React

Working with inputs in React occurs using states. Each input is assigned its own state, containingvalueinput.

Let's look at an example. Let us have an input:

function App() { return <div> <input /> </div>; }

Let us also have a state:

function App() { const [value, setValue] = useState('text'); return <div> <input /> </div>; }

Let's go to the attributevalueinput we will bind to our state:

function App() { const [value, setValue] = useState('text'); return <div> <input value={value} /> </div>; }

In this case it turns out thatwhen the state changes, the input text will also change reactively.

This, however, has an interesting side effect: now when you run the code in the browser, you can't change the text in the input! Why: because when you enter text into the input, the state does not changevalue, accordingly, the text in the input should not change.

Try it yourself. Copy my code and run it for yourself. Try changing the text in the input - nothing will work. Open your browser console - you will see a React warning in it. This warning indicates to us that we have bound a state to the input, but thereby blocked the input.

Let's make it possible to enter text into our input. To do this, we need to make sure that when entering, our state changes to the current value of the input.

To begin with, you need to attach an event to the inputonChange:

function App() { const [value, setValue] = useState('text'); return <div> <input value={value} onChange={handleChange} /> </div>; }

This event behaves differently in React compared to pure JS. In React it istriggers immediately upon input change. That is, when entering or deleting a character.

Let's now add a handler for our event:

function App() { const [value, setValue] = useState('text'); function handleChange() { } return <div> <input value={value} onChange={handleChange} /> </div>; }

In this handler we must read the current input text and set it to the state using the functionsetValue.

The problem is thatthisThis function will not point to our input - this is a feature of React. To get the element where the event happened, we need to useevent.target:

function App() { const [value, setValue] = useState('text'); function handleChange(event) { console.log(event.target); // ссылка на DOM элемент инпута } return <div> <input value={value} onChange={handleChange} /> </div>; }

Let's output it usingevent.targetcurrent input text:

function App() { const [value, setValue] = useState('text'); function handleChange(event) { console.log(event.target.value); // текущий текст инпута } return <div> <input value={value} onChange={handleChange} /> </div>; }

Now let’s write the input text into our state:

function App() { const [value, setValue] = useState('text'); function handleChange(event) { setValue(event.target.value); } return <div> <input value={value} onChange={handleChange} /> </div>; }

Now we can enter text into the input. At the same time, the statevaluewill always contain the current input text.

We can easily verify this. Let's display the contents of our text in a paragraph. In this case, when entering text into the input, the entered text will automatically appear in the paragraph:

function App() { const [value, setValue] = useState(''); function handleChange(event) { setValue(event.target.value); } return <div> <input value={value} onChange={handleChange} /> <p>text: {value}</p> </div>; }

We can rewrite it into a more compact version with an anonymous arrow function:

function App() { const [value, setValue] = useState(''); return <div> <input value={value} onChange={event => setValue(event.target.value)} /> <p>text: {value}</p> </div>; }

Thus, for any input to work, we need the following: create a state for this input, bind the state to the attributevalueinput, attach eventonChangeon the input, in the event handler change the state of the input to its text.

These operations will need to be carried out with each input. That is, if you have two inputs, then you will have two states and two event handler functionsonChange.

Make two inputs. Let the text of the first input be displayed in the first paragraph, and the text of the second input - in the second paragraph.
