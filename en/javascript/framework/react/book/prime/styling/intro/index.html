Ways to style React components

There are many different ways to style React components. Here we will look at some of the main ones.

The standard approach that we can apply, as with conventional web page styling, is to useglobal CSS. Here we create one external common CSS file with styles. With this approach, all class names are in the global scope, which can lead to conflicts between them. You can also make a lot of small CSS files. This approach is ineffective when creating large, scalable applications.

Next approach -inline stylingUsing it, we can add CSS styles inline, just like we do in regular HTML. In this case we will work with the attributestyle, passing it the necessary CSS properties. This style of styling is considered bad manners and is only recommended for adding styles that are dynamically calculated during rendering. This method is good for quickly prototyping the interface of a single component.

The next couple of methods that we will consider are modern, increasingly popular, effective methods of styling. These are exactly the methods that are recommended for styling large, scalable React applications.

The first one is using the libraryStyled Components, which uses template strings for styling. This method allows us to write regular CSS in JS code, using all its functionality.

The second way is to use CSS modules. In this case, a CSS module is a CSS file in which, by default, all class names and animations are in the local scope, that is, available only inside the component that uses it.

In these last two methods, we do not have to worry about conflicts between class names, because they are unique - the concept of local scope is used. Also, when using them, there is no need for BEM methodology.

In the following lessons we will look at all the methods given here in more detail.
