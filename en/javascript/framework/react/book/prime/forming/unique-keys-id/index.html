Unique keys via id in React

In the above code, in the attributekeywe added the serial number of the element in the array. In fact, this practice is bad and should only be used as a last resort.

The fact is that when sorting an array, the elements will have different keys and React will not be able to correctly track the relationship between the array elements and the corresponding tags.

A better practice would be to add a unique identifier to each product, which will be used as a key.

Let's add a property to each product in our arrayidwith our product number:

const prods = [ {id: 1, name: 'product1', cost: 100}, {id: 2, name: 'product2', cost: 200}, {id: 3, name: 'product3', cost: 300}, ];

Now we use this as a keyid:

function App() { const res = prods.map(function(item) { return <p key={item.id}> <span>{item.name}</span>: <span>{item.cost}</span> </p>; }); return <div> {res} </div>; }
