Adding a reducer to a Redux application

In the last lesson we created a componentUserReactionso that each product also has user reactions. Now we need to write a reducer that will process the reaction counter when the user clicks on a button for a particular reaction.

Let's open our application with products, and in it there is a fileproductsSlice.jswith all reducers. Now in propertyreducerslet's create another reducerreactionClicked(we can put it first - beforeproductAdded, although it makes no difference). As usual, we’ll pass it the parametersstateAndaction:

reactionClicked(state, action) {},

Now, inside the curly braces that we left empty in the previous step, we will write the code for it. Firstly we will extract the product id frompayloadobjectactionand the name of the reaction:

const { productId, reaction } = action.payload

Then from the transferred state we get the desired product:

const currentProduct = state.find(product => product.id === productId)

And if such a product exists, then we increase the value of the transmitted reaction by1:

if (currentProduct) { currentProduct.reactions[reaction]++ }

That's it, our code forreactionClickedready:

reactionClicked(state, action) { const { productId, reaction } = action.payload const currentProduct = state.find(product => product.id === productId) if (currentProduct) { currentProduct.reactions[reaction]++ } },

Let's add an export of the resulting action creator at the end of the file:

export const { reactionClicked, productAdded, productUpdated } = productsSlice.actions

There are two important points to note here. As you remember from previous lessons, you cannot directly update a value in the reducer code like this (look at this line:reactions[reaction]++), but we can get away with it because we do it internallycreateSlice, which uses the Immer library. She will take care of converting our code into a “safe” value update. This allows us to write complex things in a simpler way.

The next thing to remember is the objectactionshould contain the minimum possible amount of information - just to indicate what happened. You should not do any calculations in it. All calculations for updating the state should be done in the reducer; here you can write as much logic for this as needed.
