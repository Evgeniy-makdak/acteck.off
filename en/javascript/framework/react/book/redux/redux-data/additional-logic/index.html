Adding additional logic to a Redux application

In this lesson we will add the final component to our application. To be more specific, we will add to the product the reaction of the user who either ordered it or read information about it.

Let's open our products application. Just like with the merchant name, we need to show user reactions in multiple places in the app. This means that we will need a separate component. So let's start with what's in the folderproductslet's create a fileUserReactions.jsx. Let's first write in it an object that will contain our reactions:

const reactionObj = { good: '+', soso: '+/-', bad: '-', }

And below we will begin to write the function itself, into which we will pass the sliceproduct:

export const UserReactions = ({ product }) => {}

Now we write the body of the function in curly bracesUserReactions. To do this, let's sort it out usingmapour key-value pairsreactionObj, for each we get a designation for the button ('+', '+/-'or'-') and the numerical value of a particular reaction (we will extract it fromproductin store by reaction name):

const userReactions = Object.entries(reactionObj).map(([name, image]) => { return ( <button key={name} type="button" className="reaction-button"> {image} {product.reactions[name]} </button> ) })

And after that, at the end of the function code, we will return the layout with buttonsuserReactions:

return <div>{userReactions}</div>

Let's also add some more styles toindex.css:

.reaction-button { margin-right: 10px; margin-bottom: 10px; }

If this looks a little confusing, don't worry, it will become much clearer later.
