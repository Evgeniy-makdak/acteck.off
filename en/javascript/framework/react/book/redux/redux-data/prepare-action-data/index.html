Preparing data for the action object in Redux

We recently talked to you about the fact that the functioncreateSliceexpects one argument upon creationaction.payload. On the one hand, this simplifies its use, on the other hand, it may require additional steps to prepare the contents of the action object.

Let's open our application with products, and in it there is a fileNewProductForm.jsx. Pay attention to the following line of code:

dispatch(productAdded({ id: nanoid(), name, desc, price, amount }))

Here we collectpayloadobject directly in the React component and passing it to the actionproductAdded. What if we have to send the same action from several components or the logic for assembly turns out to be complex? In theory, our component should not care what the object looks like inpayloadfor a given action. In addition, we are not happy with the constant duplication of code.

Our famouscreateSlice, because when writing a reducer it allows us to use a callback functionprepare, in which you can write various logic, generate random numbers, etc. In this case, the value for the reducer field can be represented as the following object:

{reducer, prepare}

Let's open our fileproductsSlice.jsand change the code for the reducerproductAdded. Now this piece of code:

productAdded(state, action) { state.push(action.payload) },

We will change to the following one, in which the functionreducerwill update the state in the store, andprepare- return objectpayloadwith the generated id and our other data:

productAdded: { reducer(state, action) { state.push(action.payload) }, prepare(name, desc, price, amount) { return { payload: { id: nanoid(), name, desc, price, amount, }, } }, },

Since we will now be generating the id here and not in the component, let's add nanoid to the import:

import { createSlice, nanoid } from '@reduxjs/toolkit'

Finally, let's make changes toNewProductForm.jsx. Instead of the line:

dispatch(productAdded({ id: nanoid(), name, desc, price, amount }))

We will have one in which we will simply pass the data we need, separated by commas (don’t forget to removenanoidfrom import):

dispatch(productAdded(name, desc, price, amount))

Let's launch the application, then create a new product and make sure we don't break anything.

Open your application with students. In fileStudentsSlice.jsrewrite your reducerstudentAddedso that it is in the form of an object{reducer, prepare}, as shown in the lesson.

Make the appropriate changes to the fileNewStudentForm.jsx, as shown in the lesson. Launch the application and make sure everything works as before.
