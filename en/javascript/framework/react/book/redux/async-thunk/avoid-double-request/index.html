Removing Query Duplication in Redux

This lesson will be useful to us if, when launching an application, we send a duplicate data request, and then pairs of products with the same ids arrive, and, as a result, scary red warnings appear in the browser console.

The reason for this lies in the way components are mounted in React.18versions and higher in development mode (dev), when usingReact.StrictMode(they say that in production mode everything is ok with this). First the component is connected, then disconnected and connected again. Therefore, the request is sent twice.

It's possible that by the time you learn this, this double mount in React will have already been changed in some way. We won’t go deep into the weeds, but for now we’ll just use the React hookuseRef, which will help us get around this misunderstanding.

Let's open our application with products, and in it there is a fileProductsList.jsx. Import the hook into ituseRef:

import { useEffect, useRef } from 'react'

Now let's change the following block of code a little fromuseEffect:

useEffect(() => { if (productStatus === 'idle') { dispatch(fetchProducts()) } }, [productStatus, dispatch])

First, let's introduce a new variabledataFetchForuseRef. Initially, let's set its propertycurrentVfalse. If the application has already been launched and, accordingly, the request has been sent, then the propertycurrentit will already betrue, which means we will simply exit the function and the data request will not occur again. So the above piece of code will now be like this:

const dataFetch = useRef(false) useEffect(() => { if (dataFetch.current) return dataFetch.current = true if (productStatus === 'idle') { dispatch(fetchProducts()) } }, [productStatus, dispatch])

Now let’s launch our application again and note with relief the disappearance of the red warnings in the console, and we will see a list of products without duplicates, now there are8, as we planned on the server. Now in Redux DevTools, actions generated on request are no longer repeated.
