Retrieving seller data in Redux

Let's launch our product app. We see that sellers are not displayed in product cards. Let's get them from the server and output this data. To do this, we will have to carry out the already familiar sequence of actions.

First, let's open the fileserver.js. Let's find the arrayhandlersand add another GET request processing to it (after GET processing for products). As you can see, it is almost identical:

http.get('/fakeServer/sellers', async () => { await delay(ARTIFICIAL_DELAY_MS) return HttpResponse.json(db.seller.getAll()) }),

We processed the request on the server. The next step is in the filesellersSlice.jsmust write a thunk to get data from the server. To do this, we import into itclientAndcreateAsyncThunk:

import { createSlice, createAsyncThunk } from '@reduxjs/toolkit' import { client } from '../../api/client'

Now we will receive sellers from the server, so we will remove them frominitialState, leaving just empty parentheses:

const initialState = []

And then before the functionsellersSlicelet's create our thunkfetchSellers. We will pass to the client the path we specified on the server:

export const fetchSellers = createAsyncThunk( 'sellers/fetchSellers', async () => { const response = await client.get('/fakeServer/sellers') return response.data } )

All we have left is forsellersSlicecreate an additional reducer. Here we will limit ourselves to processing a successful request. All actions generatedfetchSellers, will be identical to the actionsfetchProducts:

const sellersSlice = createSlice({ name: 'sellers', initialState, reducers: {}, extraReducers(builder) { builder.addCase(fetchSellers.fulfilled, (state, action) => { return action.payload }) }, })

Let me remind you that the last time we received products, we changed the current state value. Here we took a slightly different route and simply returned the new value, so increateSliceIt's also possible. Thus, we achieved two goals. Firstly, we received the data, and secondly, now we have a guarantee that nothing else will appear in the list of sellers, because we have completely rewritten the slice with the valueaction.payload.
