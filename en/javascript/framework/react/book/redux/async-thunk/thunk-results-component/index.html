Outputting thunk results in a component in Redux

In the last lesson we sent a POST request using thunkaddProduct. Let's now display the results of its work in the component.

Let's open our application with products, and in it there is a fileNewProductForm.jsx, since this component is responsible for adding a new product. Let's look at the import lines. Let's replace the import of the actionproductAdded, to import thunkaddProduct:

import { addProduct } from './productsSlice'

Now, since we are not tracking the status in the slice'pending'request, let's make it so that the user can click on the button to save the product only once, because we don't need the same repeated requests. To do this, let's create another local state:

const [requestStatus, setRequestStatus] = useState('idle')

Further after the handlers and before the functiononSaveProductClickLet's write a code in which we check whether all form fields are filled in and whether the request status is in'idle':

const canBeSaved = [name, desc, price, amount, sellerId].every(Boolean) && requestStatus === 'idle'

Then we will change the code toonSaveProductClick. Firstly, it will be an asynchronous function and it will be executed if the above condition is true:

const onSaveProductClick = async () => { if (canBeSaved) {} }

Despite the fact that we do not track the status in the slice'rejected', we can still display an error in the console; for this we will use the constructiontry-catch. Also here we will addfinallyto set the local state to again after executing the request'idle':

const onSaveProductClick = async () => { if (canBeSaved) { try { } catch (err) { console.error(save product error: , err) } finally { setRequestStatus('idle') } } }

WITHcatchAndfinallywe understand the blocks, let's write the code for the blocktry. Here we will set the local status to'in pogress', until some response is returned to us as a result of the thunk, then we send our thunkaddProduct. Use constructtry-catchdepending on the type of response, the RTK function will help usunwrap, which it adds to the returned promise. Then, if the request was successful, we set the local states to their initial states. Full code foronSaveProductClickwill look like this:

const onSaveProductClick = async () => { if (canBeSaved) { try { setRequestStatus('in progress') await dispatch( addProduct({ name, desc, price, amount, seller: sellerId })).unwrap() setName('') setDesc('') setPrice(0) setAmount(0) setSellerId('') } catch (err) { console.error('save product error: ', err) } finally { setRequestStatus('idle') } } }

Let's launch our application and try to add a new product. As you can see, if the request is successful, the fields are cleared and the new product is added to the product list. Also, look in your browser at Redux DevTools and run through its tabs, look at the actions and how your state changes.
