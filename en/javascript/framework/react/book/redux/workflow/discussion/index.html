Discussion of the Redux application

In previous lessons, we implemented all the components necessary for a Redux application to work. Let's summarize and briefly go through the main stages of our application working with products.

When first launched, our application fetches a list of two products from the store usinguseSelectorand displays them on the screen. The screen also displays a form where we can enter data for the new product. When the user clicks the save button, the button handler sends an actionproductAdded, containing the new product data that the user entered into the form. The reducer function that we wrote for the slice with products receives this action and adds an object with a new product to the array with products. Store tells components that the data stored in state has changed. Our componentProductsListreads the modified array, causes rendering, as a result of which we see the added product in the list of products.

Now let's open Redux DevTools in the browser and add another product to the application, and then look at the Log monitor tab, here we can see what our state was when the application was launched, and how then, after clicking the save button, the action appeared. We can see its propertypayloadand changes in the global state.
