Working Redux application

Now, armed with the terms, we can briefly look at the main stages of the Redux application that you will create in the future.

The application works according to the idea of ​​unidirectionality of data, as we mentioned earlier.

At the initialization stage (when the application is launched for the first time), the store store is created using the root reducer. The store calls this reducer only once and records its return value as the initial state. During initial rendering, application components use the current state recorded in the storage. They also subscribe to storage updates to immediately learn about state changes.

As the application continues to run, the following happens. Let's assume the user clicks on a button. In this case, an action is generated using code and sent to the storage. The store runs the reducer again with the state stored in it and the resulting action. As a result, the changed state value is saved as a new one. The repository also notifies components subscribed to changes that changes have taken place. In turn, the components look if the parts of the state they need have been changed. Finally, if changes have taken place, the component calls rendering to display the new data.
