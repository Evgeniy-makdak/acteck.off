Import from site root in NextJS

In this tutorial we will continue to study imports in NextJS. But now we will study a feature that is in NextJS, but not in NodeJS.

The fact is that in NodeJS, when importing, we can use either relative paths (without the leading slash) or absolute paths from the root of the operating system (which creates problems with transferring the project to another computer).

In real life, however, we need one more type of import - from the project root. NodeJS does not have such import built-in, but NextJS does.

This import is enabled when NextJS is installed. Remember there was a questionWould you like to customize the default import alias? If you answer Yes, then we will be able to set imports from the project root.

Let's see how it's done. For example, let's say we have the following file structure:

  • /src/
    • data.js
    • /app/
      • /test/
        • page.jsx

As you can see, the filedata.jslies far from the filepage.jsx. Let's import our data file to the page file. We use a relative path for this:

import data from '../../data.js'; export default function Test() { return <h1>Test</h1>; }

As you can see, importing via a relative path is not very convenient, since we have to start the path from../. At the same time, if we move our component to another location, the path to the data file will break.

It would be more convenient to specify the path to the file with data from the project root (this is considered the foldersrc). To do this, at the beginning of the path you need to write the symbol@. Let's do this:

import data from '@/data.js'; export default function Test() { return <h1>Test</h1>; }

Let me emphasize once again that such import is possible in NextJS and only if you answered the right question correctly during installation. It just won't work in NodeJS or pure JavaScript.

The following file structure is given:

  • /src/
    • /app/
      • data.js
      • /test/
        • page.jsx

Import the data file into the componentTest. When importing, specify the path from the project root.

Change your file structure to the following:

  • /src/
    • /app/
      • data.js
      • /test/
        • /sub/
          • page.jsx

Make sure that importing from the project root will still work without problems.
