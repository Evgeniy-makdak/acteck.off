Common Component Data in NextJS

There are situations when multiple components need to access the same data. For example, let's say we have some array with users. Suppose that on one route we want to show a list of users, and on another route with a dynamic parameter - a description of a specific user according to hisid.

Let's implement what is described. Let's make the following file structure:

  • /app/
    • /users/
      • users.js
      • /list/
        • page.jsx
      • /show/[id]/
        • page.jsx

Let's create a separate file with user data:

export default users = [ { id: 1, name: 'name1', surn: 'surn1', }, { id: 2, name: 'name2', surn: 'surn2', }, { id: 3, name: 'name3', surn: 'surn3', }, ];

Let's create a component to display a list of users:

import users from '../users.js'; export default function List() { let list = users.map(user => { return <li> {user.name} </li>; }); return <ul> {list} </ul>; }

Let's make a component to display a specific user by hisid:

import users from '../../users.js'; export default function User({params}) { let user = users[params.id]; return <div> <span>{user.id}</span> <span>{user.name}</span> <span>{user.surn}</span> </div>; }
