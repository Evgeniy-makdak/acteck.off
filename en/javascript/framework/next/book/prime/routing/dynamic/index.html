Dynamic routes in NextJS

In NextJS you can createdynamicroutes. Such routes assume that part of the URL can have an arbitrary value. For example, let's take the address/users/1, in which instead of one there can be any number that technically corresponds to the id of the user we want to display.

This way we don't need to create multiple folders for every possible number. It is enough to declare this part of the address dynamic. In this case, all requests of this type will be processed by a common filepage.jsx.

In order to make a dynamic parameter, you need to put the name of the corresponding folder in square brackets. In our case, this will be the following folder structure:

  • /app/
    • /users/
      • /[id]/

Let's make the corresponding component:

export default function User() { return <p> user </p>; }

In this case, the parameter we set will be stored in a special objectparams. Let's print the value of this parameter:

export default function User({ params }) { return <p> user {params.id} </p>; }
