Production mode in the NextJS framework

Production mode is designed to run a project on a hosting. The main purpose of this mode is to ensure maximum performance and security of the project, as well as reduce the size of loaded files to save system memory.

Main features of production mode:

  • Code optimization - the code is compiled before launching in production. For example, JavaScript and CSS are minified to reduce their size and speed up page loading.
  • Static generation - the framework generates pages in advance, which speeds up their loading on the site.
  • Optimize images that are automatically compressed and adjusted to fit different screen sizes and devices.
  • Removing debugging information - all error debugging code is removed from browser pages. This is necessary for the safe operation of the project and hiding information about the file structure from users.
  • No hot reboot - Unlike development mode, there is no automatic reboot with every change made.

Production mode is launched with two commands. First you need to perform a production build of the project:

npm run build

And then run the assembled project:

npm start

After starting the project, you will see information about the basic structure of the project. Special symbols indicate the types of project content - static and dynamic. At the moment we only have static content.

To test run the project in production mode.

Turn off production mode and turn on development mode. Later in the lessons we will work in this mode.
