Types of Code Rendering

Rendering- the process of creating the final HTML code that will be processed by the browser.

There are two main types of rendering:

  • server, abbreviated as SSR (server side rendering)
  • client, abbreviated CSR (client side rendering)

In server-side rendering, the HTML code is created on the server and then sent to the browser. In client-side rendering, the HTML code is dynamically generated in the browser via JavaScript.

Both methods have their pros and cons.

SSR Features:

  • HTML code is generated on the server for each request. This increases the load on the server.
  • The user immediately receives a completed page, making the page available before the JavaScript is loaded and executed.
  • SEO-friendly: search engines will be able to process the site correctly.

Features of CSR:

  • Performance: If the browser is busy with work, such as processing data, it slows down other processes, such as building the DOM tree. During this time, users may see a blank screen.
  • Security: Data stored on the client side can be easily discovered when viewing the page in the browser debugger. Therefore, you cannot work with sensitive information, such as passwords or personal data of registered users on the client side.
  • The load on the server is reduced: the server does not need to create HTML code for each new request.
  • The load on the Internet channel is reduced: in the case of client rendering, less text needs to be transferred from the server to the browser.
