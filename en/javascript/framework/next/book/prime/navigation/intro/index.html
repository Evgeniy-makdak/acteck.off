Introduction to Navigation in NextJS

In this section of the tutorial we will talk about how links work in NextJS.

First, let's discuss what's so special about links. Let's look at them in different approaches to creating a website.

React website

Let's say you have a website in React. As you know, React is an SPA - a site in which the user always remains on the same page. You can connectReact Routerand the user will think that he is walking through different pages, while physically remaining on the same file.

In this case, the user will click on the links, it will seem to him that he is going to another page, but the site page in the browser will not be reloaded - React will dynamically form what the user sees on the screen.

In this scheme, the user receives a layout of the site when they first visit the page, and then parts of the site are simply changed via JavaScript. In this case, the site can request data from the server via AJAX.

This approach reduces the load on the hosting (whose resources cost our money), as well as the load on the user’s Internet channel.

But this approach has a drawback - an SEO problem. It seems that the site has a set of links, and something changes in the address bar, and you can even send a link to a specific page of the site through the messenger. However, the page content is generated dynamically on the client and search engines are not able to process this. Therefore, such a site will not be promoted.

Not all sites, however, are designed for search engine promotion. And if yours is not intended, then this scheme is quite working for you.

Website on back

Let your site be made in one of the back-end languages. For example, in PHP.

In this case, when the user clicks on the links, the site page will load in its entirety each time.

The problem is that the user already has a significant part of the page. After all, as you already know, usually only the content and metadata change on a page, while everything else remains unchanged.

Why do we need to download the rest if only the content has changed? This creates an increased load on the server and Internet channel.

There is still a problem. Let the page we receive contain a list of products in the content, like in an online store. Obviously, each product has the same layout. It would be more optimal to download only the product data and the product output template. And then display each product in this template on the client. But we download products along with repeating product layout.

To summarize, if we have a site on a clean back, we drive a lot of unnecessary data around the network.

However, in this case we do not have an SEO problem. The requested URL always returns static content that is understandable to search engines.

Website on NextJS

NextJS combines both approaches. When a user types a URL directly into the address bar, he receives static page content in response.

When the user starts clicking on links on our site, the site page is not updated as a whole, but the data the user needs is loaded through AJAX.

When a search engine navigates our site, it sees the static content it needs. And when a user navigates our site, our site begins to behave like an SPA.

And best of all, NextJS does it all automatically! We don't need to worry about it - he will do everything himself. Beauty!
