Default slot file in NextJS

In the previous lesson we made a dynamic slot namedinfo:

export default function RootLayout({children, info}) { return ( <html lang="en"> <body> <header> <div> {info} </div> </header> <main> {children} </main> <footer> footer </footer> </body> </html> ); }

There is, however, a problem. The fact is that our slot only works for two types of addresses:/usersAnd/posts. If we enter another address into the address bar, for which our slot is not defined, then NextJS will display404error.

To solve the problem, place a special file directly in the folder with the slotdefault.jsx. This file should contain a component that will be displayed when a URL that is not specified for our slot is entered into the address bar.

That is, in our case we get the following file structure:

  • /app/
    • /@info/
      • default.jsx
      • /users/
        • page.jsx
      • /posts/
        • page.jsx

Make sure the dynamic slot issues404an error when a URL that is not specified for our slot is entered into the address bar.
