Parallel routing in NextJS

In NextJS, you can make part of a site show different content depending on the requested URL. It's calledparallelrouting

Let's look at an example. Let's say we have the following site layout:

export default function RootLayout({children}) { return ( <html lang="en"> <body> <header> header </header> <main> {children} </main> <footer> footer </footer> </body> </html> ); }

Suppose we want dynamic information to be displayed in the header. Let by URL/usersthe component with users will be displayed, and at the URL/postscomponent with posts.

For such parallel routing, you need to make a special folder, starting its name with the symbol@. For example, let's call our folder@info.

We will place two components in this folder. The first one is for showing users in a folder/users/, and the second one is for displaying posts in the folder/posts/. We will get the following file structure:

  • /app/
    • /@info/
      • /users/
        • page.jsx
      • /posts/
        • page.jsx

Now we need to write a special insert command in the site layout. To do this you need to use a specialdynamic slot.

To understand what this is, first look at the function parameter of our site layout:

export default function RootLayout({children}) { return ( ); }

Please note that as a parameter we are passing an object from which we extract the site content into a variablechildren.

There may be other keys in this object. They correspond to those folders whose name we started with@.

In our case we have a folder@info, which means that we will have access to the variableinfo, depending on the URL, containing the text of one of the components - users or posts. Let's get this variable:

export default function RootLayout({children, info}) { return ( ); }

Now inside the layout layout we can display the variableinfo. Depending on the URL, the text of one of our components will be inserted into this place:

export default function RootLayout({children, info}) { return ( <html lang="en"> <body> <header> <div> {info} </div> </header> <main> {children} </main> <footer> footer </footer> </body> </html> ); }
