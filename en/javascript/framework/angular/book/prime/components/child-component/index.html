Child Components in Angular

As mentioned at the very beginning of the tutorial, in Angular the entire project is broken down into components. In previous lessons we only worked with the default main component. Let's now learn how to create child components.

The code for new components is automatically generated by Angular using a special terminal command:generate component. After the wordcomponentwe need to write the name of our new component. Let's say we want to generate a componentuser:

ng generate component user

After executing the command we will see that we have a new folderapp/user. It will contain all the standard component files.

In the file with the componentuser.component.tsThe component decorator will be automatically created:

@Component({ selector: 'app-user', templateUrl: './user.component.html', styleUrls: ['./user.component.css'] })

Also in this file the component class will be automatically created:

export class UserComponent { }

Now our child component needs to be connected to the parent. Let the parent be the componentapp, initially present when the framework was installed. We import our component into it:

import { UserComponent } from './user/user.component';

In the component decorator in the keyimportsLet's indicate our imported component:

@Component({ selector: 'app-root', imports: [UserComponent], // тут templateUrl: './user.component.html', styleUrls: ['./user.component.css'] })

Now we can display the contents of the child component in the parent. To do this, in the file with the parent template you need to write a special tag, the name of which is specified in the propertyselectordecorator@Componentdescendant In our case this is the tagapp-user. Let's write it in the parent's template:

some text <app-user></app-user>
