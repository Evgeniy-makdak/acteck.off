Two-way event binding in Angular

Also in Angular you can do two-way binding of events of both components: parent and child. Let's make an input in the child HTML template with a two-way binding of the username and a function that changes the name:

<input [ngModel]="userName" (ngModelChange)="onNameChange($event)" />

Let's go to our child component and import the decorator into it@Input. It is needed to work with inputs:

import { Component, Input, Output, EventEmitter } from '@angular/core';

Next we bind the decorator@Inputto the username. And the decorator@Outputbind to objectuserNameChange, which will be called in the functiononNameChange:

export class UserComponent { @Input() userName: string = ''; @Output() userNameChange = new EventEmitter<string>(); onNameChange(model: string) { this.userName = model; this.userNameChange.emit(model); } }

Now go to the main component and add the name property to its class:

export class AppComponent { name: string = 'alex'; }

And in the HTML template of the parent component, we perform two-way property bindinguserNamewith property valuenameparent component:

<user-data [(userName)]="name"></user-data> <p>You choose name: {{ name }}</p>
