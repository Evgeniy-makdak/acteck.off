Жизненный цикл компонента в Angular

Каждый компонент имеет жизненный цикл, который начинается, когда Angular создает объект класса компонента и отображает представление компонента вместе с его дочерними представлениями.

Жизненный цикл продолжается при обнаружении изменений, так как Angular проверяет, когда изменяются свойства, связанные с данными, и обновляет представление и экземпляр компонента по мере необходимости.

Жизненный цикл заканчивается, когда Angular уничтожает экземпляр компонента и удаляет его отрисованный шаблон из DOM.

В фреймворке Angular есть специальные методы, с помощью которых можно перехватить момент, когда компонент переходит к другой фазе жизненного цикла. Эти методы называются хуками.

Хуки автоматически будут вызваны при наступлении события жизненного цикла. В следующих уроках мы разберем, какие есть хуки и как ими пользоваться.
