ngOnDestroy hook in Angular

HookngOnDestroyFires when a component is removed. In this hook you can release resources, remove subscriptions to element events, leave timers, etc.

Hiding and deleting a component in Angular occurs when it is reactively shown and hidden using a conditionif. That is, Angular does not just hide components, but removes them and returns them back.

Also, components are created and deleted as they are reactively iterated through loops. In this case, Angular also creates and deletes components.

Let's try it in practice. Importing the interfaceOnDestroy:

import { OnDestroy } from '@angular/core';

Let's connect it to the class:

export class UserComponent implements OnDestroy { }

Now let's write our hook:

export class UserComponent implements OnDestroy { constructor() { console.log('constructor is started'); } ngOnDestroy() { console.log('onDestroy is applied'); } }

Make it reactiveifwith which the child component will be displayed. Check the operation of the hook you learned.

The parent component contains an array of user names. Make a child component with a user. Loop through the array and print out the components with users. Make a button that, when clicked, will remove elements from the array. Check that the deletion hook will fire in the child components.
