Exact address matching in Angular

AttributerouterLinkActiveactually checks the URL not for an exact match, but for a partial one. For example, let our links look like this:

<nav> <a routerLink="/blog/" routerLinkActive="active" > Aaaa </a> <a routerLink="/blog/page/" routerLinkActive="active" > Bbbb </a> </nav>

In this case, if we are on the url/blog/page/Both links will be styled.

You can force link addresses to be checked for an exact match using the attributerouterLinkActiveOptions. See how to use it:

<nav> <a routerLink="/blog/" routerLinkActive="active" [routerLinkActiveOptions]="{exact:true}"> Aaaa </a> <a routerLink="/blog/page/" routerLinkActive="active" [routerLinkActiveOptions]="{exact:true}"> Aaaa </a> </nav>

Check that by default the links do not catch an exact match.
