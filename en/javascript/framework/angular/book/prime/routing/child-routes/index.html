Child routes in Angular

It may happen that a component that is already involved in routing also has child components. You can make sure that these child components are also shown through routing.

For example, let's say we have a componentAaaa. We access it via URL/aaaa/. Let's create two more child components:ChixAndChiy. We will show them inside the component viewAaaa. UAaaawill have its own tagrouter-outlet, in which, depending on the URL, either one or a second child component will be shown.

So, let's implement the described.

Let's create the first child component:

ng generate component chix

Let's create a second child component:

ng generate component chiy

Let's write separate child routes:

let childRoutes: Routes = [ { path: 'chix', component: ChixComponent }, { path: 'chiy', component: ChiyComponent }, ];

Let's add them to the component routeAaaaas children:

const appRoutes: Routes = [ { path: 'aaaa', component: AaaaComponent, children: childRoutes }, ];

The path from the parent component's route will be the beginning of the child's path. This means that child components will be available at the following URLs:/aaaa/chixAnd/aaaa/chiy.

Let's now configure the componentAaaa. Import the routing tag:

import { RouterOutlet } from '@angular/router';

Let's write it in the decorator:

@Component({ ...... imports: [RouterOutlet], ...... })

Insert the tag into the view:

<p>it works</p> <router-outlet></router-outlet>

Now you can access child components by their URLs and these components will be shown. Try it!
