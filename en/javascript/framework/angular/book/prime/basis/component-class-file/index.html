Component class file in Angular

The main part of any component is its functionality, that is, its program itself. In Angular, the functionality of a component is written in TypeScript and isOOP class.

In each component, this class is located in a file with the extension.ts. In our main component this will be the fileapp.component.ts.

Let's take a look at the contents of this file. First, pay attention to the class of our component:

export class AppComponent { title: string = 'test'; }

A decorator is applied to this class@Component. This decorator takes as its parameter an object that configures our component:

@Component({ selector: 'app-root', standalone: true, imports: [FormsModule], templateUrl: './app.component.html', styleUrls: ['./app.component.css'], })

In propertytemplateUrlthe path to the file containing the layout of our component is specified.

In propertystyleUrlsAn array of paths to CSS files that stylize the layout of our component is specified. As you can see, there may be more than one style file.

In propertyselectorthe selector of the HTML tag into which (be mounted) HTML code of our component. In our case we see the nameapp-root. This name corresponds to a tag of the same name. This tag is written in the layout filesrc/index.html. The layout of our component will be inserted into the place where this tag is written.

In propertyimportsThe modules that are imported into our component are indicated.

In propertystandalonethe value will always be specifiedtrue. This is more of an Angular utility thing for now, so don't get into it yet.

Open the fileapp.component.tsand study it.

Make several files with styles and include them in the decorator.
