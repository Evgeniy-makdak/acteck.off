toPrecision Method

MethodtoPrecisionrounds a number to a given sign. UnlikeMath.roundRounding can be done not only in the fractional part. The method parameter specifies how many digits should remain in the number. The remaining digits will be discarded. The last remaining number will be rounded using mathematical rounding rules. If the parameter is empty, the original number will be returned. If the specified number of digits is not achievable by cutting off the decimal part, converts the number to exponential form.

Syntax

число.toPrecision(длина);

Example

In this example the number678.19324by usingtoPrecisionwill lead to4numbers, moreover, since after1there is a figure9, then according to the rules of mathematics, one is converted to two:

let num = 678.19324; console.log(num.toPrecision(4));

Result of executing the code:

678.2

Example

In this example, the number should be reduced to two digits and not only the fractional part will be discarded, but also the integer part. Therefore, the number will be converted into exponential form:

let num = 678.19324; console.log(num.toPrecision(2));

Result of executing the code:

6.8e+2

Example

In this example the number12leads to3numbers. Since the number is an integer, the fractional part will appear0:

let num = 12; console.log(num.toPrecision(3));

Result of executing the code:

12.0

Example

Let's now give the number12to four digits. Two will appear in the fractional part0:

let num = 12; console.log(num.toPrecision(4));

Result of executing the code:

12.00

Example

Let's now give the number12.1to five digits:

let num = 12.1; console.log(num.toPrecision(5));

Result of executing the code:

12.100
