getComputedStyle function

FunctiongetComputedStyleallows you to get the value of any CSS property of an element, even from a CSS file.

Syntax

let объект = getComputedStyle(элемент);

Details

How it works (note: not as we expect): the function takes an element as a parameter and returns an object that contains all the CSS properties of the passed element. Let's put this object into a variablestyle. The name is arbitrary, it’s just a variable - we’ll use it as we come up with:

let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); // в style лежат CSS свойства

Let's output, for example, the width. It's done like this -style.width:

let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); console.log(style.width);

To display, for example, the leftpadding- we do this -style.paddingLeft:

let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); console.log(style.paddingLeft);

In the following example, we will output all the CSS properties we are interested in for our element:

<div id="elem">text</div> #elem { width: 200px; height: 200px; padding: 30px; border: 20px solid #BCBCBC; overflow: auto; } let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); console.log('paddingLeft: ' + style.paddingLeft); // выведет '30px' console.log('borderTopWidth: ' + style.borderTopWidth); // выведет '20px' console.log('borderTopStyle: ' + style.borderTopStyle); // выведет 'solid'

getComputedStyle inaccuracy

Sometimes functiongetComputedStyledoes not work quite correctly with width and height. This is due to the fact thatpaddingand the border expand the block. In the following example, the block width is given200px, as well as the border andborder. Actual block width300px, ButgetComputedStylewill still output200px:

<div id="elem">text</div> #elem { width: 200px; height: 200px; padding: 30px; border: 20px solid #BCBCBC; overflow: auto; } let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); console.log('width: ' + style.width); // выведет '200px' console.log('height: ' + style.height); // выведет '200px'

That is: it turns out thatgetComputedStyleignores block extension, shows its dimensions as if this extension did not exist. But that's not all: the presence or absence of a scrollbar also matters - some browsers subtract the width of the scrollbar from the width calculated usinggetComputedStyle, and some don’t take it away. In general, everything here is not cross-browser at all and is bettergetComputedStyleTo determine the width and height, do not use them, but use the metrics that we will study below.

Calculated values

There is one more nuance: if the width is specified in%- then after workgetComputedStylewe will see her inpx. That is, essentiallywe do not get the specified width, but the calculated one. See the following example:

<div id="elem">text</div> #elem { width: 30%; /* ширина задана в % */ height: 200px; padding: 30px; border: 20px solid #BCBCBC; overflow: auto; } let elem = document.querySelector('#elem'); let style = getComputedStyle(elem); console.log('width: ' + style.width); // выведет ширину в px
