Error duplicating the function of editing DOM elements in JavaScript

Let us have some list:

<ul> <li>1</li> <li>2</li> <li>3</li> </ul>

Let's get the list itself and its items into separate variables:

let ul = document.querySelector('ul'); let lis = document.querySelectorAll('li');

Let's make it so that the items in our list can be edited using the input that appears:

for (let li of lis) { li.addEventListener('click', function func() { let input = document.createElement('input'); input.value = list.textContent; li.textContent = ''; li.append(input); input.addEventListener('blur', function() { li.textContent = this.value; li.addEventListener('click', func); }); li.removeEventListener('click', func); }); }

Suppose now we want to be able to add new items to the list. For this purpose, let us have the corresponding input under the list:

<input id="adder">

Let's get a link to this input into the variable:

let adder = document.querySelector('#adder');

Let's make sure that when the input loses focus, a new item with text taken from our input is added to the list:

adder.addEventListener('blur', function() { let li = document.createElement('li'); li.textContent = this.value; ul.append(li); });

Suppose now we want the newly added items to also be editable. Editing by itself will not work for them, because when we attached a click handler to the list items, these items did not yet exist.

Let's look at possible solutions to this problem.

Solution one

The simplest solution is to duplicate the function codefunc, linking it to newly created items:

adder.addEventListener('blur', function() { let li = document.createElement('li'); li.textContent = this.value; li.addEventListener('click', function func() { // тут мы дублируем код }); ul.append(li); });

Of course, we immediately see a drawback in this solution - duplicating the code is not correct.

Solution two

To solve the duplication problem, it is logical to add the functionfuncoutwards, making it a Function Declaration:

function func() { let input = document.createElement('input'); input.value = list.textContent; li.textContent = ''; li.append(input); input.addEventListener('blur', function() { li.textContent = this.value; li.addEventListener('click', func); }); li.removeEventListener('click', func); }

This is where the problem awaits us. The fact is that our function used a variableli, obtained from the outer scope. But after removing the function, this variable is no longer visible!

To solve the problem we will transfer ourliparameter:

function func(li) { let input = document.createElement('input'); input.value = list.textContent; li.textContent = ''; li.append(input); input.addEventListener('blur', function() { li.textContent = this.value; li.addEventListener('click', func); }); li.removeEventListener('click', func); }

And here our solution gives rise to another problem. The point is that you can’t just pass a parameter to an event handler:

for (let li in lis) { li.addEventListener('click', func(li)); // не работает! }

To solve this problem, simply call our function inside the anonymous handler:

for (let li of lis) { li.addEventListener('click', function() { func(li); }); }

And we’ll do the same when creating a new list item:

adder.addEventListener('blur', function() { let li = document.createElement('li'); li.textContent = this.value; li.addEventListener('click', function() { func(li); }); ul.append(li); });

Solution three

There is a more elegant solution. You can just usedelegation. In this case, the problem with new list items simply will not arise:

ul.addEventListener('click', function(event) { if (event.target.tagName === 'LI') { // ловим именно клик на li, не на инпут let li = event.target; let input = document.createElement('input'); input.value = li.textContent; li.textContent = ''; li.append(input); input.addEventListener('blur', function() { li.textContent = this.value; }); } });

In this case, we won’t need a loop through list items at all, and the code for creating a new list item will be reduced to this:

adder.addEventListener('blur', function() { let li = document.createElement('li'); li.textContent = this.value; ul.append(li); });
