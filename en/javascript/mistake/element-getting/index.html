Error getting element in JavaScript

Sometimes novice programmers make the mistake of using the methodquerySelectorAllinstead of a methodquerySelector.

Let's look at the characteristic features of this error. Let's be given a paragraph:

<p>text</p>

A certain programmer decided to obtain and change the text of this paragraph. To do this, he received a reference to this element in a variable by mistakenly using the wrong method:

let elem = document.querySelectorAll('p');

The programmer then tried to change the text of the paragraph. However, the text of the paragraph has not changed:

elem.textContent = '!!!'; // не сработало

However, no error appeared in the console. The fact is that a valid operation was performed, but not on one element, but on a group. But you can’t just read or change the text of a group of elements - only by going through them in a loop and accessing each element separately.

How to detect an error

Let's see how a programmer should act to detect this error. In fact, with some experience, it is easy to detect by its characteristic features.

Let this not happen, however. Then the first action of the programmer when searching for errors should be to output variable values ​​to the console. In his case there is only one variable -elem. You need to display the value of this variable and see what lies in it:

let elem = document.querySelectorAll('p'); console.log(elem); // выведет массив, а не один элемент

In the console you can immediately see that the variable contains not one element, but an array. This immediately gives an obvious clue that there is an error in the method for obtaining the element.
