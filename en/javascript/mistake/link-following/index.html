Error following link in JavaScript

Let's look at the error associated with an unexpected link click. Let the following link be given as an example:

<a href="">link</a>

Let's get our link into a variable:

let link = document.querySelector('a');

Let's attach a click handler to our link:

link.addEventListener('click', function() { console.log(this.textContent); });

And here a surprise awaits us. The code is written correctly, however, the output does not appear in the console. The fact is that when clicked, a link is followed. Since in the attributehrefnothing is specified, the link simply leads to the current page.

This means that clicking on the link causes the page to reload. That is, our data is output to the console, then the page is reloaded and the console becomes empty.

If you look closely, you can see how, at the moment of clicking, the data appears in the console for a moment and then disappears. This is a characteristic sign of this error.

Let's see how we can solve this problem.

Solution one

INhreflinks need to be grilled. This will cause the link to point to a specific location on the current page and will not reload when you click on the link. Let's do this:

<a href="#">link</a>

Solution two

A more advanced way is to override the default action using the methodpreventDefault:

link.addEventListener('click', function(event) { console.log(this.textContent); event.preventDefault(); });
