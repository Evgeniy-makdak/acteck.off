Error while iterating over an array with a for-in loop in JavaScript

To iterate through an array you should use a loopfor-of. Sometimes, however, beginners mistakenly try to apply a loopfor-in. Let's look at the characteristic symptoms of this error.

Let's say we have an array:

let arr = ['a', 'b', 'c'];

Let's iterate through the elements of this array and write them in one line.

A certain programmer had already solved this problem, but mistakenly used a loopfor-in. Let's see what he came up with:

let res = ''; for (let elem in arr) { res += elem; } console.log(res); // выведет '012'

So the result is the line'012', not'abc'. Let's figure out why this happened.

Why did the array go through the loop?for-in? The fact is that an array in JavaScript is a special case of an object. Therefore, in fact, in this regard, this is correct code. But you shouldn't do that.

The point is thatfor-inunder some conditions, it may catch something unnecessary, and in addition to the array elements, you will get something else parasitic during the search. Iterate over arrays only throughfor-of.

Let's now figure out why we see such a strange result. The point is that when trying throughfor-ininto a variableelemin fact, the keys will be captured, despite the confusing name of this variable.

Let's fix the mistake and write the correct code:

let arr = ['a', 'b', 'c']; let res = ''; for (let elem of arr) { res += elem; } console.log(res); // выведет 'abc'
