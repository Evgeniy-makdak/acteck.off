This error in JavaScript arrow handler

Recently, it has become fashionable to use arrow functions everywhere instead of regular ones. These functions, however, have a catch - they don't savethis. Let's see what problem you might encounter because of this.

Let's say we have a button:

<button>text</button>

Let's get a link to this button in a variable:

let button = document.querySelector('button');

Let's attach an event handler to the button using the arrow function:

button.addEventListener('click', () => { console.log(this.textContent); });

This is where a surprise awaits us:thisin an arrow function will not refer to the element in which the event occurred.

Let's see what we can do about this.

Solution one

You can refuse to usethisand use the variable to which the event is bound:

button.addEventListener('click', () => { console.log(button.textContent); });

Solution two

You can get the element that the event is attached to viaevent.target:

button.addEventListener('click', (event) => { console.log(event.target.textContent); });

Solution three

You can abandon the arrow function and use the usual one:

button.addEventListener('click', function() { console.log(this.textContent); });
