Error not following link in JavaScript

Let us have a link:

<a href="">link</a>

Let's get our link into a variable:

let link = document.querySelector('a');

Let's attach a click handler to our link. Let's cancel the link to avoid reloading the page:

link.addEventListener('click', function(event) { console.log(this.textContent); event.preventDefault(); });

It seems that we have completely protected ourselves from unexpected behavior. Actually this is not true. The fact is that if a JavaScript syntax error occurs, the code simply will not be executed, the link will not be blocked, and the link will not be followed.

In this case, we will not see either the result of executing the code or an error in the console, because the page will reload. Let's make a deliberate mistake in the code and see it in practice:

link.addEventListener('click', function(event) { thiss.textContent = 'text'; // синтаксическая ошибка event.preventDefault(); });

This problem has a characteristic symptom: if we look at the console at the moment we click on the link, we will momentarily see a red error in the console, which will disappear almost immediately.

Of course, we won’t have time to read the text of the error, which makes finding it much more difficult. There is, however, a trick. Needed inhrefplease put a grill. In this case, following the link will not happen and we will see the error thrown in the console:

<a href="#">link</a>
