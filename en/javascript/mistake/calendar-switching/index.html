Error switching calendar in JavaScript

Let's say we want to make arrows to switch the calendar. In this case, a number of typical mistakes can be made. Let's look at them.

Let us have toggle buttons and an element in which, for simplicity, we will display not a calendar, but simply the year and the name of the month:

<button id="prev">prev</button> <div id="elem"></div> <button id="next">next</button>

Let's get links to our elements:

let elem = document.querySelector('#elem'); let prev = document.querySelector('#prev'); let next = document.querySelector('#next');

Let us also have a function that takes the month number as a parameter and returns its name:

function getMonthName(month) { let names = [ 'янв', 'фев', 'мар', 'апр', 'май', 'июн', 'июл', 'авг', 'сен', 'окт', 'ноя', 'дек' ]; return names[month]; }

Let's get the current year and month after loading the page:

let date = new Date; let year = date.getFullYear(); let month = date.getMonth();

Let's display the year and month name in our element (imitation of a calendar):

elem.textContent = year + ' ' + getMonthName(month);

Let's now attach click handlers to the switches:

prev.addEventListener('click', function() { // показываем предыдущий месяц }); next.addEventListener('click', function() { // показываем следующий месяц });

Let's now consider what mistakes can be made when implementing switches.

Error one

Let some programmer implement buttons for switching months:

next.addEventListener('click', function() { elem.textContent = year + ' ' + getMonthName(month + 1); }); prev.addEventListener('click', function() { elem.textContent = year + ' ' + getMonthName(month - 1); });

This programmer, however, made a mistake that causes the month to switch only on the first click, but not on subsequent ones.

The point is that the variablemonthdoes not change inside functions - one is simply added to its contents and the result is sent to the functiongetMonthName.

Since our variable has not changed, the next time we click it will still contain the current month - and nothing will change.

Let's fix the problem:

next.addEventListener('click', function() { month = month + 1; elem.textContent = year + ' ' + getMonthName(month); }); prev.addEventListener('click', function() { month = month - 1; elem.textContent = year + ' ' + getMonthName(month); });

You can simplify:

next.addEventListener('click', function() { elem.textContent = year + ' ' + getMonthName(++month); }); prev.addEventListener('click', function() { elem.textContent = year + ' ' + getMonthName(--month); });

Error two

It is often forgotten that the month must vary within a certain range - from0to11. When we reach the range limit, we must change the year (forward or backward) and reset the month to its initial value.

Let's do this:

next.addEventListener('click', function() { if (month === 11) { month = 0; year++; } else { month++; } elem.textContent = year + ' ' + getMonthName(month); }); prev.addEventListener('click', function() { if (month === 0) { month = 11; year--; } else { month--; } elem.textContent = year + ' ' + getMonthName(month); });
