Master branch in Git

By default, the name of the master branch in Git ismaster. Technically this is not a special branch - it is exactly the same as all other branches. However, often this branch is used as the main branch of the project, and the remaining branches are made auxiliary, creating them to solve specific problems, and then deleting them.

Lately the name has often been replacedmasteronmain, renaming the master branch of the project. The fact is that the word “master” is translated as “master” and this is no longer politically correct.
