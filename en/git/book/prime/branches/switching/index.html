Switching branches in Git

Let's now learn how to switch between branches. To do this, use the commandcheckout, followed by the name of the branch. Let's switch to the branch we created in the previous lessontest:

git checkout test

You can also use a synonymswitch:

git switch test

After executing any of the above two commands, a message will appear in the terminal about moving to the specified branch:

Switched to branch 'test'

When switching from branch to branch,files in the Git working directory change too. That is, the files of one branch will physically disappear and the files of another branch will appear. Please note this as it is not very obvious to those new to Git.

It turns out that in Git, all changes in branches are isolated from each other and you can easily and quickly create several branches of a project without introducing confusion into the original version.

Switch to a new branch. While in this branch, make changes to the file, index them and commit. Then switch to the main branch and check the contents of the file - it should return to the original version.

Switch to a new branch. While in this branch, create a new file, index it and commit. Then switch to the master branch and make sure that the new file disappears. Switch back to the new branch and make sure your file is back.
