More complex rebasing of commits in Git

In Git, you can rebase up to three branches. Let us have two branchesmember1Andmember2, which have different commits.

While working on the project, we needed to insert commitscommit3, commit4from the threadmember2to the threadmember1and apply them on the branchmaster. To do this, run the following command:

git rebase --onto master member1 member2

This command tells Git to switch to a branchmember2, find commits in it that are not in the branchmember1and merge only these missing commits to the branchmaster.

Now let's perform a rewind merge:

git switch master git merge member2

One thing to remember is that you cannot rebase commits that are in an external repository. All team members pull commits from it to complete their task, but if branches are rebased, all members will have to merge the commits back into their branches, which can cause further confusion.
