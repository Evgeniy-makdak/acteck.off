Setting up a username and email in Git

When several people are working on a project, it is convenient to be able to understand who exactly wrote a given piece of code. Therefore, Git automatically leaves the name of the author of a piece of code and his email for communication.

Therefore, the author’s name and email should be configured initially. Let's see what commands this is done with.

If you want the username to apply to all projects you work on, add the option to the command--global:

git config --global user.name "MyUser" git config --global user.email MyUser@mail.com

If in any specific project you need to change the username or his email, then you should go through the terminal to a specific directory and enter the command there without the option--global:

git config user.name "MyUser" git config user.email MyUser@mail.com
