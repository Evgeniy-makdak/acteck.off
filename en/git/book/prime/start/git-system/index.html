Introduction to Git for Beginners

Let's see how the Git system works. This system takes on most of the work. It makes backup copies itself, copying only changes and not all files. It allows you to easily switch between versions of a project, developing both the first version and the second in the same folder. It makes it easy to easily share changes with colleagues. It makes it easy to develop features in parallel, then merge them with the main project.

Let's learn Git terminology. The backup folder in Git is calledrepository. Technically, it does not store all the files, but simply their changes. Based on changes, you can always restore files to any state. And storing only changes helps prevent the rapid growth of the repository size.

When we want to save something to the backup folder, in Git terms this is called docommit. In the commit we indicate a comment to our save, so that later we can easily track what we did and, if necessary, roll back.

For a situation when we want to work on a specific feature of the project in a separate copy of the project, we came up withbranches. Git allows you to switch between development branches, and Git will change files directly in the project folder. That is, when we switch from branch to branch, Git will remove some files and show others right in the project folder.

After working on a feature in a separate branch, we can easily merge the finished code into our main branch. In this case, Git will keep track of files that belong to different branches at the same time. And, if the changes in them conflict, then during the merge Git will indicate this to us and ask us to resolve the conflict. Of course, Git will not resolve the code conflict for us, but will simply point it out. We can fix the code and then merge.

Git allows you to keep your project's main repository online. There are special services for this, for example, GitHub. This means that all project developers will submit their changes to one place. And this place, thanks to its placement on the Internet, will always be available. Other developers can also easily get changes by accessing this repository.

At the same time, the Internet is not needed at all for development. You can develop on your computer in a place without the Internet: on an airplane, on a train, in the country. And then, when connected to the Internet, upload your changes to a common repository.

In the following lessons we will take a closer look at the basic concepts of the Git system and learn how to work with it. It is assumed that you already know how to work with the command line. If not, then check it out first.command line tutorialand only then move on to the next lessons.
