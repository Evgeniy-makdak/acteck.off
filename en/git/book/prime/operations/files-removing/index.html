Deleting files in Git

When working with Git, you cannot manually delete files from the project folder. If we try to do this, then after checking with the commandstatus, it will be displayed in the category'Changes not staged for commit'as a modified but not indexed file. To be sure to delete a file we don’t need, we should use the commandgit rm:

git rm text.txt

If, after executing this command, you check the status of the files in the repository, the following message will be displayed:

deleted: text.txt

Correctly delete the file from your project folder.
