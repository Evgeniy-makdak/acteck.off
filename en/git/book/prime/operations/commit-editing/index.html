Editing a commit in Git

When working on a project in Git, sometimes situations arise when a commit is submitted incorrectly, for example, prematurely or with the wrong comment. Git has the ability to submit a revised commit, which will be accepted in place of the previous commit.

To do this, first make edits, index them, and then run the commandcommit, but with a flag--amend:

git commit --amend

Intentionally submit a bad commit, and then submit the correct one.
