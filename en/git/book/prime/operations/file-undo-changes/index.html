Reverting changes to a file in Git

In the case when we need to undo changes to a file and return it to its original form, we need to use the commandrestore. Let's make changes to the file firstfile.txtand track its status:

git status

In the message we will see that the filefile.txtreceived the statusmodified.

Now let's return the file to its original state with the commandrestore:

git restore file.txt

After checking the status, we will see a message stating that there is nothing to commit. If you check the contents of the file, it will indeed revert to the version before the changes. You need to be careful with this command because Git replaces the file with its previous version and any changes in between are erased.
