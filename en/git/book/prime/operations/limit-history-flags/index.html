Flags for limiting the output of commit history in Git

You can also limit the output of commit history by taking into account the time when they were made. For example, flag--since=1daywill display the commits made during1day. For a complete list of flags for limiting the output of commits, see below:

Flag Description
-(n) Shows the latest commits based on the specified number.
--since, --after* Displays commits made after the specified date.
--until, --before* Commits made before the specified date are displayed.
--author* Displays commits for files with the specified author.
--committer* Lists commits made by the specified committer, i.e. the user making the changes. The author of the file and the committer may be different users.
grep Finds and displays commits whose message or note contains the specified string.

For some restrictive flags, the mandatory requirement is to add a sign after their name=and a parameter by which we limit the output of the commit history. These flags are marked in the table*.

Example. Outputting the latest commits

Let's print the last two commits to the console:

git log -2

After executing the command we will see:

commit 33498ca885348fba6060ad4c459963d9e756c8b6 (HEAD -> master) Author: root <root@node.com> Date: Tue Sep 12 10:52:30 2023 +0100 Story1 commit bb6945ff7c73c88981a0b2523c32e30f472fd918 Author: root <root@node.com> Date: Tue Sep 12 10:47:27 2023 +0100 text

Example. Output commits made after a specified date

Let's output to the console the commits made after11September2023year:

git log --since="2023-09-11"

After executing the command we will see:

commit 33498ca885348fba6060ad4c459963d9e756c8b6 (HEAD -> master) Author: root <root@node.com> Date: Tue Sep 12 10:52:30 2023 +0100 Story1 commit bb6945ff7c73c88981a0b2523c32e30f472fd918 Author: root <root@node.com> Date: Tue Sep 12 10:47:27 2023 +0100 text

Example. Output commits made before a specified date

Now let's output to the console the commits made before11September2023year:

git log --until="2023-09-11"

After executing the command we will see:

commit a99c51a34d6a89be9c56127e18a1e574d6188d61 Author: root <root@node.com> Date: Fri Jun 16 15:20:43 2023 +0100 initial commit

Example. Output commits made by the specified file author

Let's output to the console all commits for files created by the user'user':

git log --author="user"

After executing the command we will see:

commit 9c51a34d188d61a6a89be9c56127e18a1e574d69 Author: user <user@node.com> Date: Fri Jun 10 10:10:23 2022 +0100 initial commit

Example. Output commits made by the specified committer

Now let's output to the console all the commits made by the user'root':

git log --committer="root"

After executing the command we will see:

commit 33498ca885348fba6060ad4c459963d9e756c8b6 (HEAD -> master) Author: root <root@node.com> Date: Tue Sep 12 10:52:30 2023 +0100 Story1 commit bb6945ff7c73c88981a0b2523c32e30f472fd918 Author: root <root@node.com> Date: Tue Sep 12 10:47:27 2023 +0100 text commit a99c51a34d6a89be9c56127e18a1e574d6188d61 Author: root <root@node.com> Date: Fri Jun 16 15:20:43 2023 +0100 initial commit

Example. Output of commits, specified note

Let's print to the console all commits that contain the line'text':

git log --grep "text"

After executing the command we will see:

commit bb6945ff7c73c88981a0b2523c32e30f472fd918 Author: root <root@node.com> Date: Tue Sep 12 10:47:27 2023 +0100 text
