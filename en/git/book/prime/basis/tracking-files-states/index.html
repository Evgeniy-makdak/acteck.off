Git tracked file states

The states of the monitored files that we saw in previous lessons have names. The file can take three states:modified(changed)staged(indexed) andcommitted(fixed).

A file becomes modified when changes were made to it, but they were not saved. A file becomes indexed after it has been added for subsequent inclusion in a commit. A committed state means that the commit has already been saved and included in your repository.

What is the state of the file if, when checking the status, we received the following:

Changes to be committed: new file: index.html

What is the state of the file if, when checking the status, we received the following:

Changes to be committed: modified: index.html

What is the state of the file if, when checking the status, we received the following:

Changes not staged for commit: modified: index.html

What is the state of the file if, when checking the status, we received the following:

nothing to commit, working tree clean
