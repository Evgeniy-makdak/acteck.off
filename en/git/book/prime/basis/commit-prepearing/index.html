Preparing files for commit in Git

Tracking files is only half the battle. We also have to tell Git at what point to remember the state of the file for a later commit. This is also done using a cross-functional teamadd. Now we will discuss this delicate point in more detail.

Let's create a fileindex.html. Initially it will not be tracked:

Untracked files: index.html

Let's make it trackable:

git add index.html

Now it goes into the list for being prepared for committing as a new file:

Changes to be committed: new file: index.html

Let's now change the text of the file. In this case, the file will end up in the section not prepared for commit, but already modified:

Changes not staged for commit: modified: index.html

Moreover, this file will be in two sections at once:

Changes to be committed: new file: index.html Changes not staged for commit: modified: index.html

How so? The thing is that Git stores (indexes) exactly the current state of the file. The state refers to the current text of the file, whether it was created or deleted. If the file is changed, then its changes will not be saved (will not be indexed) in Git. To do this you need to run the command againadd. In fact this command should be considered asadding the current state of the file to a new commit.

This is why we see our file in several lists - these are different versions of the file.

Let's get Git to index the current state of our fileindex.html, to which we make changes. Let's run the command for himadd:

git add index.html

After this, the current version will be ready to commit and will be placed in the appropriate section:

Changes to be committed: modified: index.html

And this must be done every time. When you make changes to a file, you should index them for the next commit using the commandadd. Of course, if you want to take these changes into the next commit.

Create a new file. Index your file. Check the status.

Make a change to your file. Check the status. Index your file. Check the status.

Make the change to your file again. Check the status. Index your file. Check the status.
