File tracking in Git

In the previous tutorial, we initialized Git in our project. And now we can use Git commands to save the project, or, in the correct terminology, docommits.

However, everything is not so simple. The fact is that in every project there are files whose backup copies are not needed. For example, files with cache. Therefore, in the Git system we need to explicitly indicate which files we want to save.

There are two types of files in Git:untraceable (untracked) Andtrackable (tracked).

Tracked files are files that Git monitors for changes. If these files change, Git will notify us about it.

Untracked files are files for which Git does not keep track of changes. Initially, all files will be exactly like this. And for Git to start monitoring files, you need to specifically ask it to do so. Let's figure out how to do this.

First, we need to learn how to look at the states of files. There is a special command in Git for thisstatus:

git status

Let there be several files in your project folder:

  • index.html
  • styles.css

These files are initially untraceable. In this case the commandstatuswill list these files like this:

Untracked files: index.html styles.css

Check the status of the files in your project.

Make another file in the project folder. Check the status. Make sure the new file appears in the untracked list.
