The term "remote" repositories in Git

The name "external" repositories has some nuance. The fact is that in Russian literature external repositories are called “remote” - from the word “far”. This is how the translators translated the English word "remote".

But we also use the word “deleted” for another concept - “that which was deleted.” The result is confusion. Particularly pleasing are expressions likedeleting remote repositories. The reader immediately has a question: how can we delete remote repositories if they are already deleted, that is, deleted?

I don’t know what these bad people were thinking when they translated the word “remote” in such an original way, but I send them rays of hatred and encourage everyone to use the termexternalrepositories. This is what is used throughout the textbook.

But keep in mind that the word “deleted” is used everywhere, both in textbooks and on forums. In conversations and correspondence, I recommend that you use the word "external" and refer everyone to this article.
