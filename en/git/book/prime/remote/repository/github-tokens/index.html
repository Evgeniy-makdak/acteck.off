Creating a GitHub token

In the previous tutorial, we created an external repository on GitHub. In future work, we will access this repository through the terminal of our local computer.

When accessing through the terminal, we will be asked for a login and password. The login will be your GitHub login, but you will need to specify a special token as the password. Let's create this token on GitHub.

To do this, go to the pagecreating a token.

In the settings, you can select the validity period of the token, as well as access rights. For our purposes, it will be enough to give access rights in the first block with the namerepo.

Please note that the default token expiration date is30days. After that, it will stop working, and you will be told that the password is incorrect. Be mentally prepared for this. Or just set the token to expire forever.
