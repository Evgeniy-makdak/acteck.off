Pushing data to an external repository in Git

Let's prepare and push our first commit to our test external repository on GitHub.

Let's make a filetest.txtand index it:

git add test.txt

Let's commit:

git commit -m "first commit"

Now let’s send our commit to an external repository. To do this you need to use the commandpush, in which after the flag-useparated by a space, the name of the remote repository and the name of the branch that we are sending are indicated:

git push -u origin master

After executing the command, the terminal will ask you to enter your GitHub login and the token that we created in previous lessons (you only need to enter it the first time). After entering the token, all your commits that are not already on the server will be pushed.

If you then go to your project on GitHub, you will see files with your code there.

Make the first commit of your project and push it to an external repository. Search GitHub for your project and see if the changes appear there.

Make changes to the project. Make a few more commits. Push your changes to an external repository.
