Fetching changes from external repositories in Git

So, in previous lessons we learned how to send data to an external repository. Let us now imagine a situation where the data in it has changed. This situation occurs when several people are working on a project and each of them pushes data to this common repository. Or there may be a situation where you work on different computers, for example, at home and at work. At work you send changes, but at home you want to receive them on your home computer.

Let's simulate this situation. To do this, go to the project files on the GitHub website and edit a file by making changes to it. Let's get these changes onto our local computer.

This is done using the commandfetchfollowed by the name of the external repository:

git fetch origin

This command will copy all changes made to the external repository since you last accessed it. In this case, the changes will not appear immediately in our files, but will end up in a special branch. The name of this branch will be constructed according to the following principle: the name of the external repository, and then separated by a slash the name of the branch. In our case it will beorigin/master.

In order for the changes to appear in our project files, we need to merge the changes into our branch. To do this, while in the branchmaster, execute the merge command:

git merge origin/master

On GitHub, manually make changes to your project. Pull these changes into your local repository.
