Generic Type Variables in TypeScript

When we create a generic function, TypeScript forces us to use generic variables correctly in its body. Therefore, we need to consider the behavior of each parameter passed to a function in terms of all the types available to us.

Let's create a function to determine the length of a generic variable:

function myFunc <T> (data: T): T { console.log(data.length); // ошибка return data; }

The error occurs because not all types have a length. For example, this action is not possible for Boolean values. However, if we qualify the type and specify it as an array, the code will work correctly. After all, every array, no matter what type it is, has a length:

function myFunc <T> (data: T[]): T[] { console.log(data.length); // 3 return data; } console.log(myFunc([1, 3, 5])); // [ 1, 3, 5 ]
