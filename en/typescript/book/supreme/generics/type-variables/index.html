Type Variables in TypeScript

To better understand the concept of generics, let's create our first function -identity function. Such a function returns exactly what was passed to it - with the same type.

Let's first write the variants of our function separately.

Let's write a function that takes a number as a parameter and returns the number:

function func(data: number): number { return data; }

Now let's write a function that takes a string as a parameter and returns a string:

function func(data: string): string { return data; }

Let us now want to make sure that all this is done by one function. That is, we want to fulfillgeneralization.

For this we usegeneric variable, which stores the data type. Such a variable is declared in angle brackets before the parentheses containing the function parameters. Let's declare such a variable by giving it a nameT(any is possible):

function func <T>(тут параметры) { return data; }

After this we will have a variableT, which can be used to put the type of the parameter into it, and then specify it as the return value.

Let's indicate the type of our parameter in the form of our variable:

function func <T>(data: T) { return data; }

It turns out that the parameter can be passed in any type, and this type will be saved in our variableT.

Now we use the value of the variableTas an indication of the function result:

function func <T>(data: T): T { return data; }

Now it turns out that the function result type will be obtained from the parameter type. Let's check it out.

Let's call the function with a number:

console.log( func(3) ); // выведет 3

Let's call the function with the line:

console.log( func('x') ); // выведет 'x'
