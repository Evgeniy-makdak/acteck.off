Controlling Object Value Types in TypeScript

When an object is declared, TypeScript remembers the data type of all its elements and then ensures that those types do not change.

Let's look at the example of our object with a user. Let's declare it:

let user = {name: 'john', age: 30};

After the declaration, TypeScript analyzed the data type of each value and remembered that in the field'name'- line, and in the field'age'- number.

Now an attempt to write a value of a different type into the field will result in an error.

Example:

user.name = 123; // ошибка

Example:

user.age = 'eee'; // ошибка

Example:

user.age = '30'; // ошибка
