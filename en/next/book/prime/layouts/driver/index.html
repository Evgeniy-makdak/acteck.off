NextJS framework as a website engine

In ancient times, when the Internet was just beginning, websites were developed in pure HTML. This is, however, somewhat inconvenient. Therefore, special programs appeared, written in one of the backing languages ​​(PHP, NodeJS, etc.), simplifying website development. The program that supports the site is calledenginesite.

Let's discuss what the problems are with pure HTML sites, how engines help solve these problems, and what NextJS has to do with it.

Suppose you have a multi-page website in pure HTML. On each page of the site you will have the same parts (for example, header, footer, sidebars) and different parts (page content, its title).

Let's say you need to make some change, say, to the website header. In this case, you need to make changes to all files with this header. This, of course, is not very convenient.

That’s why modern websites run on engines. The engines use a special approach. The essence of this approach is that one file is created with the general layout of the site. This file will contain the general parts of the site.

Depending on the requested URL, site content corresponding to that URL will be inserted into a special place in the layout.

In addition to the content on the page, the title usually changes. In this case, the content and title are inserted into different places in the layout. This is also provided by the site engine.

If SEO is important to us, then for each page of the site we should have its own meta description. It should also be inserted into its place on the layout.

The NextJS framework works as described. That is, it is the engine of our site. Accordingly, we have a special file somewhere that contains the general layout of the site. We will study it in this section of the textbook.
