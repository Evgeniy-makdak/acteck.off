Invalid parameter error in NextJS

In previous lessons we gave404error when the route for a given URL was not found. However, in the case of a dynamic route, there may be a situation where the route exists, but the parameter value is incorrect.

Let's look at an example. Let us have a route like this:/user/:id.

Let us also have an object with users:

let users = { 1: 'user1', 2: 'user2', 3: 'user3', 4: 'user4', 5: 'user5', };

Suppose we want to show the user by his ID:

export default function User({ params }) { return <p> { users[params.id] } </p>; }

Obviously, the ID in the URL can not be any, but only one of the keys of our object. Attempting to contact a non-existent address/user/6should result in an error being thrown. Let's fix our component code and display the error text:

export default function User({ params }) { if (users[params.id]) { return <p> { users[params.id] } </p>; } else { return <p> user not found </p>; } }

This, however, is not enough. Indeed, in this case we are accessing a non-existent page, and the readout must be given HTTP status404. This is done using a special functionnotFound, built into NextJS.

Let's import this function:

import { notFound } from 'next/navigation';

Now we will call it in the case when the parameter value is incorrect:

export default function User({ params }) { if (users[params.id]) { return <p> { users[params.id] } </p>; } else { return notFound(); } }

The complete code for our component will look like this:

import { notFound } from 'next/navigation'; let users = { 1: 'user1', 2: 'user2', 3: 'user3', 4: 'user4', 5: 'user5', }; export default function User({ params }) { if (users[params.id]) { return <p> { users[params.id] } </p>; } else { return notFound(); } }
