Types of React components in NextJS

In classic React, rendering happens on the client side, which creates an SEO problem. Therefore, the React developers decided to make React support not only client rendering, but also server rendering.

To do this, two types of components were introduced into React: client-side (RCC) and server-side (RSC).

Server-side components are rendered on the server and the rendering result is sent to the browser as HTML code. Client components are rendered directly in the browser.

In server components we can use JSX, but we cannot use DOM hooks (such asuseState), as well as access to those JavaScript commands that are only available in the browser. Instead, we can use NodeJS, for example, to read the contents of files, or access a database.

Client components are actually the components we are used to when learning React. In them we can use both JSX and hooks to work with the DOM.

To declare a component as a client component, you need to write a special directive at the beginning of the component file'use client'. Here is an example of the client component code:

'use client'; import { useState } from 'react'; export default function Test() { let [name, setName] = useState('user'); return ( <div> <p>hello, {name}!</p> </div> ); }

To declare a component as a server component, you need to write a special directive at the beginning of the component file'use server'. Here is an example of the server component code:

'use server'; export default function Test() { let name = 'user'; return ( <div> <p>hello, {name}!</p> </div> ); }

NextJS supports both types of components. By default, all components are server-side and the directive'use server'You don't have to write in them.

We'll start learning NextJS with server components. And then we will add client ones.
