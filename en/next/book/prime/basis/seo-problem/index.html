SEO problem and NextJS framework

When we develop a website, it is important for us to understand whether we will promote this site in search engines. By search engines we mean specialized sites like Google or Yandex, in which the user searches for the information he needs.

If our site is shown in the first positions for a specific request, this will provide it with a continuous flow of visitors. And we won’t have to pay anything for these visitors, unlike visitors who come from advertising.

And here a problem awaits us: sites developed in React will not be promoted in search engines. The fact is that the content of the pages of such a site is generated dynamically in the browser via JavaScript. Search engines do not yet know how to process such pages. Accordingly, such a site will not be able to fully display for search queries.

In order for search engines to fully work with our site, the texts of its pages must be created on the server side and sent to the browser in the form of ready-made HTML code.

The NextJS framework solves this problem. It allows you to write JSX code on the server side. The server converts this JSX into normal HTML and serves it to the browser. Thus, the HTML code necessary for search engines to work correctly with our site will be sent to the client side.

We will study exactly how this is done in this tutorial.

It should be said that search engines are already learning to recognize content generated in JavaScript. When they fully learn to do this, the entire development of websites will change radically. And frameworks like NextJS will either not be needed at all, or will become completely different in their principles.

So, let's go :)
