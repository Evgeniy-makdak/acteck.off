Checking content in the NextJS framework

In previous lessons we tried to make two types of components - server and client. Let's make sure that the SEO problem is solved in the back-end components. That is, we must check that the layout with the text of our component arrives in the browser. Let's do this.

Let our server component look like this:

export default function Test() { let name = 'user'; return <h1>hello, {name}!</h1>; }

Open the page with this component in your browser. Then clickCtrl + U. This key combination will open the source HTML code of the page. This is the code that comes from the server. That is, before applying browser JavaScript to it. This is exactly how search engines see our page. So, open the source code and study it. There you will find the text of our component.

Let's now make a client component as an example:

'use client'; import { useState } from 'react'; export default function Test() { let [name, setName] = useState('user'); return <h1>hello, {name}!</h1>; }

Click againCtrl + Uand study the source code. You will not find the text of our component, as its text is generated dynamically in the browser.

This is the main difference between server and client components.
