The basic principle of operation of the NextJS framework

In previous lessons, we installed the NextJS framework and became familiar with its basic structure. It's time to do the main work.

The main part of NextJS isrouting. Routing works like this: when a user types a URL into the address bar of the browser, the NextJS framework will send a specific file corresponding to this URL to the browser.

Routing in NextJS is designed in a special way. Its essence is that each requested URL corresponds to a certain folder insidesrc/app.

In this case, inside the folder there should be a file calledpage.jsx. It is this file that will be sent to the browser. Moreover, in this file we will write code in JSX, and the finished HTML code of the page will be sent to the browser.

Let's look at an example. Let's say we want to by URL/test/some text was given.

Let's create the corresponding folder:src/app/test. Inside this folder we will create a filepage.jsxwith the following content:

export default function Test() { return <h1>hello, user!</h1>; }

Open our URL in your browser:/test. And you will see the text we wrote inside the JSX file.

Try changing the text and saving the file. After this, the text will automatically change in the browser. This is how NextJS works in development mode.

Make sure that when you contact the address/aboutA message with your first and last name was displayed in the browser.
