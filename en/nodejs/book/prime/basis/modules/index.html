ES Modules in NodeJS

By default, NodeJS uses modules according to the CommonJS standard. This is how it happened historically. However, starting from version13.2.0Full support for ES modules has appeared. Since ES modules are the official module standard, I recommend that you use it.

To switch to ES modules you need to place the file in the folder with the file you are runningpackage.jsonwith the following content:

{ "type": "module" }

Example

Let's make a module as an examplemathfor mathematical operations. Let's place its code in a filemath.js:

function square(num) { return num * num; } function cube(num) { return num * num * num; }

Let's export our functions:

export function square(num) { return num * num; } export function cube(num) { return num * num * num; }

Now import this module into the fileindex.js:

import { square, cube } from './math.js';

Let's use the functions of our module:

let res = square(2) + cube(3); console.log(res);

Built-in modules

Built-in modules are imported in the same way. For example, let's import the modulefsto work with the file system:

import fs from 'fs';

It is not necessary to import all module functions. We can import only those we need:

import { open, read, close } from 'fs';

Connecting installed via npm

Modules installed via npm are imported in the same way. Let's install the library as an exampleunderscore:

npm install underscore

Let's import it:

import _ from 'underscore';

Currently, in the documentation of almost all libraries, the connection instructions are described in the CommonJS style. You can, however, convert it yourself to ES and connect it to your project.

Install the librarylodash. Connect it to your project and use several methods from this library.
