Understanding how a server works on NodeJS

You should understand that our server, once started, processes requests from all users of our site. This is how the NodeJS approach differs from the PHP approach in which the script is launched, executed and dies.

In the case of NodeJS, it turns out that our server’s callback function is called for every request. This means that the external variables of this function will be common to all requests.

In this way, you can make, for example, a counter of requests to our server and give its value to each request:

let i = 0; http.createServer((request, response) => { response.setHeader('Content-Type', 'text/html'); response.statusCode = 200; response.write(String(++i)); response.end(); }).listen(3000);

Let our counter initially have a value of 100. Each request, decrease this value by one and send the new value to the browser. As soon as the counter reaches zero, print a message about this as a result of the query.
