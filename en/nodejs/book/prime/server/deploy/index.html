Creating an HTTP server on NodeJS

Let's now deploy an HTTP server on NodeJS. This server will accept requests from the browser and respond with the HTML code of the pages.

First we need to import the modulehttp:

import http from 'http';

After this we can start our server using the methodcreateServer:

http.createServer();

This method takes a callback as a parameter. This callback will be executed every time someone accesses our site through a browser:

http.createServer(() => { });

In our callback we must define two parameters. The first parameter will contain an object with the user’s request data, and the second parameter will contain an object with which we will form our response sent to the browser:

http.createServer((request, response) => { });

Using the methodwritewe can tell what text to send to the browser:

http.createServer((request, response) => { response.write('text1'); response.write('text2'); response.write('text3'); response.end(); });

Using the methodendwe command to complete our response and send it to the browser:

http.createServer((request, response) => { response.write('text1'); response.write('text2'); response.write('text3'); response.end(); });

After starting the server, we must say on which port our server will listen for requests from the browser. This is done using the methodlisten. Let's specify the traditional port3000:

http.createServer((request, response) => { response.write('text1'); response.write('text2'); response.write('text3'); response.end(); }).listen(3000);

Now we can access our server through the browser by typinghttp://localhost:3000, where after the colon the port we specified is indicated.

Stopping the server

In previous lessons, our scripts were processed immediately and after that the console was ready to enter new commands. This will not be the case with the server, since it is an executable process.

To complete such a process, you need to press the keys in the terminalCtrl + C. In this case, the server will stop and then it can be started again.
