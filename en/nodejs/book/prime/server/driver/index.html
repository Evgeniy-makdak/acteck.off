Website engine on NodeJS

Let's look at the structure of a typical website:

<!DOCTYPE html> <html> <head> <title>page title</title> </head> <body> <div id="wrapper"> <header> common header <nav> common menu </nav> </header> <main> page content </main> <aside> common sidebar </aside> <footer> common footer </footer> </div> </body> </html>

As you can see, a website page usually has a header, footer, sidebars and content.

Moreover, on a multi-page site, all pages of the site differ only in content and title, while the header, footer and sidebars remain the same.

This creates certain inconveniences in maintaining the site: if you have a hundred pages on your site, then they will have a hundred identical headers scattered across a hundred files. If you need to change something in the website header, you will have to do it a hundred times! This, of course, is not convenient.

Therefore, in modern website building a different approach has been adopted: creatingonea site template file containing all the same places, as well as places where changeable parts will be inserted, for example, title and content.

In this case, the tile and page content are stored in separate files. That is, it turns out that for each page of the site, only the changing title and content are stored in the file, and the general header, footer and sidebars are stored in a common file with the site template.

When a browser requests a site page, the server receives a file with the title and content at the requested URL, then takes the template file and inserts the tile and content into specially designated places in the template.

The result is a finished HTML page with all parts, which is sent to the browser.

Places for inserting changing parts in the template are indicated using the commands we created. For example, I can say that the place where the title is inserted is denoted as{% get title %}, and the place where the content is inserted is{% get content %}. With these commands, the template file might look like this:

<!DOCTYPE html> <html> <head> <title>{% get title %}</title> </head> <body> <div id="wrapper"> <header> common header </header> <main> {% get content %} </main> <aside> common sidebar </aside> <footer> common footer </footer> </div> </body> </html>

Implementation

So, let's implement the described. First, let's think about the file structure. Let it be in a folderpagethere are changing parts, with the title and content of one page in different files in the same folder. Let their folder address correspond to the page URL.

Let, for example, the address is requested/dir/name/. Then the title of this page will be in the file/page/dir/name/title.html, and the content is in the file/page/dir/name/content.html.

In a folderrootLet's place the site resources (for simplicity, we'll ignore them for now), and the site template in the filelayout.html.

Let's now write the code. When requested, our server must read the title file, the content file, and the template file. Then you need to change our special commands in the template text to the title text and content text. The resulting page can be sent to the browser.

Let's implement the described:

http.createServer(async (request, response) => { let lpath = 'layout.html'; let cpath = 'page' + request.url + 'content.html'; let tpath = 'page' + request.url + 'title.html'; let layout = await fs.promises.readFile(lpath, 'utf8'); let content = await fs.promises.readFile(cpath, 'utf8'); let title = await fs.promises.readFile(tpath, 'utf8'); layout = layout.replace(/\{% get content %\}/, content); layout = layout.replace(/\{% get title %\}/, title); response.writeHead(200, {'Content-Type': 'text/html'}); response.write(layout); response.end(); }).listen(3000);
