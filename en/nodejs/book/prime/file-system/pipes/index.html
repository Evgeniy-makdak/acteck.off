Working with Channels in NodeJS

The situation in which we need to read data from one thread and write it to another is quite common. To simplify this operation, we have inventedchannels (pipes).

Let's see how it's done. Let's say we have a read thread and a write thread:

let readableStream = fs.createReadStream('readme.txt', 'utf8'); let writeableStream = fs.createWriteStream('writeme.txt');

Let's make sure that the reading stream is immediately redirected to the writing stream. For this purpose, reading threads have a methodpipe:

readableStream.pipe(writeableStream);

Use a pipe to read a file and write its data to another file.

Chains

Methodspipecan be called in a chain one after another. Moreover, each method call in the chain allows you to perform certain operations on the data:

readableStream.pipe(операция).pipe(операция).pipe(операция);

For example, let's archive a file. To do this, we need to first read the file, then compress the data and finally write the compressed data to an archive file. The library built into NodeJS will help us with this.zlib:

import { createGzip } from 'zlib';

Now, using a chain of methods, we will first archive the reading stream, and then write to the writing stream:

readableStream.pipe(createGzip()).pipe(writeableStream);
