Synchronous reading and writing of files in NodeJS

In this lesson we will deal with synchronous work with files, as it is easier to understand.

Synchronous file reading

To read a file synchronously, use the methodreadFileSync. The first parameter it takes is the file name or path to the file, and the second is the file encoding, usually thisutf8.

Let there be a file in the folder with our scriptreadme.txt. Let's read its text and display it on the screen:

let text = fs.readFileSync('readme.txt', 'utf8'); console.log(text);

Make two files, the text of which will be some numbers. Write a script that will read numbers from files and print the sum of these numbers to the console.

Synchronous file recording

To synchronously write to a file, use the functionwriteFileSync. The first parameter it takes is the file name or path to it, and the second is the text that we want to write to this file:

fs.writeFileSync('readme.txt', 'text');

If the file already exists, its text will be overwritten. And if the file does not exist, it will be automatically created.

let obj = { 'file1.txt': 'text1', 'file2.txt': 'text2', 'file3.txt': 'text3', }

Synchronous file reading and writing

You can read data from a file, perform some operation on it, and then write it back.

For example, let's read text from a filereadme.txt, add a sign to the end!and write the changed text back to the file:

let text = fs.readFileSync('readme.txt', 'utf8'); fs.writeFileSync('readme.txt', text + '!');

Given a file, the text of which contains a certain number. Write a script that will read a number from a file, add one to it, and write the new number back to the file.

Dans3file with numbers. Write a script that reads numbers from files, finds their sum and writes it to a new file.

Exceptional situations

Various exception situations may occur when working with the file system. For example, a file may be missing, the hard drive may fail, the drive may be full, the drive may crash and not respond, you may not have permission to write to a particular folder, and so on.

Therefore, any work with the file system must be wrapped intry-catch:

try { let text = fs.readFileSync('readme.txt', 'utf8'); console.log(text); } catch (err) { console.log('при чтении файла возникла ошибка', err); }

Try reading a non-existent file. Make sure that this throws an exception. Modify your code to handle this situation.
