Passing data to the Handlebars view

You can pass data from a route to a view. To do this, the second parameter of the methodrenderyou need to pass an object with data:

app.get('/page/', function(req, res) { res.render('page', {text1: 'aaa', text2: 'bbb'}); });

As you can see, my object has two keys:text1Andtext2. This means that two variables with that name will appear in the view file (both content and layout). You can display the values ​​of these variables by writing their names in double curly braces.

Let's try it. Let's say we have the following content file:

<div> content </div>

Let's display our variables in this file:

<div> content {{text1}} {{text2}} </div>

You can display them in some tags:

<div> content <span>{{text1}}</span> <span>{{text2}}</span> </div>

You can also display them in tag attributes:

<div> content <input value="{{text1}}"> <img src="{{text2}}"> </div>

Tag Escaping

To display variables, you can use two or three curly braces. In the first case, tags will be escaped for security purposes, and in the second case, tags will be converted into commands for the browser.

Let's look at an example. Let us pass some variable containing tags inside:

app.get('/page/', function(req, res) { res.render('page', {text: '<b>aaa</b>'}); });

Let's print the contents of a variable using both commands:

<div> {{text}} </div> <div> {{{text}}} </div>

Run the given code in your browser and look at the difference in the given commands.
