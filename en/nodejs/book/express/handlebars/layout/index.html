General site layout in Handlebars in Express

The Handlebars template engine allows you to easily create one general site layout to which changing site content will be connected. Let's do this. To do this in the folderviews/layoutsplace the filemain.hbswith the following content:

<!DOCTYPE html> <html> <head> <title></title> </head> <body> <header> header </header> <main> {{{body}}} </main> <footer> footer </footer> </body> </html>

To folderviewsplace the filespage1.hbs, page2.hbs, page3.hbs. These will be the plug-in contents of the pages of our site.

Let's now write the routes for our files. To do this, instead of the methodsendlet's write a methodrender. This method will take the specified content file, take the site layout, and insert it into it instead of the command{{{body}}}page content. Moreover, in the methodrenderfile extensions do not need to be specified - and it is assumed that theyhbs.

So here are our routes:

app.get('/page/1/', function(req, res) { res.render('page1'); }); app.get('/page/2/', function(req, res) { res.render('page2'); }); app.get('/page/3/', function(req, res) { res.render('page2'); });

Let's rewrite our routes into one using the parameter:

app.get('/page/:page/', function(req, res) { res.render(req.params.page); });
