Route options in Express

Let's say we have the following set of routes:

app.get('/test/1/', function(req, res) { res.send('1'); }); app.get('/test/2/', function(req, res) { res.send('2'); }); app.get('/test/3/', function(req, res) { res.send('3'); });

It is easy to notice that these routes are built according to the same pattern: first comes/test/, and then some number. Express allows you to group such routes usingparameters.

Parameters allow you to put part of an address into a variable. In this case, it turns out that our route will process not one address, but a whole group of addresses. For example, let's specify a parameter with the namenum:

app.get('/test/:num/', function(req, res) { });

The passed parameters are contained in the propertyparamsrequest object. Let's print the value of our parameter to the console:

app.get('/test/:num/', function(req, res) { console.log(req.params.num); });

Now let's send the resulting parameter back to the browser:

app.get('/test/:num/', function(req, res) { res.send(req.params.num); });

Pass some number in the route. As an answer, return the square of this number to the browser.

Multiple route options

You can add more than one parameter to a route:

app.get('/test/:num1/:num2/', function(req, res) { res.send(req.params.num1 + ' ' + req.params.num2); });

Pass two numbers in the route. As an answer, return the sum of these numbers to the browser.

Pass three numbers in the route. As an answer, return the sum of these numbers to the browser.
