Options when setting cookies in Express

When setting cookies, you can pass an object with options as the third parameter:

res.cookie('test', 'abcde', {});

Let's see what options you can specify in it.

domain option

Optiondomainmanages the domains that the cookie is associated with. This allows you to bind a cookie to a specific subdomain.

Example:

res.cookie('test', 'abcde', { domain: 'site.com', });

path option

Optionpathmanages the site URLs for which cookies will be installed.

For example, let's set a cookie for pages whose URLs begin with/test/:

res.cookie('test', 'abcde', { path: '/test/', });

Now let’s set cookies for all pages of the site:

res.cookie('test', 'abcde', { path: '/', });

maxAge option

OptionmaxAgedetermines how long (in milliseconds) the browser should store a cookie before it is automatically deleted. For example, let's set the cookie for one hour:

res.cookie('test', 'abcde', { maxAge: 1000 * 60 * 60, });

Secure option

Optionsecureindicates that this cookie will only be sent through a securehttpscompound:

res.cookie('test', 'abcde', { secure: true, });

httpOnly option

OptionhttpOnlyin meaningtruePrevents client JavaScript from changing this cookie. Changes can only be initiated by the server. Example:

res.cookie('test', 'abcde', { httpOnly: true, });
