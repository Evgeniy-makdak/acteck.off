Removing data from the database

Let's now delete the entry from the database by specifying it in the route parameterid.

Let us have a route in which the variabledelpassed as a parameter:

app.get('/remove/:del', async function(req, res) { res.render('delete'); });

Let this parameter contain information aboutidthe user to be deleted. After declaring a variabledellet's pass it to the SQL query through interpolation as${del}:

app.get('/remove/:del', async function(req, res) { let del = req.params.del; let query = `DELETE * FROM users WHERE id=${del}`; res.render('delete'); });

It would be convenient to display a message on the page with the view indicating successful deletion or an error. To do this, let's declare a variablemessage, which will be empty at first. Next we write the constructiontry-catch. In the blocktryvariables will be declaredresultsAndfields. If the operation is successful, the message will contain information about whichidthe user has been deleted. If an error occurs, its text will also be sent to the view:

app.get('/remove/:del', async function(req, res) { let del = Number(req.params.del); let query = `DELETE FROM users WHERE id=${del}`; let message = ''; try { let [results, fields] = await connection.query(query); message = `user with id=${del} is deleted`; } catch (error) { message = error.message; } res.render('delete', {message: message}); });

In the view filedelete.hbslet's write the code with a variablemessage:

<p>{{ message }}</p>
