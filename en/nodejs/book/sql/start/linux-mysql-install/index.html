Installing MySQL on Linux OS

The Linux OS includes quite a few different distributions. A distribution is an operating system based on the Linux kernel and includes various components (utilities, graphical interfaces, libraries and other software packages).

Ubuntu is one of the most popular Linux distributions. Therefore, the installation of MySQL will be considered using the example of its latest stable version at this moment -22.04. If you use other distributions, then to install MySQL I recommend going todocumentation.

The installation will be carried out through a special package manager APT. First, let's run the command to update the package manager:

sudo apt update

Next, download the packagemysql-server:

sudo apt install mysql-server

Now let's start the servermysqlwith the following command:

sudo systemctl start mysql.service

Next, let's start setting up the MySQL configuration. Let's run the secure installation first:

sudo mysql_secure_installation

Messages will appear in the terminal asking you to make changes to your MySQL security settings. The first message will ask you to configure the password strength checker plugin for logging into the database:

Securing the MySQL server deployment. Connecting to MySQL using a blank password. VALIDATE PASSWORD COMPONENT can be used to test passwords and improve security. It checks the strength of password and allows the users to set only those passwords which are secure enough. Would you like to setup VALIDATE PASSWORD component? Press y|Y for Yes, any other key for No: Y There are three levels of password validation policy: LOW Length >= 8 MEDIUM Length >= 8, numeric, mixed case, and special characters STRONG Length >= 8, numeric, mixed case, special characters and dictionary file Please enter 0 = LOW, 1 = MEDIUM and 2 = STRONG: 2

Next, you will need to set a password for the MySQL root user. Enter and confirm your secure password:

Please set the password for root here. New password: Re-enter new password:

If you used the password checker plugin, a message will appear indicating the strength of your new password. You will then be asked if you want to continue using this password or if you want to enter a new one:

Estimated strength of the password: 100 Do you wish to continue with the password provided?(Press y|Y for Yes, any other key for No) : Y

To work safely with the database, it is recommended to create a new user. First let's get into the MySQL environment:

sudo mysql

Next, create a user who will be identified by password:

CREATE USER 'username'@'host' IDENTIFIED WITH authentication_plugin BY 'password';

For example, let's create a useradmin, which will enter the database using a password'123456':

CREATE USER 'admin'@'localhost' IDENTIFIED WITH authentication_plugin BY '123456';

After creating a new user, you can grant him privileges (permissions) to work with data. The general syntax for assigning privileges is:

GRANT PRIVILEGE ON database.table TO 'username'@'host';

MeaningPRIVILEGEdetermines what actions the user is allowed to perform on the specified database and table. You can grant multiple privileges by separating them with a comma:

GRANT CREATE, ALTER, DROP, INSERT, UPDATE, INDEX, DELETE, SELECT, REFERENCES, RELOAD on *.* TO 'admin'@'localhost' WITH GRANT OPTION;

You can also assign all privileges to a user at once:

GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost' WITH GRANT OPTION;

After that we write the commandFLUSH PRIVILEGES. It is needed to free up server memory:

FLUSH PRIVILEGES;

Then exit the MySQL environment:

exit

Next time you can log into the MySQL database using the following command in the terminal:

mysql -u admin -p

Let's check if the MySQL server is running. To do this, write the following command:

systemctl status mysql.service

After executing the command, the following message should appear:

● mysql.service - MySQL Community Server Loaded: loaded (/lib/systemd/system/mysql.service; enabled; vendor preset: enabled) Active: active (running) since Mon 2025-12-31 16:04:39 UTC; 2h 36min ago Process: 2593 ExecStartPre=/usr/share/mysql/mysql-systemd-start pre (code=exited, status=0/SUCCESS) Main PID: 2601 (mysqld) Status: "Server is operational" Tasks: 38 (limit: 1119) Memory: 354.3M CPU: 19.944s CGroup: /system.slice/mysql.service └─2601 /usr/sbin/mysqld

If MySQL does not start, then you need to run the following command:

sudo systemctl start mysql

Install MySQL. Make sure in your terminal that MySQL is active.
