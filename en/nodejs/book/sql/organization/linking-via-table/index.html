Linking via link table in NodeJS

Let now the user be in different cities. In this case, the table with users could look like this:

users
id name city
1 user1 city1, city2, city3
2 user2 city1, city2
3 user3 city2, city3
4 user4 city1

It is clear that storing data this way is incorrect - cities need to be placed in a separate table. Here it is:

cities
id name
1 city1
2 city2
3 city3

However, we need to make sure that each user can link to several cities. This cannot be done using two tables.

We will need to introduce the so-calledcommunication table, which will connect the user with his cities.

Each entry in this table will store a connection between the user and one city. Moreover, for one user in this table there will be as many records as in how many cities he has been to.

Here is our connection table:

users_cities
id user_id city_id
1 1 1
2 1 2
3 1 3
4 2 1
5 2 2
6 3 2
7 3 3
8 4 1

The table with users will store only user names, without connections:

users
id name
1 user1
2 user2
3 user3
4 user4
5 user5

Requests

Let's make a request with which we will pull out users along with their cities. To do this, we need to make two joins: the first join will attach the connection table to the users, and the second join will connect the cities via connections:

SELECT users.name as user_name, cities.name as city_name FROM users LEFT JOIN users_cities ON users_cities.user_id=users.id LEFT JOIN cities ON users_cities.city_id=cities.id

Query result

The result of our query will contain the name of each user as many times as there are cities he is associated with:

[ { user_name: 'user1', city_name: 'city1' }, { user_name: 'user1', city_name: 'city2' }, { user_name: 'user1', city_name: 'city3' }, { user_name: 'user2', city_name: 'city1' }, { user_name: 'user2', city_name: 'city2' }, { user_name: 'user3', city_name: 'city2' }, { user_name: 'user3', city_name: 'city3' }, { user_name: 'user4', city_name: 'city1' }, { user_name: 'user5', city_name: null }, { user_name: 'user6', city_name: null } ]

It would be more convenient to convert such an array and turn it into the following object:

{ user1: [ 'city1', 'city2', 'city3' ], user2: [ 'city1', 'city2' ], user3: [ 'city2', 'city3' ], user4: [ 'city1' ] }

Let's write code that performs this conversion. Let's make an empty objectuserCities, to which we will gradually add data about users and the cities they visited. Using the methodforEachto everyonerow(row) ofresultsLet's use the arrow function. In the body of this function we will declare two variables:userNameAndcityName, which will store the user name and city name. Next, we will write down the condition - if the user is not inuserCities, then it will be added to this object as a key. We also indicate the condition that ifcity_namenot equalnull, then it will be added as a key value:

let [results, fields] = await connection.query(query); let userCities = {}; results.forEach(row => { let userName = row.user_name; let cityName = row.city_name; if (!(userName in userCities)) { userCities[userName] = []; } if (cityName !== null) { userCities[userName].push(cityName); } }); console.log(userCities);
