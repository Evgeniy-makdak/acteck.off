Linking tables in databases in NodeJS

Let us have a table with user names and the cities in which they live:

users
id name city
1 user1 city1
2 user2 city1
3 user3 city2
4 user4 city1
5 user5 city3
6 user6 city2

The disadvantage of this table is that the same city is repeated several times for different users. This leads to some problems.

First, constant repetition causes the database to take up a lot more space.

Secondly, it is quite inconvenient to perform operations with cities. For example, we want to display a list of all cities. It won't be possible to do it that easily. We will have to get all users along with their cities, then remove duplicates from the resulting cities and only then will we get this list.

Now let’s imagine that in the database10000users from10cities - for these10cities, we will have to get the entire table from a huge number of rows - it will be a very slow operation with a pointless waste of resources.

Solving the problem

We need to split our table into two: cities will be stored in one, and users in the second. In this case, the table with users will have a columncity_idwhich willreferto the user's city.

So let's make two tables. Table with cities:

cities
id name
1 city1
2 city2
3 city3

Table with users:

users
id name city_id
1 user1 1
2 user2 1
3 user3 2
4 user4 1
5 user5 3
6 user6 2
