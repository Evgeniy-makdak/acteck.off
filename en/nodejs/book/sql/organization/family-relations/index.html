Data Relationships in NodeJS

Let us be faced with the task of protecting fathers and sons. Let each father have only one son, and the son, in turn, can also have one son.

We need to figure out how we will store the data. The first idea that might come to mind is to make two tables:parentsfor fathers andsonsfor sons. Then link these tables with some field:son_idorparent_id.

However, this is not a very good idea - after all, the same person can be both father and son at the same time - and you will have to store it in both tables, which is inconvenient, takes up more space and easily leads to errors.

A better option is to link the table to itself: let's make a tableusers, we will store all users in it and make a field for eachson_id, in which it will be storedidson from the same table:

users
id name son_id
1 user1 2
2 user2 3
3 user3 null

Requests

Let's now write a request that will get the user along with his son.

First, let's just get the users:

SELECT * FROM users

Now let's join the users of their sons. We will join the table to itself, so we need to rename it:

LEFT JOIN users as sons

Now we can specify the relationship as the basis of the table and the renamed one:

LEFT JOIN users as sons ON sons.id=users.son_id

Let's now specify the fields:

SELECT users.name as user_name, sons.name as son_name

Let's put everything together and get the following request:

SELECT users.name as user_name, sons.name as son_name FROM users LEFT JOIN users as sons ON sons.id=users.son_id
