Multiple children in sibling relationships in NodeJS

In the previous lesson, a father could only have one son. This was achieved due to the fact that the father had a connectionson_id.

Let now a father can have several sons. In this case, we can still only make one table that is related to itself. We will only store in the communication field notidson, andidfather. In this case, several users will be able to refer to their father - and thus the father will have several sons:

users
id name father_id
1 user1 3
2 user2 3
3 user3 4
4 user4 null

You can get a user along with his father as follows:

SELECT users.name as user_name, fathers.name as father_name FROM users LEFT JOIN users as fathers ON fathers.id=users.father_id

Let us have users. Every user has a father and mother. Describe the storage structure.

Write a request that will reach the user along with his father and mother.
