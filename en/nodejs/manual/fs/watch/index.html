watch method

Methodwatchsets up observations for changes in a file or folder. In the first parameter of the method we indicate the name of the file we need or set the path to it. In the second optional parameter we specify an object with options. In the third optional parameter we set a listener in which we indicateeventType, indicating the type of changes in the file andfilename, passing the name of the file in which the changes occur.

Syntax

fs.watch(имя файла или путь к нему, [опции], [слушатель]);

Object with options

Option Description
persistent Specifies that the process continues to run while viewing files. Default valuetrue.
recursive Iftrue, watches the folder recursively, that is, with all its contents. By default its value isfalse.
encoding Specifies the character encoding that was used for the file passed to the listener. By default its value is'utf8'.
signal The time after which monitoring of the folder will stop. Default valuenull.

Example

Let's start observing changes in the file asynchronously:

import fs from 'fs'; try { await fs.watch('file.txt'); console.log('file watched'); } catch (err) { console.log(err); }

Example

Now let's set the listener parameters -eventTypeAndfilenameto track what types of changes are happening in our file:

import fs from 'fs'; fs.watch('file.txt', (eventType, filename) => { console.log('file', filename, 'was modified!'); console.log('type of change was:', eventType); }); setTimeout( () => fs.renameSync('file.txt', 'fileNew.txt'), 1000 );
