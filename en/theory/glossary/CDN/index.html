CDN Content Delivery Network

Content Delivery NetworkCDN (Content Delivery Network) is a network of interconnected servers that speeds up the loading of high-load web application pages.

For example, when a user opens a website, all images placed on it must first be downloaded from the server. If the user is geographically far from the site server, then the time it takes to download images can increase significantly. It is in order to avoid this that CDN servers that are geographically located closer to users are used.

The main purpose of a CDN is to open points of presence (POPs) or groups of edge CDN servers in different geographic locations.

How CDN works:

  1. Caching- the process of saving copies of data in order to quickly access them if necessary. In CDN, caching involves storing static website content on multiple network servers.
  2. Dynamic acceleration- reducing server response time when user interacts with dynamic web content. For dynamic acceleration, intermediate CDN servers are used between websites and users.
  3. Edge logic computingrun on a CDN edge server and are designed to speed up data transfer between client and server. Therefore, distributing computing between web servers and network edges helps improve the loading speed and overall performance of websites.

Areas of application of CDN:

  • High-speed delivery or download of content.
  • Real-time streaming, for example to broadcast live streams or news.
  • User scaling, i.e. supporting a large number of visitors to the website at the same time.
