CQS programming principle

Programming principleCQS (Command-query Separation, Command-demand separation) assumes that each function in the project can only execute a command (any active action) or only a request to retrieve data.

For example, in your project there is a function that performs input highlighting, but it should not simultaneously parse the information entered by the user and send it to the database; for this, another function is needed.
