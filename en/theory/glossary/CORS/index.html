CORS mechanism

MechanismCORS (Cross-Origin Resource Sharing, Sharing resources between sources) ensures that the client agent obtains permission to access resources from another server on the source (domain).

At the moment, many pages of modern websites can load various files, incl. images from third-party resources (domains). To ensure the security of such a site, browsers restrict cross-origin requests caused by scripts. Such requests are subject to policyCORS. This policy requires that the site you send the request to must agree to respond. To do this, it must send the appropriate HTTP headers.

Also, to eliminate such blocking, the CORS mechanism is used, which supports cross-domain requests over a special secure connection. In modern browsers, CORS is used in API containers (XMLHttpRequest or Fetch).
