PRINCE2 methodology

MethodologyPRINCE2 (PRojects IN Controlled Environments, Projects in a controlled environment) uses a waterfall model to form project workflow stages. This methodology was developed by the UK government specifically for the implementation of large-scale projects in the IT field, which in turn are associated with traditional marketing projects. However, it is worth noting that using PRINCE2 in small teams and projects has significant disadvantages - work processes will be complicated and take longer than necessary.

The workflow using the PRINCE2 methodology is based on the following steps:

  1. Start of the project.
  2. Project management.
  3. Project initiation.
  4. Project control.
  5. Product transfer management.
  6. Managing stage boundaries.
  7. Closing the project.
