YAGNI programming principle

Programming principleYAGNI (You aren't gonna need it, You won't need it) involves abandoning the redundant functionality of the system you are developing, i.e. if there is no immediate need for a task or method, then it should be abandoned.

When planning a project, following the YAGNI principle, you should not write code that you may need someday, but instead solve the most relevant and important problems first.
