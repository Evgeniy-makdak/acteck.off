Design Pattern

Design Patternrepresents a sample or example of the correct solution to any common problem in software development.

The benefit of using a design pattern is that it reduces the complexity of the workflow because the time required to write code is reduced. All templates already have pre-written names, which also facilitates interaction between developers.

Among the disadvantages, it can be noted that developers are tempted to use the selected template, even when not all of its functions are needed for the product being sold.

There are main types of design patterns:

  • Delegation template (Delegation pattern) transfers responsibility for performing its behavior to a related object.
  • Functional Design Template (Functional design) provides for each module a specific function that is executed with minimal side effect for other modules.
  • Immutable interface (Immutable interface) creates an immutable object.
  • Interface (Interface) provides structuring of programs for easier understanding.
  • Interface marker (Marker interface) is used to indicate the presence or absence of an implementation of an attribute or annotation.
  • Properties container (Property container) adds additional properties to the inner container of the class instead of simply extending the class with new properties.
  • Event channel (Event channel) creates a channel through which messages flow in a publisher-subscriber pattern. For this purpose, this template has a proxy object that is neither a publisher nor a subscriber, but only provides a connection between them.
