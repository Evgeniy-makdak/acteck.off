Heap data type

Data typea bunch (heap) is a portion of RAM that is dynamically allocated for each variable. Moreover, each variable can be called globally throughout the entire application, and not just in a separate thread as in stacks.

Once the work is completed, all the memory occupied by the variables on the heap is freed. The heap size can be set at application startup.

The disadvantage of the heap is that it is slower compared to the stack. For example, the last declared variables will be retrieved last and the system will need more time to find them. However, only the heap data type allows correct work with global variables and dynamically changing objects.
