API component set

Set of componentsAPI (Application Programming Interface, Application programming interface) is used for programs to interact with each other and exchange data. All interactions of API components are carried out using functions, classes, methods, structures, as well as constants of one application, which are accessed by others.

All APIs can be divided into two categories:

  1. APIs built into the browser use data from the browser and the computer environment to work with it in more complex ways. For example, the Geolocation API provides data that you can already use to mark your location on an interactive map.
  2. Third-party APIs are provided by services and sites that you need to find on the Internet yourself. For example, a bank's API can provide you with real-time information on exchange rates.

The main ways the API works:

  • SOAP APIrepresents the simplest protocol and at the same time the least flexible API. The SOAP protocol allows server and client data to be exchanged via XML.
  • RPC APIbelong to the category of remote procedure call systems, in which the client sends a request to the server, and the latter sends a response to the same client.
  • Websocket APIsupports two-way data exchange between the server and client applications, and transmits all information via JSON objects. Websocket API is the latest development of web API.
  • REST APIis currently the most popular and flexible API. In this way of working, client requests to the server are used to run internal functions, which then return data back to the client.
