RUP methodology

MethodologyRUP (Rational Unified Process, Rational Unified Process) is an Internet-based knowledge base with a search engine. The RUP methodology was created by Rational Software.

The RUP methodology is based on the following principles:

  • Early detection and continuous elimination of risks until development is completed.
  • Full focus on customer requirements for the product being developed.
  • Constant readiness to make changes to the project objectives and methods of its implementation.
  • Application of component architecture when working on a project.
  • Monitoring the quality of the project at all stages of its development.
  • The work process on the project takes place in a close-knit team, in which the main role belongs to architects.

Working on a project using the RUP methodology includes the following stages:

  1. Initial stage (Inception), in which the project concept and its economic justification are formed, risks are assessed, and a precedent model is created.
  2. Clarification (Elaboration). At this stage, the future product is analyzed and all its details are documented, the project architecture is designed and tested, the timing and cost of the work are clarified, and, if possible, the main risks are reduced.
  3. Construction (Construction), where the main product development takes place. Upon completion of construction, the first external release of the system occurs and the milestone of initial functional readiness is celebrated.
  4. Implementation (Transition). At this stage, a product has already been created, which, after testing, is transferred to the customer. If the customer is not satisfied with the results obtained, then the product again goes through the cycle, which includes all the above stages.
