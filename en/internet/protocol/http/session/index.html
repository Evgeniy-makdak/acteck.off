Session in HTTP

They work on the basis of cookiessessions. Sessions allow the server to remember the client and store information for it.

For example, a website visitor can add items to a store cart. And after the page is reloaded, the server identifies the client and shows him the products that were pending in this browser.

Let's figure out how it works. As you already know, the HTTP protocol is stateless in the sense that the server cannot distinguish which client the request came from. Therefore, a workaround is used in this case.

Its essence is as follows. When a client first contacts the server, it sets a cookie with a random unique string. This cookie is calledsessionaland usually has a namesessid.

The server also stores a list of all session cookies and binds the necessary data to them. For example, a list of selected products in a store.

The next time the client accesses it, he will send a session cookie to the server and the server will identify him by this cookie and will be able to receive the data that is stored on the server for this client.

The HTTP protocol does not store data between requests. However, the server can distinguish between users. How is this implemented?
