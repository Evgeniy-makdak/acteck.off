Cookie in HTTP

Cookies are small text that a server can store in a user's browser. The server can set several cookies with different names.

Using cookies, the server can identify the user and store each client's data between requests.

Cookies are passed between the client and server using HTTP headers (remember, this is important). Every time a browser makes a request to the server, the browser will send all the cookies for that site to the server.

Using the headerCookiethe client sends cookies to the server on every request:

Cookie: name=john

If more than one cookie was set, the client will send them all in one header, separated by a semicolon:

Cookie: name=john;surn:smit

Using the headerSet-Cookiethe server can set a cookie with the desired name and value:

Set-Cookie: name=john

Open a website and examine the request and response headers. Look there for headers that transmit cookies.

In the browser debugger, on the "Network" tab, find your request. Click on it. In the request details that appear, find the "Cookies" tab. Study it.

In the browser debugger, on the "Application" tab, find the "Cookies" tab. Examine the cookies stored in your browser for this site. Try changing and deleting individual cookies (be careful, you may break authorization; make sure you have a password for this site).
