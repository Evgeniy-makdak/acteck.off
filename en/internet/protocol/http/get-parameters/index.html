GET parameters in the HTTP protocol

In the HTTP protocol, you can pass additional parameters to the site URI. Such parameters are calledGET parameters, and the line of parameters itself is calledQuery String.

The presence of GET parameters does not change the page address. You will be taken to the same page, but with parameters that will be parsed by the server.

To pass parameters after the page address, place a question mark and an ampersand&parameters are listed in the formatname=value. See example:

GET /page/?param1=value1¶m2=value2 HTTP/1.1

Study the URLs of the sites you use. Determine which sites use GET parameters. Try to determine by name the purpose of the parameters you found.
