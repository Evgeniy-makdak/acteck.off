Lack of state in the HTTP protocol

The HTTP protocol does not contain a direct way to make the server remember us. This means that when you see a loaded page in your browser, at that moment the server has already processed our request a long time ago, immediately forgetting about us.

However, when receiving requests, the server cannot distinguish whether these requests were sent by the same client or by different ones. That is, moving through the pages of the site, when opening each new page, the server does not know whether we have performed any actions on the site or not yet.

In real life, as you know from personal experience, websites remember us. We can log in to the site, add goods to the online store, and so on. This is done using special workarounds that allow the server to differentiate between clients. We will study these ways in the future.
