HTTP Requests and Responses

Let the user want to open some page of some website in the browser. To do this, the user either follows a link or enters the URL page into the browser address page. Moreover, both methods are technically equivalent.

After this, the browser sends to the serverrequest (request), which asks the server to serve the page at the specified URL. The server receives the browser request and generatesanswer (response) and sends it back to the browser.

The request and response are simply strings formatted in a special way. The rules for formatting these lines are regulated by the HTTP protocol.

Thus, all interaction between the client and the server comes down to throwing messages, like balls in a game of ping pong.

In this case, initiation of a request is required. This means that the server always sends a response to the browser's request. It cannot be that the server will send something to the browser itself, without a preliminary request from the browser.
