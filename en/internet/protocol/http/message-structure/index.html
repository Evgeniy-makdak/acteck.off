HTTP Message Structure

So, the client and server exchange messages with each other, formatted in a special way. Let's look at the structure of these messages.

Each HTTP message consists of three parts, which are transmitted in the specified order: start line (starting line), headers (headers) and message body (message body).

Technically, the start line is the first line of the message, then there is one header on each new line, then a blank line, and then the body of the request.

In this case, the start line determines the type of message, the headers characterize the body of the message, transmission parameters and other information, and the body contains the message data itself.

The start line and headers are required elements, but the body is not. In this case, the starting lines are different for the request and response.

In the following lessons we will separately analyze each part of the HTTP message.
