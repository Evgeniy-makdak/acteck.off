The !optional tag in @extend in SASS

When expanding a selector, you can catch an error when the directive@extenddidn't work correctly. For example, if you have the following code:

a.info { @extend .main; }

If no selector contains.main, then an error will occur during compilation. In this case, we need to combine sequences of selectors, for which we use the directive@extend.

In addition, there will be an error if the selector contains.mainwill be like this:h1.main, which is explained by the conflict betweenaAndh1.

Therefore, if necessary, you can allow the directive@extenddo not create new selectors using the optional flag!optional, written after the selector. For example:

a.info { @extend .main !optional; }
