Importing files into SASS

When working with SASS files, it may be necessary to transfer code from a document.scss, but without creating a clone of it with the extension.css, i.e. import its fragment.

For example, we have a filenavbar.scsswith the following code:

.active { color: red; }

In order to carry out this type of import, you first need to add an underscore before the file name, like this:_navbar.scss. Then to another filestyle.scsswrite down the directive@import, but in the name of the imported file we do not indicate the underscore, as well as the file extension:

@import "navbar";

And we see that when we compile instyle.cssfile code was imported_navbar.scss:

.active { color: red; }

Create a filemain.scsswith the following content:

.block-content { padding: 10px; margin-left: auto; width: 600px; font-size: 14px; }

Then import its fragment into a filestyles.scssand compile tostyles.css.
