Combining selector sequences into @extend in SASS

If we need to combine sequences of selectors, then we also use the directive@extend:

Consider the following example:

#main .list p { font-weight: bold; } #sub .resume .link { @extend p; }

Note the compiled code in the filecss, when combining two sequences that do not have common selectors, new selectors are automatically formed: one with the first sequence before the second and one with the second sequence before the first:

#main .list p, #main .list #sub .resume .link, #sub .resume #main .list .link { font-weight: bold; }

In the case where sequences have a common selector, they will be merged together and only the differences (if any) will be alternated. In the following example, both sequences have an ID#main:

#main .list p { font-weight: semi-bold; } #main .resume .link { @extend p; }

As a result of compilation, these two identifiers will be combined:

#main .list p, #main .list #sub .resume .link, #sub .resume #main .list .link { font-weight: bold; }

div p { font-size: 12px; padding-bottom: 4px; } link button #card-title { @extend p; }
