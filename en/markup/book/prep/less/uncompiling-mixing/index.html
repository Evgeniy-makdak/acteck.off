Uncompiling mixin in LESS

The class we use as a mixin will also end up in the compilation result.

There are, however, situations when we want to create some piece of code, as a template, for use in other places in the code, and this piece itself will not be used.

To cancel compilation of a mixin, you need to put parentheses after its name, like this:

.mix() { width: 100px; height: 100px; }

Tell us what the result of compiling the following code will be:

.mix() { color: white; background: black; } p { .mix; width: 300px; }

Tell us what the result of compiling the following code will be:

.mix1() { color: white; background: black; } .mix2() { width: 300px; height: 300px; } p { .mix1; .mix2; }

Make a mixin.tnr, which will specify the font familyTimes New Roman.

Make a mixin.df, which will set the propertydisplayin meaningflex.

Make a mixin.bold, which will set the propertyfont-weightin meaningbold.

Make a mixin.nowrap, which will set the propertywhite-spacein meaningnowrap.
