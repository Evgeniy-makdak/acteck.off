Comma Parameters in LESS

Let's say we have a function that creates a block shadow:

.shadow(@s) { box-shadow: @s; }

As you can see, our function has one parameter. This means that we will pass all the values ​​of our shadow into this parameter, separated by a space, like this:

div { .shadow(1px 1px red); }

In CSS, however, you can specify multiple shadows by separating them with commas. In our case, this, alas, will not work, since the comma will be perceived by the compiler as a separator between function parameters:

div { .shadow(1px 1px red, 1px 1px blue); // выдаст ошибку }

However, there is a way out. We can force LESS to treat our semicolon string as a single parameter. To do this, we need to end the listing of our values ​​with a semicolon:

div { .shadow(1px 1px red, 1px 1px blue;); }
