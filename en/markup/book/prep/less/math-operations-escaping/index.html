Undoing Math Operations in LESS

Sometimes there are situations when we have some kind of mathematical operation that should not be perceived by LESS as a command to calculate. For example, this problem occurs in the CSS functioncalc:

div { width: calc(100% - 10px); }

Fortunately, LESS can handle such places correctly and will not perform the calculation during compilation.

There are, however, situations when the expression that does not require evaluation is located in another place, for example, in a variable, like this:

@exp: 100% - 10px; div { width: calc(@exp); }

In this case, the result of the calculation will be written to the variable, andcalca variable with the calculated value will be sent. To prevent this from happening, we can explicitly specify that the expression must remain unchanged.

To do this, you need to put the expression in quotes and put a tilde sign in front of it:

@exp: ~'100% - 10px'; div { width: calc(@exp); }

Correct the code so that it works correctly:

@exp1: 80% + 10px; @exp2: 30vh - 10px; div { width: calc(@exp1); margin-top: calc(@exp2); }
