Mixins in LESS

In this lesson we will get acquainted withhagfishes(mixing), allowing one class to be included within another. Let's figure out what it is and how to work with it.

Essentially, mixing is the use of some classes inside others, when one class is a template - a specific piece of code that we can use in many places.

How is this convenient? With pure CSS, we would have to type a piece of code wherever it is needed, but in LESS we can write it in one place and then connect it to the necessary places in the code.

Let's look at an example. Let us have the following class:

.mix { width: 100px; height: 100px; }

Let's use this class as a mixin:

p { .mix; color: red; }

As a result, after compilation, the styles of our class will be substituted into the paragraph:

p { width: 100px; height: 100px; color: red; }

Tell us what the result of compiling the following code will be:

.mix { color: white; background: black; } p { .mix; width: 300px; }
