Displaying grids in the browser debugger

When working with grids during the website layout process, it may be necessary to check how correctly the elements are arranged in the grid. In addition, if we make a mistake, we must figure out which element is causing the incorrect placement. For this purpose we can use a debugger in the browser.

Let's look at the grid using the example of a debugger in the Chrome browser. First, select the object we are interested in from the grid and by right-clicking, select the option from the list'Просмотреть код':

Now, in the browser panel that opens on the right, we move the cursor over our grid object and notice that there is a button in the line next to the parent elementgrid:

After pressing the buttongrid, we will see that our object is displayed in a visually understandable grid system, indicating the numbering of all columns and rows:

Now take a look at the panel in the debugger. It contains tabsStyles, Computed, Layoutand so on. Switch from tabStyles, which is active by default, to the tabLayout. You will see a list of your grid settings:

Let's select the first option to displayShow track sizes, which shows the size of the tracks:

Using the optionShow area namesyou can display the names of grid areas if they are given. And by selecting the optionExtend grid lineswe will see track lines extended beyond the grid:
