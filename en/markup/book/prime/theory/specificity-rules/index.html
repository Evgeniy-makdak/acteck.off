Specificity Rules in CSS

In the previous example, both selectors were the same and had the same priority. This means that the property that was written below won.

There are also situations when several selectors correspond to one page element. In this case, if there is a conflict of properties, more will winspecific, that is, a more precise selector. Let's look at the rules of specificity.

Rule one

The class always wins over the tag selector:

<p class="text">text</p> p { color: red; } .text { color: green; /* применится этот цвет */ }

:

Rule two

Identifier always beats class:

<p id="elem" class="text">text</p> .text { color: red; } #elem { color: green; /* применится этот цвет */ }

:

Rule three

Under equal conditions, the selector with the most parts wins. For example, if we have two tag selectors, then the one with more tags will win:

<div> <p>text</p> </div> p { color: red; } div p { color: green; /* применится этот цвет */ }

:

If there are two class selectors, then the one with more classes will win (if there are tag selectors, they have no effect):

<div class="block"> <p class="text">text</p> </div> .text { color: red; } .block .text { color: green; /* применится этот цвет */ }

:
