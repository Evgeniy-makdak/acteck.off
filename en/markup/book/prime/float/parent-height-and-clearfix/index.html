Clearfix and parent height in CSS

Let's leave one div with classparent, remove the text from it, leaving only the floating image. As you already know, in this case the height of the div will collapse to zero, leaving only the top and bottom border. The picture will appear behind our div below:

<div class="parent"> <img src="img.png" alt=""> </div> .parent { border: 1px solid red; } .parent img { float: left; }

:

Let's make the floating image expand our div in height. To do this, we use a tricky trick - after the picture we put divs without text with a classclearfix.

Our image is a floating element and does not expand the parent's height, but the clearfix div is not a floating element and affects the height of the parent.

Since div-clearfix is ​​given the propertyclear, it will turn out that it will push the picture down and stand under it, while expanding the parent in height.

The div-clearfix itself is empty and not visible on the screen, but at the same time it expands the parent in height.

So let's try:

<div class="parent"> <img src="img.png" alt=""> <div class="clearfix"></div> </div> .parent { border: 1px solid red; } .parent img { float: left; } .clearfix { clear: both; }
