Vendor prefixes in CSS

In the past, the situation among browsers was that before a CSS property appeared in the specification, browsers would implement a trial version of that property with a special prefix calledvendor prefix.

Let's use the property as an examplebox-sizingLet's see what it looks like:

p { box-sizing: border-box; }

This property began to be supported only with Firefox29, but since Firefox2 it was available with the prefix-moz:

p { -moz-box-sizing: border-box; }

Other browsers had similar prefixes:-moz - Mozilla Firefox, -webkit- browsers based on the Webkit and Blink engines,-o- Opera on the Presto engine,-ms - IE.

Thus, the most cross-browser option for the property isbox-sizing, using vendor prefixes, looked something like this:

p { -webkit-box-sizing: border-box; -moz-box-sizing: border-box; -ms-box-sizing: border-box; -o-box-sizing: border-box; box-sizing: border-box; }

Currently, browsers have decided to abandon prefixes due to their inconvenience. And now instead of them they use the so-calledflags. The properties are now embedded in the browser but are disabled by default, however they can be enabled in the browser settings by checking the appropriate box in the preferences. This is done so that developers have the opportunity to play with the new property before its official release.
