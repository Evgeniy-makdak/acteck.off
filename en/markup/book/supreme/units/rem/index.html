rem units in CSS

Unitsemnot always convenient. The problem is that if you change the font size of one tag, it will change for all its descendant tags, which calculate their size relative to it.

Therefore, units were introducedrem. The values ​​of these units are always calculated relative to the font size specified for the taghtml.

As you already know, by default this value is16px:

html { font-size: 16px; }

Let's look at an example. Let us have these tags:

<div> <p> text </p> </div>

Let's give them the dimensions inrem:

div { font-size: 2rem; /* соответствует 32px */ } p { font-size: 2rem; /* соответствует 32px */ margin: 2rem; /* соответствует 32px */ }

Let us have HTML code for which we will solve problems:

<main> <h1>header</h1> <section> <h2>header</h2> <p> text </p> <p> text </p> </section> <section> <h2>header</h2> <p> text </p> <p> text </p> </section> </main>

Rewrite toremall units specified in pixels:

main { margin: 64px auto 32px; } h1 { font-size: 32px; } section { font-size: 16px; margin-bottom: 32px; } h2 { font-size: 24px; margin-bottom: 32px; } p { font-size: 20px; margin-bottom: 12px; }
