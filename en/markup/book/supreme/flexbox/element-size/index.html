Flex element size along the main axis

PropertieswidthAndheightset the width and height of the flex element regardless of the direction of the axes. That is, if the axis is horizontal, thenwidthwill set the width, but if the axis is vertical, thenwidthwill still set the width. Sometimes this behavior is not convenient.

There is a special property in the flex modelflex-basis, which specifies the element's size along the major axis. This means that if the main axis is horizontal, this property will set the width of the elements, and if it is vertical, then the height. This property must be set to the flex elements themselves, and not to their parent. Let's look at examples.

Let now the main axis be horizontal, andflex-basismatters50px. In this case, the width of the elements will be50px:

<div class="parent"> <div class="child">1</div> <div class="child">2</div> <div class="child">3</div> </div> .parent { display: flex; flex-direction: row; /* ось горизонтальна */ width: 300px; height: 300px; border: 1px solid red; } .child { flex-basis: 50px; /* размер элемента */ border: 1px solid green; }

:

Let's now flip the axis without changing the property valueflex-basis. In this case, the height of the elements will be50px:

<div class="parent"> <div class="child">1</div> <div class="child">2</div> <div class="child">3</div> </div> .parent { display: flex; flex-direction: column; /* ось вертикальна */ width: 300px; height: 300px; border: 1px solid red; } .child { flex-basis: 50px; border: 1px solid green; }

:

Do5flex blocks. Give them a width along the main axis in100px. Look at the behavior of blocks along different axes.

If the axis is horizontal and the block is given the propertyflex-basisand at the same time a propertywidth, Thatflex-basiswill have priority. Check it out.

If the axis is vertical and the block is given the propertyflex-basisand at the same time a propertyheight, Thatflex-basiswill have priority. Check it out.
