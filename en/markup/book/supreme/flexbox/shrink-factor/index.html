Compressibility factor for flex blocks in CSS

In the previous lesson, we found out that when flex elements don't have enough space, they begin to shrink. The extent to which the total width is greater than the width of the parent is callednegative free space.

Technically, compression means that a piece of the width of each element is plucked off so that all the elements fit into their parent.

Let's do some calculations. Let, for example, we have4element width200px. Let the width of the parent be equal to700px. It turns out that the total width of the elements is equal to:

200px * 4 = 800px

This is the width at100pxgreater than the width of the parent. Let's calculate how much of each element needs to be plucked off so that the elements fit into their parent:

100px / 4 = 25px

That is, the width of the elements will be:

200px - 25px = 175px

Implement the described blocks and check by measurement that the width of the elements will actually be equal to what we calculated.
