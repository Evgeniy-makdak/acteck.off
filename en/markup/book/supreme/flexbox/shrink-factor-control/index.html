Adjusting compression of flex blocks in CSS

By default, pieces proportional to the width of the element are plucked from all elements. However, you can make it so that larger or smaller pieces are pinched off from some elements. There is a special property for thisflex-shrink.

This property represents a certain weight by which the width of the element will be multiplied when calculating the plucked piece using the formula already given above. For example, if the width of the element is200px, and hisflex-shrinkequals3, Thatweighted(i.e. multiplied by weight) the width of the element will be equal to:

200px * 3 = 600px

Formula taking into accountflex-shrinkwill look like this:neg. St. space * (weighted element width / sum of all weighted element widths).

Let's look at an example. Let us have4element. Let the width of the first element be400px, Aflex-shrinkequals2, the width of the remaining elements is200px, and theirflex-shrinkequals1. Let the width of the parent900px.

The total width of the elements is:

400px + 3 * 200px = 1000px

Negative free space will be equal to:

1000px - 900px = 100px

The total weighted width of the elements is:

400px * 2 + 200px * 1 + 200px * 1 + 200px * 1 = 1400px

The weighted width of the first element is:

400px * 2 = 800px

The following piece will be plucked from the first element:

100px * (800px / 1400px) = 100px * 0.57 = 57.14px

The width of the element will be:

400px - 57.14px = 342.86px ~ 343px

The weighted width of each of the remaining elements is:

200px * 1 = 200px

The following piece will be plucked from each element:

400px - 57.14px = 342.86px ~ 343px

The width of the element will be:

200px - 14.2px = 185.8px ~ 186px

Implement the described blocks and check by measurement that the width of the elements will actually be equal to what we calculated.
