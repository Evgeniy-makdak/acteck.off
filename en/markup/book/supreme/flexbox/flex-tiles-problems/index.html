Problems with flex tiles in CSS

The first problem with the tile is, in general, immediately obvious - it is very inconvenient that the parent has to explicitly set the height. After all, we may well have such a situation that the number of blocks will change dynamically and their number will be either less or more than9.

But the second problem will arise if the number of blocks is such that there is an insufficient number of blocks in the last row. In this case, the last row will look disgusting:

<div class="parent"> <div class="child">1</div> <div class="child">2</div> <div class="child">3</div> <div class="child">4</div> <div class="child">5</div> <div class="child">6</div> <div class="child">7</div> <div class="child">8</div> </div> .parent { display: flex; flex-direction: row; justify-content: space-between; flex-wrap: wrap; width: 320px; border: 1px solid red; } .child { box-sizing: border-box; width: 100px; height: 100px; border: 1px solid green; }

:

It turns out that even horizontally the valuespace-betweendoesn't work well in our case.

To summarize: if the number of your children always fits normally into the parent, then the tile onspace-betweenquite a convenient thing. Otherwise, you'll have to come up with something else.

Dans12elements. Make them into tiles4elements in a row with the width of each element in100pxand the distance between them in20px.

Dans12elements. Make them into tiles3elements in a row with the width of each element in150pxand the distance between them in10px.
