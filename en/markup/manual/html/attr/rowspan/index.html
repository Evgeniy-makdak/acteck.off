rowspan attribute

Attributerowspanmanages the associationrowsin HTMLtable. Applies to tagsthAndtd.

Accepted value: integer starting from1.

How to use: add this attribute to any celltdorth. If, for example, you setrowspanin meaning2- then the cell to which this is specified will occupy two rows in the height of the table. In this case, the cells from the row below will not go anywhere, our expanded cell will displace them and the table will fall apart. To prevent this from happening, you need to remove one of the cells from the row below. See examples for better understanding.

Example. Table without joins

Let's look at the table without joins that we will work with next:

<table> <tr> <td>cell1</td> <td>cell2</td> <td>cell3</td> </tr> <tr> <td>cell4</td> <td>cell5</td> <td>cell6</td> </tr> <tr> <td>cell7</td> <td>cell8</td> <td>cell9</td> </tr> </table> table, td, th { border: 1px solid black; }

Example. Expand the cell by two rows

Now let's expand the cell1into two rows, giving it the attributerowspanin meaning2. We will see that it displaces cells from the row below it and the table falls apart:

<table> <tr> <td rowspan="2">cell1</td> <td>cell2</td> <td>cell3</td> </tr> <tr> <td>cell4</td> <td>cell5</td> <td>cell6</td> </tr> <tr> <td>cell7</td> <td>cell8</td> <td>cell9</td> </tr> </table> table, td, th { border: 1px solid black; }

Example. Let's expand the cell without collapsing the table

To fix the problem with the table falling apart from the previous example, let's try to delete one of the cells from the second row (and this is not necessarily the cell4, which is located directly below our cell1, you can delete any one, we will delete the cell with the number5):

<table> <tr> <td rowspan="2">cell1</td> <td>cell2</td> <td>cell3</td> </tr> <tr> <td>cell4</td> <td>cell6</td> </tr> <tr> <td>cell7</td> <td>cell8</td> <td>cell9</td> </tr> </table> table, td, th { border: 1px solid black; }
