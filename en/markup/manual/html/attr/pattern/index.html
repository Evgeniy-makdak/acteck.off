pattern attribute

Attributepatternspecifies HTML input field validationformsByregular expression.

Regular expressions are special commands that allow you to create almost any validation rules. You can find out more about them inPHP regular expression bookor inbook on JavaScript regular expressions.

When you try to submit a form if the field with this attribute is not filled in, the browser will not allow you to submit the form and will display an error in the form of a tooltip. Unfortunately, the error text and appearance cannot be changed using HTML or CSS.

Please note that the presence of an attributepatterndoes not free you from checking that the form is filled out correctly on the PHP server side (since attribute protection is easy to bypass).

Attributepatternshould be applied to tagsinputortextarea.

Example

Let's go to the taginputand add the attributepattern, in which we will put a regular expression that checks what is entered into the inputtwo digit number(For example,25).

Enter any number and try clicking the button to submit the form. If a non-two-digit number is entered, the browser will not allow you to submit the form and will display an error message, otherwise the form will be sent:

<form action=""> <input type="text" pattern="\d{2}"> <input type="submit"> </form>

:

Example. Empty field

In the previous example, the browser would only throw an error if the field was not empty (even though an empty field is not a two-digit number). Let's try to make sure that the error is also thrown on an empty field - along with the attributepatternlet's also write the attributerequired:

<form action=""> <input type="text" pattern="\d{2}" required> <input type="submit"> </form>
