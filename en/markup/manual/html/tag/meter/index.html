Tag meter

Tagmeteris used to output some value in a nicely formatted form.

The appearance of the tag is a partially or completely filled stripe. The strip can be green, yellow or red (the color of the strip depends on the attribute values, more on this below):

Tagmetershould be used, for example, in the following situation: you want to display the current temperature value, but the temperature may change from0to100. You should specify the following attributes: attributevaluemust indicate the current temperature value, attributemin- minimum possible value (in our case 0), attributemax- the maximum possible (in our case 100).

Depending on the attribute valuevaluethe strip will be small or large. If you specifyvaluein meaning10, then the strip will be small (since10very close to the minimum - to zero), if you specify the value90, then the strip will be large (since90close to the maximum - 100). If you specify a value of 100, the strip will occupy the entire element (since the value has reached its maximum).

If you specify a value greater than100, then the strip will still occupy the entire element. If you specify0and less - there will be no stripes at all.

Adding the low and high attributes

In the tagmeterthere is more2attribute:lowAndhigh.

Let's consider their work using temperature as an example. Let me remind you that our minimum temperature is0, and the maximum -100. Suppose now we want to make the following ranges - “cold temperature” from0to20, "normal" from20to80and "hot" from80to100.

To set the cold temperature, we indicate the attribute valuelownumber20. It turns out that from zero (this is the attribute valuemin) to20(attribute valuelow) with uslow value area.

To set the hot temperature, we indicate the attribute valuehighnumber80. It turns out that from80(this is the attribute valuehigh) to100(attribute valuemax) with ushigh value area.

Normal temperature will be from20(this is the attribute valuelow) to80(attribute valuehigh). Let's call this areaarea of ​​normal values.

Now, depending on which area the attribute value falls intovalue, the strip will turn green or yellow. Green will be if we hitnormal range, and yellow - if inarea of ​​high or low values.

Adding the optimum attribute

Attributeoptimumdetermines the optimal value. For our example, the optimal temperature may be cold, normal or hot - your choice. Suppose you specify thatoptimal temperature is cold. Now, depending on the attribute valuevalue(that is, depending on the current temperature), the color of the strip will be green, yellow or red.

If the current temperature falls within the region of optimal temperatures (and we indicated that the optimal temperature is cold), then the color of the strip will be green. If the current temperature falls within the range adjacent to the optimal one (in our case, this is normal temperature), the bar will turn yellow. If the current temperature is not in the range adjacent to the optimal one (in our case it is a hot temperature), then the bar will turn red.

Everything will happen in a similar way if you specify hot as the optimal temperature. In this case, if the current temperature is in the area of ​​hot temperatures, the bar will be green; if it is in the area of ​​normal temperatures, it will be yellow; if it is in the area of ​​cold temperatures, it will be red.

If you specify the optimal temperature as the normal temperature, then the bar can only be green (if the current temperature falls within the normal temperature) or yellow (if the current temperature falls within the neighbor of the normal temperature, which is hot or cold).

To indicate which range is optimal, you need to specify as the attribute valueoptimumany number from this range.

In the examples below the attribute valueoptimumfalls intolow value area(i.e. cold temperatures).

Old browsers

Older browsers will not be able to show the tagmeterin a beautiful form and will display the contents of the tag itself instead:

<meter value="10" min="0" max="100">Этот текст покажется в старых браузерах.</meter>

Attributes

The attribute value can be a fractional or a negative number.

Attribute Description
value Sets the current value.Required attribute.
min Sets the minimum possible value. Default value:0.
max Sets the maximum possible value. Default value:1.
low Sets the boundary of the low value area. Low values ​​will be from the number in the attributeminto the number in the attributelow.
high Sets the boundary of the low value area. Low values ​​will be from the number in the attributehighto the number in the attributemax.
optimum Defines the area of ​​optimal values.
