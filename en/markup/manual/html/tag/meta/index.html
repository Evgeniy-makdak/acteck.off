meta tag

Tagmetasets some service commands to the browser or search engines: page encoding, page description for search engines, page author, and so on.

The principle of operation of the tag is as follows (with some exceptions): the command name is specified (in the attributenameor in the attributehttp-equiv), and the command value is specified in the attributecontent. Essentially, this tag contains a group of tags (commands), the general name of which ismeta tags

Tagmetadoes not require a closing tag.

Tagmetashould be used inside a taghead.

Popular use cases

Document encoding

The current encoding standard isutf-8. Theoretically, explicitly (via the tagmeta) you don’t have to specify the encoding - the browser must understand you. In practice, I would not recommend doing this - the encoding may go wrong and instead of the Russian text you will see crocodiles.

In HTML5, the encoding is specified in a simplified form:

<meta charset="utf-8">

Previously, the encoding was set like this (now you shouldn’t do this, it’s outdated, you can find it in outdated textbooks):

<meta http-equiv="content-type" content="text/html; charset=utf-8">

Description of the document

The meta description of the document is intended for search engines; it should contain a brief description of the page. From this description, under certain conditions, a search engine can makesnippetsite pages. A snippet is a short description of a site page during a search. RespectivelyThe meta description of the page should be made the way you want to see the page snippet in searchand for each page of the site there is one.

It should be used like this:

<meta name="description" content="Эта страница рассказывает от мета-тегах.">

Document Keywords

Meta-keywords (keywords) are intended for search engines to indicate the most important words by which a page will be searched. Currently, due to abuse by webmasters, search engines do not attach importance to this meta tag.

You should use it like this (keywords and phrases are listed separated by commas):

<meta name="keywords" content="Метатег, Мета-описание, HTML редирект" >

Redirect (tossing to another page)

You can make sure that when entering a certain page, the user is automatically redirected to another (for example, if the page has moved). Use it like this (5 is the number of seconds after which the transfer will occur):

<meta http-equiv="refresh" content="5; url=http://www.example.com/">

Possible attributes

Attribute Description
name Meta tag name. The value of a meta tag with this name is specified by the attributecontent.
http-equiv Allows you to installHTTP headers, to send them to the browser. Specifies the title name. The header value is specified by the attributecontent.
content Sets the value of an attribute specified by attributenameorhttp-equiv.
charset Specifies the document encoding. Currently the standard isutf-8.
