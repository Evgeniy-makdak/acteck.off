form tag

Tagformcreates a form to send data to the server, where this data will be processed usingPHP language. Also, form data can be processed usingJavaScript language.

The tag serves as a container for tagsinput, textarea, select, button, fieldset. The specified tags do not have to be placed in the tagform, but if this is not done, the form will not be sent to the server.

The form is sent to the server by clicking on the button created by the taginputwith attributetypein meaningsubmit, or on the buttonbutton.

Attributes

Attribute Description
action Where do we send the form, to which page of the site (ours or someone else’s). If left blank, the form will go to the current page. Optional attribute. The attribute can contain a link to a specific location on the page (via#, as in the taga). In this case, after submitting the form, the browser will open the page specified in the attributeaction, at the indicated location.
method The form submission method may bePOSTorGET. The layout designer does not need to know this.
Optional attribute. The default value isGET.
target Indicates the link in which tab to open: in a new tab, in the current tab, or in a frame.
Optional attribute.
enctype Determines how the form data is encoded.
Optional attribute. See attribute for detailsenctype.

target attribute values

Meaning Description
_blank The page will open in a new browser tab.
_self The page will open in the current browser tab.
_parent The page will open in the parent frame.
_top Discards all frames and loads the page in a full browser window.

Default value:_self.

Example form

<form action="" method="GET"> <input type="text"><br><br> <input type="checkbox"> <input type="checkbox" checked> <input type="radio" name="radio"> <input type="radio" name="radio"> <br><br> <textarea></textarea><br><br> <input type="submit"> </form>
