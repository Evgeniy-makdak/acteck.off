input tag

Taginputcreates various HTML form elements: a regular input field, a password field, a checkbox, a radio button, a button.

Taginputmust be inside a tagform. This is not necessary, it is necessary to send data to the server and then process it through PHP.

Does not require a closing tag.

Attributes

Attribute Description
type Specifies the type of input field. See options below.
value Default value in the input field. In the case of a button, sets its text. See attribute for detailsvalue.
placeholder Tooltip in the input field, see attribute for more detailsplaceholder.
name The name of the input field. Needed to get input field data in PHP. For the form to work correctly, the names of the input fields should not coincide with each other (in the same form). If they match, PHP will receive the data from the input field that is below in the HTML code.
disabled Blocks a form element (not just input, but almost any element), for more details, see the attributedisabled.

Type attribute values

Meaning Description
text Creates a regular text input field.
password Creates a text input field for a password. Try typing text into it - it will appear as asterisks.
checkbox Creates a checkbox. See more details.checkbox.
radio Creates a radio switch. See more details.radio.
hidden Creates a hidden input that will not be visible on the screen, but will send the content contained in the attributevaluedata to the server.
button Creates a button. Clicking this button will not submit the form to the server. It can be used insidelinksor throughJavaScript. By default, a button has no text (example of a button without text:), it is specified usingvalue. See also tagbutton, which also makes a button.
submit Creates a button that will send data to the server. The button text depends on the browser, it can be changed usingvalue. See also tag button, which also makes a button.
reset Creates a button that clears a completed form. The button text depends on the browser, it can be changed usingvalue.
file Creates a file selection button. The design of this button cannot be changed via CSS (however, there are clever ways). If you require such a field in the form, then the tagformmust have an attributeenctypein meaningmultipart/form-data.

New values ​​for the type attribute in HTML5

These attribute values ​​are new, only introduced in HTML5, so they may not work in some browsers or may work differently in different browsers.

In case the browser cannot understand the content of the attributetype(for example, if it is not supported or is entered incorrectly), it will treat the element as a regular text input, as if the type had a valuetext.

Check out the examples below in different browsers. Try entering text into the inputs and clicking the send button. If the text is incorrect or the field is empty, the browser will throw an error. For example, if in a field with typeemailenter an incorrect email - the browser will not allow you to submit the form and will display an error message (the text of the error and its appearance in html css cannot be changed). If the field is empty, the browser will also throw an error; this is achieved using the attributerequired:
