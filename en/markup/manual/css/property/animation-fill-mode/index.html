animation-fill-mode property

Propertyanimation-fill-modesets the position at which the animation stops after finishing. By default, the animation returns to its original value and this may not look very nice.

Syntax

селектор { animation-fill-mode: backwards | forwards | both | none; }

Values

Meaning Description
none If an animation delay is set, then during the delay time the element will remain in place, and then jump to0%frame. After the animation ends, the element will not remain in the state where it stopped, but will jump to the initial state.
backwards Causes the element to move after the page has loaded to0%frame, even if a delay is setanimation-delay, and stay there until the animation starts. After the animation ends, the element will not remain in the state where it stopped, but will jump to the initial state.
forwards Tells the browser that after the animation ends, the element will remain in the state where it left off.
both Includes backwards and forwards - after the page loads the element will be set to0%frame, and after the animation ends, the element will remain where it left off.

Default value:none.

Example. Value none

In this examplemargin-leftset to value300pxfor the element, and for the first frame of the animation - in0px. Because there is no delayanimation-delay, then the element after loading the page will be in0px, not in300px. In addition, the animation will jump to the starting value after finishing:

<div id="elem"></div> @keyframes move { from { margin-left: 0; } to { margin-left: 400px; } } #elem { margin-left: 300px; animation-fill-mode: none; animation-delay: 0s; animation-duration: 3s; animation-name: move; animation-iteration-count: infinite; animation-timing-function: linear; border: 1px solid black; width: 50px; height: 50px; }
