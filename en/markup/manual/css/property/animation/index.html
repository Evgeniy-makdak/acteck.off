animation property

Propertyanimation- this property is a shorthand for animation; it allows you to set all properties for animation at once:animation-name, animation-duration, animation-delay, animation-timing-function, animation-iteration-count, animation-direction, animation-fill-mode, animation-play-state.

The order doesn't matter, the timing of the animation isanimation-durationmust come before the delayanimation-delay. Only properties are requiredanimation-nameAndanimation-duration.

The property can also take the valuenone, which disables animation completely. This value is the default value.

Syntax

селектор { animation: значения; }

Example

In this example there will be a delay of3seconds (before each new playback), and then the animation will play for6-ty seconds:

<div id="elem"></div> @keyframes move { from { margin-left: 0px; } to { margin-left: 400px; } } #elem { animation: move 6s infinite 3s linear; border: 1px solid black; width: 50px; height: 50px; }

:

Example. Possible problems

Animation recorded in short form may not work due to its title. Let's look at the following animation:

@keyframes reverse { from { left: 0; } to { left: 300px; } }

Notice I'm using the name of the animationreverse. At first glance, everything is fine, but the animation does not work, because...reverse- a valid keyword for the propertyanimation-direction.

Also, the animation will not work if other keywords are used in short form in the title. But everything works well when using the "full" description form.

To keywords-meaningsanimation-directionthat break animations, it is worth including keywords related to anti-aliasing functions, as well asinfinite, alternate, running, pausedand so on.
