background-origin property

Propertybackground-originspecifies how the background image (namely the image, not the fill) will be placed relative to the element: part of the background image will fit under the border, the background will not fit under the border, or the background will be placed only above the contents of the element (that is,paddingwill move the background away).

Syntax

селектор { background-origin: padding-box | border-box | content-box; }

Values

Meaning Description
border-box The background image will fit under the border.
padding-box The background image will not fit under the border.
content-box The background image will only appear above the content.

Default value:padding-box.

Notes

Propertybackground-origindoesn't work whenbackground-attachmentmattersfixed. Also whenbackground-repeatin meaningrepeatpropertybackground-originalways works as if the valueborder-box.

Example. Default

By default, the background will not go under the border:

<div id="elem"></div> #elem { background-image: url("bg.png"); background-repeat: no-repeat; border: 10px dashed black; padding: 30px; width: 250px; height: 150px; }

:

Example. border-box meaning

Now the background will go under the border:

<div id="elem"></div> #elem { background-origin: border-box; background-image: url("bg.png"); border: 10px dashed black; padding: 30px; width: 250px; height: 150px; }

:

Example. Content-box meaning

And now the background will move away throughpadding:

<div id="elem"></div> #elem { background-origin: content-box; background-image: url("bg.png"); background-repeat: no-repeat; border: 10px dashed black; padding: 30px; width: 250px; height: 150px; }

:

Example. With background-repeat: repeat

Atbackground-repeatin meaningrepeatpropertybackground-originalways works as if the valueborder-box, that is, the background always goes under the border:

<div id="elem"></div> #elem { background-repeat: repeat; background-image: url("bg.png"); border: 10px dashed black; padding: 30px; width: 350px; height: 300px; }

:
