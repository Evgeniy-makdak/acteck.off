Assets in Webpack

Every project has some resources orassets (assets). These assets are images, font files, and the like.

Typically, we want to simply move assets from the project folder to the build folder. But again there is a problem with the cache. For example, the browser caches images. This means that if we change the image in the picture without changing its file name, then site users will see the previous version of the picture, since it is cached for them.

The solution to the problem, as you already know, is to add a hash to the file name. Webpack allows you to copy resource files to the build folder, adding hashes to them and, most importantly, changing the paths to these resources in all files to new ones.

In Webpack, working with assets is done using special asset modules. There are four types:asset/resource, asset/inline, asset/source, asset.

Moduleasset/resourcecopies files from the project folder to the assembly folder and replaces their paths with new ones. Moduleasset/inlinetakes files from the project folder and in the assembly code replaces the file paths withbase64. Moduleasset/sourcetakes files from the project folder and returns them as a text string. Moduleassetchooses whether to copy the file to the build folder or turn it into base. The choice is made depending on the file size. By default, files are more than8kbcollected into files.

The general scheme for using these modules is as follows:

module: { rules: [ { test: /\.png$/, // файлы type: 'asset/resource' // тип модуля } ] },
