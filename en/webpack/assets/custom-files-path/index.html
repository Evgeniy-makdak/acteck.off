Customizing the path to asset files in Webpack

You can set the path to asset files in the build folder. This is done using the settingassetModuleFilename. In this setting, we can specify the folder where assets are stored, as well as a file name template.

In the name template the command[name]stands for file name, command[hash]- hash, and the command[ext]- extension of the initial file.

So let's do the setup:

export default { entry: './src/index.js', output: { filename: 'bundle.js', path: path.resolve('dist'), assetModuleFilename: 'assets/[name].[hash][ext]', // наша настройка }, module: { rules: [ { test: /\.png$/, type: 'asset/resource' } ] }, plugins: [ new HtmlWebpackPlugin(), ], };

Make sure all assets are placed in a folderresources.
