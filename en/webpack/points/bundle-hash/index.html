Hash in the build file in Webpack

We previously discussed the problem of file caching. Using Webpack you can easily deal with this problem. Webpack allows you to add a special name to the file namehash, which is a unique random string generated based on the contents of the assembly file.

This means that with each new build, if changes have been made to the project files, then the hash of the build file will change.

A hash can be added to the file name using a special command[contenthash]. Let's do this:

export default { entry: [ './src/test1.js', './src/test2.js' ], output: { filename: 'build.[contenthash].js', } };

Add a hash to your build file name. Assemble the project. Look at the file name.

Make the change to the project files. Assemble the project. Look at the name of the new file.
