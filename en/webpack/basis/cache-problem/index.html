Browser cache problem

Browsercachesincluded files of CSS styles, JavaScript scripts and images. Caching means that the browser downloads the connected files only the first time the user visits the site. On subsequent visits, these files will no longer be downloaded again, but will be taken fromcachebrowser.

Caching is useful. It was created to increase site loading speed. After all, it is faster for the browser to take a file from its storage than to download it every time from the Internet.

However, speed comes at the cost of inconvenience during development. The fact is that if you change something in your code and then upload the changes to your hosting, all users who have already visited your site will have an old cached copy of the code.

First solution to the problem

To combat this behavior, you need to rename the changed files each time. In practice this is not convenient, so a tricky trick is used. Its essence is that when connecting a file, after the file name we put a question, an equal sign and the version number of your script. This design is calledGET parameter.

The presence of a GET parameter in the file name does not "spoil" the path from the server's point of view, still pointing to the same file. But from the browser's point of view, changing the GET parameter causes the browser to consider the path to the file to have changed and download the file again.

To combat the cache, when you make changes to your code file, you will need to increase the value of the GET parameter by one. See an example of this approach:

<!DOCTYPE html> <html> <head> <meta charset="utf-8"> <title></title> <link rel="stylesheet" href="styles.css?v=1"> <script src="script.js?v=1"></script> </head> <body> </body> </html>

Second solution to the problem

There is also a more advanced approach. It consists of adding random strings to file names, for example, like this:

<!DOCTYPE html> <html> <head> <meta charset="utf-8"> <title></title> <link rel="stylesheet" href="styles.398db7afe3b52e94bb25.css"> <script src="script.1d12c304c284a752cb9a.js"></script> </head> <body> </body> </html>

These lines are calledhashes. The hash is a unique string. It is calculated from the contents of the file in a special way. This means that each text has its own unique hash. If the file text has been changed, then its hash will be different and we will need to change it in the file name.

Of course, calculating hashes and renaming files by hand is not a task. Therefore, this approach is only used if we have some tool that allows us to automatically calculate hashes and rename files, as well as change file names to new ones in HTML files. Webpack allows you to do all this. We will learn this throughout the tutorial.
