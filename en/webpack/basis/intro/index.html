How the Webpack bundler works

As a rule, during development we end up with a lot of JavaScript files containing various pieces of code. These can be parts of our code, or third-party libraries. It turns out that we must connect each such file to the HTML file via the tagscript.

This is not very good, since many connected files slow down the loading speed of the site. Therefore, to speed up loading, you need to put all the code into one file.

However, developing code in one common file is also not very convenient. Therefore, the following approach is currently practiced: the code is developed in separate files, and then, using an assembler, it is assembled into one common file, which is connected to the HTML file.

The individual files areES modules. These modules are connected to other files via the commandimport.

Usually they create a certain main file to which other files are connected. This file is calledentry point.

The assembler enters the entry point and looks at what modules are connected to it. Other modules can also be connected to these modules. The collector follows all connections and collects all the code into one file. This file is calledbundle.

As a rule, the code that a programmer writes is located in the foldersrc, and the collected code is placed in the folderdist.
