Unidirectional data flow in Vue

All input parameters form a one-way binding between the child property and the parent: when the parent property is updated, it will be passed to the child, but not vice versa. This prevents child components from accidentally changing their parent state, which can make your application's data flow difficult to understand.

Additionally, every time the parent component is updated, all input parameters of the child component will be updated with the latest values. This means thatyou shouldn't try to change an input parameter inside a child component. If you do this, Vue will display a warning in the console.

There are usually two cases where the temptation to change an input parameter arises. Let's look at them.

Case one

The input parameter is used to pass the initial value; the child component wants to use it as a local data property later on. In this case, it is best to define a local property in the data that uses the value of the input parameter as an initial one:

props: ['initialCounter'], data() { return { counter: this.initialCounter }; }

Case two

The input parameter is passed as a raw value that needs to be converted. In this case, it is best to define a computed property using an input parameter:

props: ['size'], computed: { normalizedSize: function() { return this.size.trim().toLowerCase(); } }

Comment

Note that objects and arrays in JavaScript are passed by reference, so if the input parameter is an array or an object, then changes within the child component of that object or array will affect the state of the parent and Vue has no way to warn about this. Any mutation of input parameters, including objects and arrays, should be avoided, since ignoring one-way data binding can lead to undesirable results.
