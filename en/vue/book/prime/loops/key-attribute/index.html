Key attribute in Vue

When Vue updates a list of elements rendered by a directivev-for, the default is to use the in-place update strategy. If the order of the elements of an array or object has changed, Vue will not move the DOM elements, but will simply update each element in-place so that it displays the new data at the appropriate index.

To tell Vue how to determine the identity of each element, and thus reuse and arrange existing elements, you need to specify a unique attributekeyfor each element.

Without keys, Vue uses an algorithm that minimizes movement of elements and will try to change/reuse elements of the same type as much as possible. When using keys, elements will be reordered according to the reordering of the keys, and elements whose keys are no longer present will always be deleted/destroyed.

It is recommended to always specify the attributekeyWithv-for, except when the DOM content being iterated is simple, or when deliberately relying on the default update strategy to improve performance.

As a rule, the question of adding keys arises when iterating through an array of objects. In this case, one of the object keys is a unique field, for example,id:

data() { return { users: [ { id: 1, name: 'name1', surn: 'surn1', }, { id: 2, name: 'name2', surn: 'surn2', }, { id: 3, name: 'name3', surn: 'surn3', }, ] } }

Let's loop through the given array, specifying the appropriate keys:

<template> <p v-for="user in users" :key="user.id"> {{ user.name }} {{ user.surn }} </p> </template>

data() { return { products: [ { id: 1, name: 'product1', }, { id: 2, name: 'product2', }, { id: 3, name: 'product3', }, ] } }
