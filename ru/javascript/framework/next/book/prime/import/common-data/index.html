Общие данные компонентов в NextJS

Бывают ситуации, когда несколько компонентов должны получить доступ к одним данным. Для примера пусть у нас есть некоторый массив с юзерами. Пусть на одном роуте мы хотим показать список юзеров, а на другом роуте с динамическим параметром - описание конкретного юзера по его id.

Давайте реализуем описанное. Сделаем следующую файловую структуру:

  • /app/
    • /users/
      • users.js
      • /list/
        • page.jsx
      • /show/[id]/
        • page.jsx

Сделаем отдельный файл с данными юзеров:

export default users = [ { id: 1, name: 'name1', surn: 'surn1', }, { id: 2, name: 'name2', surn: 'surn2', }, { id: 3, name: 'name3', surn: 'surn3', }, ];

Сделаем компонент для показа списка юзеров:

import users from '../users.js'; export default function List() { let list = users.map(user => { return <li> {user.name} </li>; }); return <ul> {list} </ul>; }

Сделаем компонент для вывода определенного юзера по его id:

import users from '../../users.js'; export default function User({params}) { let user = users[params.id]; return <div> <span>{user.id}</span> <span>{user.name}</span> <span>{user.surn}</span> </div>; }

Дан следующий массив:

let prods = [ { id: 1, name: 'prod1', cost: 100, desc: 'desc1', }, { id: 2, name: 'prod2', cost: 200, desc: 'desc2', }, { id: 3, name: 'prod3', cost: 300, desc: 'desc3', }, ];

Сделайте два компонента. Пусть первый показывает список продуктов, а второй - подробное описание продукта.
