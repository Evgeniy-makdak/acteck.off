Создание юзера для БД на хостинге

Пусть вы создали базу данных. Этого, однако, недостаточно, чтобы ваша программа могла подключиться к этой базе. Дело в том, что к базе могут подключаться только юзеры, связанные с этой базой.

Каждая база должна иметь одного или несколько юзеров, связанных с ней. В этом случае наша программа сможет подключиться к заданной базе под заданным юзером.

Разные юзеры нужны, чтобы разграничить права доступа к базе. Технически это означает, что юзеры смогут выполнять не все SQL запросы, а только те, которые разрешены для данного юзера.

Давайте создадим нового юзера и привяжем его к нашей базе данных. Для этого в панели управления хостингом найдите, где создаются новые юзеры для баз данных. Как правило, это делается на той же странице, где и создаются базы данных.

Вам нужно проделать два шага.

Шаг 1. Создайте нового юзера и установите ему логин и пароль.

Шаг 2. Привяжите вашего юзера к вашей базе данных.

В момент привязки вам предложат выбрать привилегии пользователя для данной базы данных. Для безопасности нам следует дать минимум привилегий. Как правило это SELECT, INSERT, UPDATE и DELETE.

Создайте нового юзера базы данных.

Привяжите нового юзера к вашей базе данных. Назначьте ему правильные привилегии.
