Имена файлов сборки в Webpack

С помощью команды [name] можно указать место, куда вставится имя точки входа. Такая возможность позволяет конфигурировать имена файлов сборки. Для примера давайте сделаем так, чтобы файлы сборки имели имена f1.build.js и f2.build.js:

export default { entry: { f1: './src/test1.js', f2: './src/test2.js' }, output: { filename: '[name].build.js', } };

Даны следующие настройки:

export default { entry: { test1: './src/test1.js', test2: './src/test2.js', test3: './src/test3.js' }, };

Модифицируйте настройки так, чтобы итоговые файлы имели имена test1.bund.js, test2.bund.js и test3.bund.js.
