Диаграмма сгорания задач

Диаграмма сгорания задач предназначена для графического представления объема уже проделанной и предстоящей работы.

Диаграмма сгорания задач очень широко применяется в гибкой методологии SCRUM.

Существует два типа Диаграмм сгорания задач:

  1. Диаграмма сгорания работ для спринта - отображает количество решенных и предстоящих задач в текущем спринте.
  2. Диаграмма сгорания работ для выпуска проекта - представляет проделанные и предстоящие задачи, оставшиеся до релиза или выпуска разрабатываемого продукта.

Смотрите также

  • язык моделирования UML,
    который описывает процессы с помощью диаграмм и схем
  • методология SCRUM,
    которая применяется для гибкого управления проектами
  • методология Kanban,
    которая применяется для реализации проекта точно в срок
  • набор концепций Парадигма программирования,
    который определяет подход к программированию
  • схема MVC,
    которая применяется для организации кода по блокам для каждой задачи
  • техника planning-poker,
    которая предназначена для оценки объема и сложности задач
