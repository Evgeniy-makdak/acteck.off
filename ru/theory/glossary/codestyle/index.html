Стандарт оформления кода Code style

Code style в общем смысле представляет собой рекомендации и соглашения о стиле используемого кода.

Как правило, в каждом языке программирования есть свои общепринятые стандарты оформления кода. Эти стандарты как правило регулируют расстановку пробелов, скобок, а также стиль написания имен переменных и функций: camelCase, snake_case, kebab-case, PascalCase.

Соблюдение стандартов кода важно по ряду причин:

  • Зачастую в рабочем процессе чтение кода преобладает над его написанием. Прежде чем дописать код или исправить возникшую ошибку нужно прочитать большой массив уже написанных ранее строк. И здесь большое значение имеет единый стиль кода, который ускорит выполнение этой задачи.
  • Написание и поддержка кода - основа разработки любого вида ПО, на которую приходится около двух третей расходов от общего бюджета.
  • В большинстве случае код пишет не один человек, а целая команда. Причем часто возникают ситуации, когда нужно изменить или дополнить код, написанный другим человеком.
